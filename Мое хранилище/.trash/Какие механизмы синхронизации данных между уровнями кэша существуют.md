Эти механизмы решают разные задачи: обеспечивают видимость изменений переменных, синхронизируют доступ к общим ресурсам или гарантируют атомарность операций.                                                                                                                                                                                                                                                                                                                |
| https://metanit.com/java/tutorial/8.3.php                                                                         | **volatile**<br><br>**Гарантирует видимость изменений переменных** между потоками. Когда переменная объявлена как volatile, происходят: <br><br>- Запись в переменную всегда производится немедленно в основную память.<br>- Чтение volatile-переменной всегда происходит напрямую из основной памяти, аннулируя любое кэшированное значение.<br>- Модель памяти Java (JMM) предотвращает переупорядочивание инструкций вокруг volatile-переменных.<br><br>**Ограничение**: volatile не гарантирует атомарность операций — например, операция <br><br>counter++<br><br> не является атомарной, так как включает несколько шагов: чтение значения, его инкремент, а затем запись нового значения обратно. |
| https://kz.hexlet.io/courses/java-advanced/lessons/sync-primitives/theory_unit                                    | **synchronized**<br><br>**Гарантирует, что только один поток может выполнять операторы синхронизированного блока** одновременно. Когда поток входит в синхронизированный блок, он захватывает монитор этого объекта. Если монитор уже занят другим потоком, текущий поток будет ждать его освобождения.<br><br>**Особенности**:<br><br>- Если один синхронизированный метод вызывает другой синхронизированный метод в том же объекте, поток, который уже захватил блокировку, может войти в этот метод без повторного захвата замка.                                                                                                                                                                    |
| https://habr.com/ru/companies/simbirsoft/articles/812097/                                                         | **locks**<br><br>**Реализуют эксклюзивный доступ к ресурсу** — только один поток может выполнять определённый код в данный момент времени. Блокировки могут быть реализованы как на уровне объекта, так и на уровне класса:<br><br>- **На уровне объекта** — блокировка применяется к конкретному экземпляру класса, только один поток может выполнять синхронизированный нестатический метод или блок кода для этого объекта в любой момент времени.<br>- **На уровне класса** — блокировка относится ко всему классу, а не к отдельным экземплярам, только один поток может выполнять синхронизированный статический метод или блок кода для данного класса в любой момент времени.                    |

Если операция является атомарной, то она либо выполняется полностью, либо не выполняется вообще.<br><br>**Пример**: механизм <br><br>CAS (Compare-And-Swap)<br><br> обеспечивает безопасную операцию записи, которая выполняется только в том случае, если значение переменной соответствует ожидаемому.                                                                                                                                                                                                                                                            |
