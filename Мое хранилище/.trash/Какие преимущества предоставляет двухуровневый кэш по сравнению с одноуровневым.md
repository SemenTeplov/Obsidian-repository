| https://www.cyberforum.ru/blogs/2408867/9933.html                                                   | **Двухуровневый кэш в Java предоставляет преимущества по сравнению с одноуровневым** в разных сценариях кэширования данных. Это связано с тем, что двухуровневый кэш позволяет: <br><br>- **Обеспечить быстрый доступ** к часто запрашиваемым данным за счёт локального кэша.<br>- **Снизить нагрузку на базу данных** — большинство запросов обслуживается из кэша, уменьшая число обращений к БД.<br>- **Обеспечить масштабируемость** — удалённый кэш позволяет хранить больше данных и делиться ими между серверами.                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| https://proglib.io/p/kak-vybrat-strategiyu-keshirovaniya-razbor-7-populyarnyh-algoritmov-2025-03-20 | **Одноуровневый кэш**<br><br>**Локальный кэш (первый уровень)** — располагается на том же сервере, где работает приложение (чаще всего — в оперативной памяти). Обеспечивает быстрый доступ к часто используемым данным, но **не позволяет совместно использовать объект между разными сеансами**. Например, если один и тот же объект требуется для разных сеансов, все они отправляют базу данных для его загрузки, что увеличивает трафик базы данных.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| https://dzen.ru/a/ZZQViKqN-jZtYsdV                                                                  | **Двухуровневый кэш**<br><br>**Удалённый кэш (второй уровень)** — располагается на отдельном сервере или кластере серверов, представляет собой общий кэш для нескольких приложений. Доступ к данным медленнее, чем у локального кэша, но **позволяет кэшировать большие объёмы данных**. Например, в Hibernate кэш второго уровня связан с объектом-фабрикой сессий (Session Factory) и доступен для всего приложения, а не для одного сеанса.<br><br>**Процесс работы двухуровневого кэша**:<br><br>1. Запрос клиента → проверка локального кэша: если данные найдены, они возвращаются мгновенно, если не найдены — переходят к следующему шагу.<br>2. Проверка удалённого кэша: если данные найдены, они возвращаются и сохраняются в локальном кэше для будущих запросов, если не найдены — переходят к следующему шагу.<br>3. Запрос к основной базе данных: полученные данные возвращаются клиенту, одновременно данные сохраняются в локальном и удалённом кэшах. |
| https://www.alachisoft.com/ru/blogs/using-ncache-as-hibernate-second-level-java-cache/              | **Сравнение**<br><br>**Одноуровневый кэш** не позволяет совместно использовать объект между разными сеансами, что может вызывать проблемы с масштабируемостью. **Двухуровневый кэш** решает эту проблему, так как большая часть данных обслуживается из кэша второго уровня, а не из кэша первого уровня.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| https://habr.com/ru/articles/135176/                                                                | **Инструменты**<br><br>**Двухуровневый кэш** реализуется с помощью специализированных библиотек и фреймворков, например:<br><br>- **Ehcache** — поддерживает многоуровневое кэширование, при заполнении памяти редко используемые записи каскадно перемещаются на более низкие уровни.<br>- **Hibernate** — предоставляет инфраструктуру кэширования, включающую кэш первого и второго уровня, что позволяет использовать распределённое кэширование.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
