Когда процессору нужно обратиться в память для чтения или записи данных, он сначала проверяет, доступна ли их копия в кэше. В случае успеха проверки процессор производит операцию, используя кэш, что быстрее использования более медленной основной памяти.<br><br>**Виды**:<br><br>- **Кэш инструкций** — для ускорения загрузки машинного кода.<br>- **Кэш данных** — для ускорения чтения и записи данных. Часто реализуется в виде многоуровневого кэша (L1, L2, L3, L4).                                                                                           |
| https://www.dbserv.ru/blog/kak-rabotaet-kehshirovanie-baz-dannyh                                                           | **Дисковый кэш**<br><br>**Использует часть оперативной памяти (ОЗУ) для временного хранения блоков данных**, которые были недавно считаны с диска или записаны на него. Может работать в режиме обратной записи: данные сначала записываются в кэш, а затем сбрасываются на диск, или в режиме сквозной записи: данные одновременно записываются в кэш и на диск.<br><br>**Типы**:<br><br>- **Кэш операционной системы** — управляется ядром операционной системы и используется для буферизации операций дискового ввода-вывода.<br>- **Кэш дискового контроллера** — некоторые дисковые контроллеры имеют встроенную кэш-память для ускорения операций с диском. |
| https://ru.wikipedia.org/wiki/Кэш_процессора                                                                               | **Кэш веб-сервера**<br><br>**Сохраняет копию полученного ресурса** для возврата этой копии в ответ на дальнейшие запросы. Запрос на ресурс, уже имеющийся в веб-кеше, перехватывается, и вместо обращения к исходному серверу выполняется загрузка копии из кеша. <br><br>**Виды**:<br><br>- **Приватные кеши** — предназначены для отдельного пользователя.<br>- **Кеши совместного использования** — хранят копии, которые могут направляться разным пользователям.                                                                                                                                                                                              |

Это позволяет не обращаться каждый раз к основному хранилищу, ускоряя доступ и снижая нагрузку на базу данных.<br><br>**Типы**:<br><br>- **Кэш запросов** — сохраняет результаты SQL-запросов. Если один и тот же запрос выполняется многократно, его результат можно сохранить в кэше и доставлять без повторного обращения к БД.<br>- **Кэш объектов** — работает на уровне бизнес-логики.<br>- **Кэш страниц** — используется для хранения готовых HTML-страниц.                                                    |
