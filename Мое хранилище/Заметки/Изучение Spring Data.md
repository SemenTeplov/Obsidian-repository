### Что такое Spring Data?

| Под темы             | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| -------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение          | Spring Data —  это  **подпроект экосистемы Spring Framework**,  предназначенный для упрощения работы с различными  системами хранения данных (базами данных,  NoSQL‑хранилищами и др.).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| Основная цель        | Обеспечить **унифицированный, простой в использовании API** для доступа к данным — независимо от типа underlying‑хранилища. При этом сохраняется возможность использовать специфические особенности конкретной СУБД.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| Ключевые особенности | 1. **Репозитории (Repository‑ориентированное программирование)**<br>    - Абстрагируют низкоуровневые детали работы с БД.<br>    - Предоставляют высокоуровневый объектно‑ориентированный интерфейс.<br>    - Включают готовые методы для типовых операций (CRUD).<br>        <br>2. **Автогенерация запросов**<br>    <br>    - SQL‑запросы (и запросы для NoSQL) формируются автоматически на основе имён методов репозитория.<br>    - Пример: метод `findByName(String name)` автоматически генерирует запрос на поиск по полю `name`.<br>        <br>3. **Гибкая конфигурация**<br>    <br>    - Настройка через конфигурационные файлы или программный код.<br>    - Поддержка различных стратегий подключения и транзакций.<br>        <br>4. **Расширяемость**<br>    <br>    - Возможность кастомизации репозиториев.<br>    - Поддержка нативных запросов (SQL, JPQL и др.).<br>        <br>5. **Интеграция с экосистемой Spring**<br>    <br>    - Совместная работа с другими компонентами Spring (DI, транзакции, AOP).<br>    - Не зависит от Spring Web — может использоваться в десктопных и других приложениях.<br>        <br>6. **Аудит данных**<br>    <br>    - Отслеживание авторов и времени изменений объектов в БД. |

| Основные преимущества | 1. **Резкое сокращение объёма рутинного кода**<br>    <br>    - Автоматически генерируются  реализации CRUD‑операций (создание, чтение, обновление, удаление).    <br>    - Не требуется писать шаблонный код для базовых запросов — достаточно объявить интерфейс репозитория.  <br>    - Возможность комбинировать автоматические и кастомные запросы.<br>    - Поддержка нативных SQL/NoSQL‑запросов через `@Query`.<br>    - Кастомизация репозиториев через наследование и добавление собственных методов.<br>        <br>7. **Улучшенная читаемость и поддерживаемость кода**<br>    <br>    - Интеграция с Spring Cache для снижения нагрузки на БД.<br>    - Возможность использования Redis, Ehcache и других провайдеров.<br>    - Ускорение отклика системы за счёт хранения часто запрашиваемых данных.<br>        <br>9. **Упрощённая обработка ошибок**<br>    <br>    - Трансляция исключений JDBC в иерархию `DataAccessException`.<br>    - Единообразная обработка ошибок для разных типов хранилищ.<br>    - Повышение устойчивости приложения к сбоям.<br>        <br>10. **Поддержка аудита данных**<br>    <br>    - Автоматическое отслеживание авторов и времени изменений (`@CreatedBy`, `@LastModifiedDate`).<br>    - Ведение истории модификаций сущностей.<br>    - Соответствие требованиям регуляторов (например, для финтех‑проектов).<br>        <br>11. **Быстрая разработка прототипов**<br>    <br>    - Минимальные настройки для старта работы с БД.<br>    - Возможность итеративной доработки запросов.<br>    - Ускорение вывода MVP на рынок.<br>        <br>12. **Активное сообщество и документация**<br>    <br>    - Обширная официальная документация и гайды.<br>    - Большое количество примеров и решений в открытых источниках.<br>    - Поддержка со стороны VMware (разработчика Spring). |

### Какие основные интерфейсы предоставляет Spring Data для работы с базами данных?

| Под темы                      | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| ----------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Базовые интерфейсы            | 1. **`CrudRepository<T, ID>`**<br>    - **Назначение**: базовый интерфейс для CRUD‑операций.<br>    - **Основные методы**:<br>        - `$<S\ extends T>\ S\ save(S\ entity)$` — сохранение/обновление сущности;<br>        - `$Optional<T>\ findById(ID\ id)$` — поиск по ID;<br>        - `$boolean\ existsById(ID\ id)$` — проверка существования записи;<br>        - `$Iterable<T>\ findAll()$` — получение всех сущностей;<br>        - `$void\ deleteById(ID\ id)$` — удаление по ID;<br>        - `$void\ delete(T\ entity)$` — удаление переданной сущности.<br>    - **Когда использовать**: если нужны только базовые операции CRUD без пагинации и сортировки.<br>        <br>2. **`PagingAndSortingRepository<T, ID>`**<br>    - **Наследует**: `CrudRepository`.<br>    - **Добавляет**:<br>        - `$Iterable<T>\ findAll(Sort\ sort)$` — сортировка результатов;<br>        - `$Page<T>\ findAll(Pageable\ pageable)$` — пагинация результатов.<br>    - **Когда использовать**: когда требуется разбиение на страницы и сортировка данных.<br>        <br>3. **`JpaRepository<T, ID>`** (для JPA/Hibernate)<br>    - **Наследует**: `PagingAndSort grindingRepository`.<br>    - **Дополнительные методы**:<br>        - `$List<T>\ findAll(Sort\ sort)$`;<br>        - `$void\ flush()$` — принудительное сохранение изменений в БД;<br>        - `$<S\ extends T>\ S\ saveAndFlush(S\ entity)$` — сохранение и немедленный сброс контекста;<br>        - `$void\ deleteInBatch(Iterable<T>\ entities)$` — массовое удаление;<br>        - `$long\ count()$` — подсчёт записей.<br>    - **Когда использовать**: для работы с реляционными БД через JPA (Hibernate и др.). |
| Специализированные интерфейсы | 4. **`MongoRepository<T, ID>`**<br>    - Для работы с MongoDB.<br>    - Включает методы для геопространственных запросов и работы с коллекциями.<br>        <br>5. **`RedisRepository<T, ID>`**<br>    - Для Redis (key‑value хранилище).<br>    - Оптимизирован для операций кэширования и очередей.<br>        <br>6. **`CassandraRepository<T, ID>`**<br>    - Для Apache Cassandra.<br>    - Поддерживает CQL‑запросы и репликацию.<br>        <br>7. **`ElasticsearchRepository<T, ID>`**<br>    - Для Elasticsearch.<br>    - Добавляет методы для полнотекстового поиска и агрегаций.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| Вспомогательные интерфейсы    | 8. **`QuerydslPredicateExecutor<T>`** <br>    - Позволяет строить динамические запросы с помощью Querydsl.<br>    - Методы: `$Iterable<T>\ findAll(Predicate\ predicate)$` и др.<br>        <br>9. **`ReactiveCrudRepository<T, ID>`**<br>    - Реактивная альтернатива `CrudRepository` (для Spring WebFlux).<br>    - Возвращает `Mono<T>`/`Flux<T>` вместо `T`/ `Iterable<T>`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |

| Реализация | 1. Базовый интерфейс репозитория<br>Для поддержки пагинации и сортировки репозиторий должен наследовать:<br>java<br><br>java<br>public interface YourRepository extends PagingAndSortingRepository<YourEntity, Long> {<br>    // ваши методы<br>}<br>Или (если нужен JPA‑специфичный функционал):<br><br>java<br>java<br>public interface YourRepository extends JpaRepository<YourEntity, Long> {<br>    // ваши методы<br>}<br>Оба интерфейса включают методы для пагинации и сортировки.<br><br>2. Классы‑помощники<br><br>- **`Pageable`** — интерфейс, описывающий параметры страницы (номер, размер, сортировка).<br>- **`PageRequest`** — реализация `Pageable` для создания запроса страницы.<br>- **`Sort`** — класс для задания правил сортировки.<br>    <br><br>3. Реализация пагинации<br><br>**Шаг 1.** В контроллере принимайте параметры пагинации (обычно через HTTP‑запрос):<br><br>java<br><br>```java<br>@GetMapping("/items")<br>public Page<YourEntity> getItems(<br>        @RequestParam(defaultValue = "0") int page,<br>        @RequestParam(defaultValue = "10") int size) {<br><br><br>    Pageable pageable = PageRequest.of(page, size);<br>    return yourRepository.findAll(pageable);<br>}<br>```<br><br>**Что происходит:**<br><br>- `page` — номер страницы (начиная с 0).<br>    <br>- `size` — количество элементов на странице.<br>    <br>- `PageRequest.of(page, size)` создаёт объект `Pageable`.<br>    <br>- `findAll(pageable)` возвращает объект `Page<T>`.<br>    <br><br>**Шаг 2.** Объект `Page<T>` содержит:<br><br>- `getContent()` — список сущностей на текущей странице;<br>    <br>- `getTotalElements()` — общее количество записей в БД;<br>    <br>- `getTotalPages()` — общее число страниц;<br>    <br>- `hasNext()`/`hasPrevious()` — есть ли следующая/предыдущая страница.<br>    <br><br>4. Реализация сортировки<br><br>**Вариант 1.** Через параметры запроса (автоматическая сортировка):<br><br>java<br><br>```java<br>@GetMapping("/items")<br>public Page<YourEntity> getItems(<br>        @RequestParam(defaultValue = "0") int page,<br>        @RequestParam(defaultValue = "10") int size,<br>        @RequestParam(required = false) String sortField,<br>        @RequestParam(defaultValue = "asc") String sortDir) {<br><br>    Sort sort = Sort.by(sortDir.equals("asc") ? Sort.Direction.ASC : Sort.Direction.DESC, sortField);<br>    Pageable pageable = PageRequest.of(page, size, sort);<br>    return yourRepository.findAll(pageable);<br>}<br>```<br><br>**Вариант 2.** Жёстко заданная сортировка в репозитории:<br><br>java<br><br>```java<br>List<YourEntity> findAllByOrderByNameAsc();  // по имени по возрастанию<br>List<YourEntity> findAllByOrderByPriceDesc();  // по цене по убыванию<br>```<br><br>**Вариант 3.** Сортировка по нескольким полям:<br><br>java<br><br>```java<br>Sort sort = Sort.by("name").ascending()<br>                .and(Sort.by("price").descending());<br>Pageable pageable = PageRequest.of(0, 10, sort);<br>```<br><br>5. Комбинированная пагинация + сортировка<br><br>java<br><br>```java<br>@GetMapping("/items")<br>public Page<YourEntity> getItems(<br>        @RequestParam(defaultValue = "0") int page,<br>        @RequestParam(defaultValue = "10") int size,<br>        @RequestParam(defaultValue = "name") String sortField,<br>        @RequestParam(defaultValue = "asc") String sortDir) {<br><br>    Sort sort = Sort.by(Sort.Direction.fromString(sortDir), sortField);<br>    Pageable pageable = PageRequest.of(page, size, sort);<br>    return yourRepository.findAll(pageable);<br>}<br>```<br><br>**Параметры запроса в URL:**<br><br>```<br>/items?page=0&size=5&sortField=name&sortDir=desc<br>```<br><br>6. Дополнительные возможности<br><br>- **Возврат `Slice<T>` вместо `Page<T>`**  <br>    Если не нужно знать общее количество страниц (экономия на `COUNT`-запросе):<br>    <br>    java<br>    <br>    ```java<br>    Slice<YourEntity> findAll(Pageable pageable);<br>    ```<br>    <br>- **Пагинация для кастомных запросов**  <br>    Добавьте `Pageable` в метод репозитория:<br>    <br>    java<br>    <br>    ```java<br>    Page<YourEntity> findByNameContaining(String name, Pageable pageable);<br>    ```<br>    <br>- **Сортировка для кастомных запросов**  <br>    Передайте `Sort` как параметр:<br>    <br>    java<br>    <br>    ```java<br>    List<YourEntity> findByNameContaining(String name, Sort sort);<br>    ```<br>    <br><br>7. Важные нюансы<br><br>1. **Нумерация страниц** начинается с `0` (страница `0` — первая).<br>    <br>2. **По умолчанию** сортировка — по возрастанию (`asc`).<br>    <br>3. **Поля для сортировки** должны совпадать с именами полей сущности (или JPQL‑алиасами).<br>    <br>4. **Для производительности** индексируйте поля, по которым часто сортируете.<br>    <br>5. **Валидируйте** параметры `page` и `size` (например, через `@Min`, `@Max`). |

### Как использовать аннотацию @Query для создания пользовательских запросов в Spring Data?

| Под темы            | Заметка                                                                                                                                                                                               |
| ------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Ответ               | Аннотация `@Query` в Spring Data позволяет определять  **пользовательские запросы** (JPQL или native SQL)  прямо в методах репозитория, минуя ограничения  автоматических запросов по именам методов. |
| Типы запросов       | **JPQL (по умолчанию)**<br>- Объектно‑ориентированный язык запросов (работает с сущностями, а не таблицами).                                                                                          |
|                     | **Native SQL (через `nativeQuery = true`)**<br>- Прямой SQL для конкретной СУБД.                                                                                                                      |
| Передача параметров | **Позиционные параметры (`?N`)**<br>- Нумерация начинается с 1.                                                                                                                                       |
|                     | **Именованные параметры (`:имя` + `@Param`)**<br>- Более читаемо и безопасно при рефакторинге.                                                                                                        |

### Как настроить кэширование запросов в Spring Data?

| Под темы                       | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| ------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение                    | Кэширование в Spring Data реализуется через абстракцию  кэширования Spring Framework, которая  позволяет сохранять результаты выполнения методов в  кэш для ускорения последующих запросов. Это особенно  полезно для методов, которые часто вызываются с  одинаковыми параметрами и выполняют дорогостоящие  операции, например, запросы к базе данных.                                                                                                        |
| Активация кэширования          | Для начала необходимо активировать механизм  кэширования в приложении. Это делается с помощью аннотации `@EnableCaching`, которую добавляют к классу конфигурации или к основному классу приложения                                                                                                                                                                                                                                                             |
| Настройка CacheManager         | `CacheManager` — это объект, который управляет кэшем и  определяет, где и как будут храниться данные. Spring поддерживает различные реализации кэша, включая:<br><br>- `ConcurrentMapCacheManager` — простой кэш на основе  `ConcurrentHashMap`, подходит для тестирования и разработки;<br>- `RedisCacheManager` — кэш на основе Redis;<br>- `CaffeineCacheManager` — кэш на основе  библиотеки Caffeine; <br>- `EhCacheCacheManager` — кэш на основе EhCache. |
| Аннотации для управления кэшем | Эта аннотация используется для кэширования результатов метода. При первом вызове метода результат сохраняется в кэш, а при последующих вызовах с теми же параметрами данные берутся из кэша, а не выполняется сам метод.                                                                                                                                                                                                                                        |
|                                | `@CachePut`<br><br>Эта аннотация обновляет кэш, при этом метод  выполняется всегда, а его результат сохраняется в кэш.                                                                                                                                                                                                                                                                                                                                          |
|                                | `@CacheEvict`<br><br>Используется для удаления данных из кэша. Можно  удалить отдельный элемент или весь кэш.                                                                                                                                                                                                                                                                                                                                                   |
|                                | `@Caching`<br><br>Позволяет объединить несколько операций кэширования  для одного метода.                                                                                                                                                                                                                                                                                                                                                                       |

### Как использовать Spring Data REST для создания RESTful сервисов?

| Под темы                                  | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| ----------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Определение                               | Spring Data REST — это модуль, который позволяет автоматически генерировать RESTful API на основе Spring Data репозиториев без необходимости вручную писать контроллеры и обрабатывать HTTP-запросы. Он упрощает создание CRUD-операций (создание, чтение, обновление, удаление) и поддерживает принципы HATEOAS (Hypermedia as the Engine of Application State), что делает API более обнаруживаемым.                                                                                                                               |
| Настройка проекта                         | Для работы с Spring Data REST обычно используется Spring Boot, так как он предоставляет автоконфигурацию. Необходимо добавить зависимость spring-boot-starter-data-rest в проект.                                                                                                                                                                                                                                                                                                                                                    |
| Создание сущности                         | Определите JPA-сущность с помощью аннотаций JPA.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| Создание репозитория                      | Создайте интерфейс репозитория, который расширяет один из базовых интерфейсов Spring Data (например, CrudRepository или PagingAndSortingRepository). Чтобы сделать репозиторий доступным через REST, аннотируйте его @RepositoryRestResource.                                                                                                                                                                                                                                                                                        |
| Автоматически генерируемые REST-endpoints | После настройки репозитория Spring Data REST автоматически генерирует следующие endpoints:<br><br>\|HTTP-метод\|URL\|Действие\|<br>\|---\|---\|---\|<br>\|GET\|/users\|Получение списка всех пользователей\|<br>\|GET\|/users/{id}\|Получение пользователя по ID\|<br>\|POST\|/users\|Создание нового пользователя\|<br>\|PUT\|/users/{id}\|Обновление пользователя по ID\|<br>\|DELETE\|/users/{id}\|Удаление пользователя по ID\|<br>Ответы формируются в формате HAL JSON, который включает ссылки на связанные ресурсы (_links). |
| Настройка базового пути                   | По умолчанию Spring Data REST обслуживает ресурсы по корневому URI (/). Чтобы изменить базовый путь, можно использовать свойство spring.data.rest.basePath в application.properties или application.yml.                                                                                                                                                                                                                                                                                                                             |

### Какие стратегии оптимистичного и пессимистичного блокирования поддерживает Spring Data JPA, и как их применять?

| Под темы                    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| --------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Определение                 | Spring Data JPA поддерживает две основные стратегии управления конкурентным доступом к данным: **оптимистическую** и **пессимистическую блокировку**. Каждая из них решает проблему «потерянного обновления» (lost update), но использует разные подходы.                                                                                                                                                                                                                                                                                                                                                                                                    |
| Оптимистическая блокировка  | **Оптимистическая блокировка** предполагает, что конфликты при одновременном изменении данных редки. Она не блокирует данные при чтении или изменении, а вместо этого использует поле версии (`@Version`) для обнаружения конфликтов. Если при обновлении данные были изменены другой транзакцией, возникает исключение `OptimisticLockException`, которое приложение должно обработать (например, путём повторной попытки транзакции или информирования пользователя о конфликте).                                                                                                                                                                          |
| Пессимистическая блокировка | **Пессимистическая блокировка** предполагает, что конфликты вероятны, и блокирует данные, чтобы другие транзакции не могли получить к ним одновременный доступ. Блокировка удерживается на время транзакции и автоматически снимается при её фиксации или откате.                                                                                                                                                                                                                                                                                                                                                                                            |
|                             | **Типы пессимистической блокировки:**<br><br>1. `PESSIMISTIC_READ` (общая блокировка) — предотвращает изменение данных другими транзакциями, но позволяет их читать. Используется для предотвращения «грязного чтения».<br>2. `PESSIMISTIC_WRITE` (эксклюзивная блокировка) — предотвращает как чтение, так и изменение данных другими транзакциями.<br>3. `PESSIMISTIC_FORCE_INCREMENT` — сочетает пессимистическую `WRITE`-блокировку с оптимистической. Используется для предотвращения ситуаций, когда одна транзакция считывает данные, а другая блокирует их до завершения первой транзакции, что может привести к потере изменений первой транзакции. |

### Как настроить аудит сущностей в Spring Data JPA?

| Под темы                                         | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| ------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Включение аудита                                 | Добавьте `@EnableJpaAuditing` в класс конфигурации                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| Базовые аннотации для полей                      | В сущности добавьте поля с аннотациями:<br><br>- `@CreatedDate` — дата создания;<br>- `@LastModifiedDate` — дата последнего изменения;<br>- `@CreatedBy` — автор создания;<br>- `@LastModifiedBy` — автор последнего изменения.                                                                                                                                                                                                                                                                                                                                                                                                |
| Предоставление информации о текущем пользователе | Для `@CreatedBy`/`@LastModifiedBy` нужно реализовать `AuditorAware<T>`                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| Настройка через конфигурацию                     | Укажите bean `AuditorAware` в конфигурации                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| Наследование для повторного использования        | Чтобы не дублировать поля в каждой сущности, создайте базовый класс                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| Особенности и рекомендации                       | **Типы данных**:<br>    <br>    - Для дат: `LocalDateTime` (Java 8+), `Date`, `Instant`.  <br>    - Для авторов: `String` (логин), `Long` (ID пользователя) или объект `User`.<br>- **Производительность**: аудит добавляет небольшие накладные расходы на каждый `save()`/`update()`.<br>- **Миграции БД**: убедитесь, что в таблицах есть соответствующие колонки (`created_at`, `updated_at` и т. д.).<br>- **Тестирование**: проверяйте, что поля аудита заполняются корректно в разных сценариях.<br>- **Безопасность**: если используете Spring Security, убедитесь, что `SecurityContext` доступен в потоке выполнения. |
