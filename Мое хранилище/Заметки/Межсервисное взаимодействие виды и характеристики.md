| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| ----------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | Существует два вида межсервисного взаимодействия: синхронный - задачи выполняются друг за другом, и асинхронный - чередуются и выполняются частями.                                                                                                                                                                                                                                                                                                                                           |
|             | Синхронное взаимодействие - один сервис отправляет запрос другому и ждет, когда второй обработает данные. Минусы: клиент должен ожидать ответа на запрос, необходима стратегия обработки долгих запросов и ответов (время ограничено), возможен риск перегрузки сервера.                                                                                                                                                                                                                      |
|             | Асинхронное взаимодействие - использование посредника (брокер сообщения) между клиентами, ему отправляется сообщение и получатель забирает его при необходимости. Особенности: процессы не блокируются, сервер не перегружается избыточными запросами, на очередь могут подписаться несколько серверов, нет нужды в обработке ответов, для реализации асинхронного взаимодействия нужно разворачивать дополнительный сервис, при большом количестве сообщений допустимый объем израсходуется. |
