### Что такое загрузчик классов в Java?

| Под темы       | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| -------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение    | **Загрузчик классов (ClassLoader) в Java** — это компонент виртуальной машины Java (JVM), отвечающий за **динамическую загрузку классов** в память во время выполнения программы. С помощью ClassLoader можно загрузить класс, которого ещё не было на момент компиляции, например, при создании плагинов или модулей, добавляемых на лету.<br>**Функции Classloader**:<br>- находит и загружает файлы классов (обычно с расширением .class) из classpath в память;<br>- после загрузки проверяет байт-код класса для обеспечения его корректности и безопасности;<br>- связывает загруженные классы, включая проверку ссылок и разрешение символов;<br>- инициализирует загруженные классы, вызывая их статические инициализаторы и инициализаторы переменных. |
| Принцип работы | Процесс загрузки классов в JVM динамический: классы загружаются не заранее, а по требованию (ленивая загрузка). При запуске программы JVM загружает класс, содержащий метод main(), и все классы, на которые он ссылается. В дальнейшем при появлении нового типа подсистема загрузчика классов запрашивает его загрузку.<br>**Принцип делегирования** — когда загрузчик получает запрос на загрузку класса, он сначала пытается делегировать этот запрос своему родителю. Если родительскому загрузчику не удаётся загрузить класс, то он пытается загрузить его самостоятельно, отыскивая файл .class в определённых путях.                                                                                                                                   |

### Какие виды загрузчиков классов существуют в JVM?

| Под темы                | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| ----------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение             | В JVM (Java Virtual Machine) существуют **три основных вида загрузчиков классов**:<br>1. **Bootstrap ClassLoader** (также известен как Primordial ClassLoader и Null ClassLoader);<br>2. **Extension ClassLoader** (Platform ClassLoader);<br>3. **Application ClassLoader** (System ClassLoader).<br>Загрузчики классов построены в виде иерархии с отношениями «родитель-ребёнок». Каждый загрузчик, за исключением базового, является потомком абстрактного класса java.lang.ClassLoader.                                                                                                                                                                                                                                                               |
| Bootstrap (Примитивный) | **Базовый загрузчик классов**, встроенный в JVM. Загружает **основные классы Java** из файла rt.jar, такие как классы из java.lang, java.util и других.<br>**Особенности**:<br>- Не является классом Java, а представляет собой платформозависимый машинный код.<br>- В версиях Java до 8 загружал основные файлы Java с rt.jar, но начиная с Java 9 — из Java Runtime Image (JRT).<br>- Работает независимо, без каких-либо родительских загрузчиков классов.                                                                                                                                                                                                                                                                                             |
| Extension (Расширения)  | **Загружает классы из стандартных расширений Java**. По умолчанию находится в директории jre/lib/ext или другой директории, определённой системным параметром java.ext.dirs.<br>**Особенности**:<br>- Сам загружается с помощью Bootstrap ClassLoader.<br>- В старых версиях Java директория jre/lib/ext используется, а более новые версии могут использовать другой подход.<br>**Важно**: в Java 9 и более поздних версиях механизм расширений не рекомендуется — для обмена кодом между приложениями рекомендуется использовать модули.                                                                                                                                                                                                                 |
| System (Системный)      | **Загружает классы приложения** из директории classpath. Обычно это директория, указанная в переменной окружения CLASSPATH или параметре командной строки -cp.<br>**Особенности**:<br>- Используется для загрузки классов приложений и библиотек.<br>- Также загружает сторонние библиотеки, которые включены в classpath проекта.<br>**Важно**: по умолчанию в Java работает **делегирование вверх**: если AppClassLoader не находит класс, он передаёт запрос родителю (Platform → Bootstrap). Это защищает от подмены базовых классов.<br>**Важно**: можно создавать собственные загрузчики классов, расширяя класс java.lang.ClassLoader, для загрузки классов из нестандартных источников (например, из базы данных, сети или зашифрованных архивов). |

### Для чего нужен загрузчик классов в Java?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Загрузчик классов (ClassLoader) в Java нужен для динамической загрузки классов** в виртуальную машину Java (JVM). Это позволяет:<br>- **Загрузить классы, которых ещё не было на момент компиляции**. Это удобно для динамической загрузки плагинов или кода, полученного из сети.<br>- **Обеспечить эффективное использование ресурсов** — определения классов подгружаются в память только при реальной необходимости.<br>- **Предотвратить конфликты между классами** — разные загрузчики могут загружать классы с одинаковыми именами, что предотвращает конфликты в модулях приложений или при интеграции сторонних плагинов.<br>- **Вносить новое поведение в выполняющуюся программу** — загрузчики позволяют динамически загружать новые классы, что важно для создания архитектур плагинов и расширяемых программ на Java. |

### Как работает механизм делегирования загрузки классов в Java?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Определение | **Механизм делегирования загрузки классов в Java** заключается в том, что **загрузчик классов (ClassLoader) сначала делегирует поиск класса своему родительскому загрузчику**, прежде чем пытаться найти сам класс. Это позволяет: <br>- **Загружать классы тем загрузчиком, который максимально близко находится к базовому**.<br>- **Достигать максимальной области видимости классов** — каждый загрузчик ведёт учёт классов, которые были загружены именно им, помещая их в свой кэш.<br>- **Обеспечивать уникальность классов** — класс может быть загружен только однажды, и механизм делегирования позволяет убедиться, что загрузчик, инициирующий загрузку класса, не перегрузит загруженный ранее в JVM класс. |

### Можно ли создать собственный загрузчик классов в Java? Если да, то как?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| ----------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **В Java можно создать собственный загрузчик классов** — наследник класса java.lang.ClassLoader. Это позволяет контролировать процесс загрузки классов, загружать их из любого источника (например, из собственной системы каталогов, из базы данных) и использовать загрузчик вместо системного.<br>Пример применения — Java-апплеты, где классы автоматически загружаются с веб-сервера благодаря специальному загрузчику, реализованному «внутри» броузера.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| Реализация  | Для реализации собственного загрузчика классов необходимо переопределить методы класса ClassLoader. Например:<br>- **Метод findClass(String name)** — пытается найти файл класса по определённым путям и загрузить его данные (байтовый массив). Затем с помощью метода defineClass создаётся эквивалентный объект Class.<br>- **Метод loadClass(String name, boolean resolve)** — любая загрузка класса начинается с вызова этого метода, который использует все вышеперечисленные методы и реализует модель делегирования.<br>**Важно**:<br>- **Не можно создать новый класс, не располагая его байт-кодом** — загрузчик обязан получить корректный байт-код класса (образ в памяти обычного .class-файла) в виде массива byte.<br>- **Можно реализовать свою версию loadClass**, которая вначале пытается загрузить .class-файл самостоятельно и лишь в случае неудачи (например, для классов из пакета java.lang, которые обычно упакованы в JAR-архив) обращается к системному загрузчику. |

### Какие особенности безопасности следует учитывать при создании собственного загрузчика классов?

| Под темы     | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| ------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Архитектура  | - **Иерархия загрузчиков**. В одной виртуальной машине Java может быть несколько экземпляров объектов-загрузчиков. Важно, чтобы один и тот же класс не загружался одним и тем же загрузчиком более одного раза. Если класс не входит в число уже загруженных, текущий загрузчик пытается делегировать задачу загрузчику родительского класса (это может происходить рекурсивно). Это гарантирует, что используется соответствующий загрузчик.<br>- **Множественные пространства имён**. Каждый загрузчик имеет своё пространство имён для создаваемых классов. Если классы одинаковы и находятся в одном пакете, но загружаются разными загрузчиками, они считаются разными.<br>- **Изоляция пространств имён**. Загрузчик создаёт изолированные пространства имён для каждого класса-загрузчика, предотвращая конфликты при использовании нескольких версий одной библиотеки в JVM.                                                                                                                                                                                                                                                                                                        |
| Политика     | - **Координация с менеджером безопасности**. Загрузчик позволяет менеджеру безопасности находить информацию о классе, что позволяет применять правильную политику безопасности в зависимости от контекста запроса. Например, загрузчик знает, пришёл ли класс из сети (класс — часть апплета и не должен быть доверенным) или из локальной файловой системы (класс — часть браузера и должен быть доверенным).<br>- **Ограничения на создание экземпляров загрузчика**. Из-за возможностей загрузчиков классов система строго ограничивает, кто может создавать экземпляры. Например, веб-браузер может повторно использовать существующий загрузчик для загрузки классов апплетов с той же веб-страницы, но не другие типы загрузчиков.<br>- **Учёт контекста загрузки**. Ненадёжный код (включая конфигурацию) не должен иметь возможности добавлять к системным классам. Некоторые пакеты с префиксом имени могут быть ограничены с помощью свойства безопасности.                                                                                                                                                                                                                       |
| Кодирование  | - **Переопределять методы загрузчика**. Например, загрузчик может переопределить метод findClass(String name), чтобы указать, как должен искаться класс. Например, загрузчик апплета может переопределить этот метод, чтобы вернуться к хосту апплета и попытаться найти файл класса и загрузить его по сети.<br>- **Реализовывать правила загрузки классов**. Например, если нужно загружать классы из стандартного источника (из набора каталогов, который может соответствовать стандартной переменной CLASSPATH), нужно реализовать свою версию loadClass, которая вначале пытается загрузить .class-файл самостоятельно и лишь в случае неудачи обращается к системному загрузчику.<br>- **Учитывать «true-статические» классы**. Загрузчик должен опознавать библиотечные классы из пакетов типа java.lang, которые не нужно загружать самостоятельно. Например, можно проверить существование некоторого ключевого static-поля или не реализован ли в классе некоторый специальный пустой интерфейс.<br>- **Избегать сериализации чувствительных к безопасности классов**. В этом случае можно получить по сериализованному файлу интерфейс класса, что может привести к уязвимости. |
| Тестирование | - **Модульные тесты**. Например, можно написать тест, который проверяет, что загрузчик загружает класс так, как ожидалось. Например, если есть файлы parent.jar и child.jar, можно создать стандартный URLClassLoader для parent.jar и использовать пользовательский загрузчик для загрузки child.jar, а затем загрузить класс и проверить, что он на самом деле возвращает «дочерний» вместо «родительского».<br>- **Интегрированные тесты**. Например, можно написать тест, который проверяет, что загрузчик загружает классы из разных источников, и убедиться, что они загружаются так, как ожидалось.<br>- **Использование инструментов для проверки качества кода**. Например, можно использовать инструменты, которые непрерывно проверяют качество кода и ищут новые уязвимости.<br>**Важно**: возможность создавать и использовать загрузчик классов ограничена — например, Java-апплеты обычно не могут создавать свой загрузчик, так как используют загрузчик, предоставленный для них браузером.                                                                                                                                                                                |

### Опишите процесс загрузки класса в JVM и как он связан с моделью делегирования

| Под темы         | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| ---------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение      | **Загрузка классов в виртуальной машине Java (JVM) — это динамический процесс, при котором классы загружаются не заранее, а по требованию** (ленивая загрузка). Это осуществляется с помощью **загрузчика классов** — компонента JVM, который отвечает за поиск файлов классов и их загрузку в память.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| Процесс загрузки | Подсистема загрузчика классов работает в три фазы:<br>1. **Loading** — поиск и загрузка байткода. Загрузчик ищет .class-файл в файловой системе, JAR-архиве или по сети, считывает байткод и передаёт его JVM.<br>2. **Linking** — подготовка класса к использованию (verification, preparation, resolution). Верификация — проверка корректности импортируемого типа, подготовка — выделение памяти для статических переменных класса и инициализация памяти значениями по умолчанию, разрешение — преобразование символьных ссылок типов в прямые ссылки.<br>3. **Initialization** — выполнение статических инициализаторов и присвоение значений static-полям.<br>**Особенности процесса**:<br>- Классы загружаются по мере использования — при запуске программы JVM загружает класс, содержащий метод main(), и все классы, на которые он ссылается.<br>- Каждый загрузчик ведёт учёт классов, которые были загружены именно им, помещая их в свой кэш.<br>- Если два разных загрузчика загрузят один и тот же .class — это будут два разных класса для JVM. |

### Как загрузчик классов может влиять на пространство имен классов?

| Под темы    | Заметка                                                                                                                                                                                                                                                             |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Загрузчик классов в Java (ClassLoader) создаёт уникальное пространство имён** для каждого экземпляра загрузчика. Это означает, что в программе может существовать несколько классов с одним и тем же полным именем, если они обрабатывались разными загрузчиками. |

### В чем преимущества создания собственного загрузчика классов для загрузки классов из нестандартных источников?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| ----------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Создание собственного загрузчика классов (кастомного ClassLoader) в Java позволяет загружать классы из нестандартных источников** (например, из базы данных, сети или зашифрованных архивов). Это полезно, когда требуется загрузить обновляемые модули или плагины, которые могут меняться без необходимости перезапуска всей системы.<br>Также кастомный ClassLoader позволяет:<br>- **Контролировать процесс загрузки** всех Java-классов. Можно загружать их из любого источника, не отражённого в CLASSPATH.<br>- **Предотвращать конфликты** между классами — разные ClassLoaders могут загружать классы с одинаковыми именами, что полезно в модулях приложений или при интеграции сторонних плагинов.<br>- **Динамически генерировать классы** — загрузчик может сконструировать байт-код класса в памяти и после этого работать с ним, как с классом, загруженным из .class-файла. |
