### Какая команда используется для клонирования репозитория из удаленного источника?

| Под темы  | Заметка                                                                                                                                                                                                                                                                                |
| --------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| git clone | команда в Git, которая используется для клонирования репозитория из удалённого источника (например, с GitHub, GitLab, Bitbucket). Она создаёт локальную копию удалённого репозитория, загружая не только файлы, но и всю историю изменений, включая коммиты, ветки и удалённые ссылки. |

### Какая команда используется для просмотра истории коммитов?

| Под темы | Заметка                                                                                                                                                                                               |
| -------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| git log  | команда в системе Git, которая используется для просмотра истории коммитов в репозитории. Она предоставляет информацию о каждом коммите, включая SHA-1 хэш коммита, автора, дату и сообщение коммита. |

### Какая команда используется для создания новой ветки в Git?

| Под темы   | Заметка                                 |
| ---------- | --------------------------------------- |
| git branch | команда для создания новой ветки в Git. |

### Какая команда позволяет сливать изменения из одной ветки в другую?

| Под темы  | Заметка                                                                                                                 |
| --------- | ----------------------------------------------------------------------------------------------------------------------- |
| git merge | команда в системе контроля версий Git, которая позволяет сливать изменения из одной ветки в другую (выполнять слияние). |

### Как можно отменить изменения в файле, возвращая его к последнему состоянию в коммите?

| Под темы     | Заметка                                                                                                                                                                                                                                                                                                                                                              |
| ------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| git checkout | **Возвращает файл к состоянию последнего коммита**. Например, чтобы вернуть файл example.txt к состоянию последнего коммита, нужно выполнить команду: git checkout HEAD -- example.txt.                                                                                                                                                                              |
| git reset    | **Позволяет сбросить историю коммитов до определённого момента**. Существует три основных режима для этой команды: <br>1. **soft** — откатывает историю, но оставляет изменения в рабочей области.<br>2. **mixed** — откатывает историю и рабочую область, но оставляет файлы неизменёнными.<br>3. **hard** — полностью откатывает историю, рабочую область и файлы. |

### Какая команда позволяет изменить последний коммит без изменения его содержимого (например, для исправления сообщения коммита)?

| Под темы           | Заметка                                                                                  |
| ------------------ | ---------------------------------------------------------------------------------------- |
| git commit --amend | команда в Git, которая позволяет изменить последний коммит, будто его никогда и не было. |

### Какая команда позволяет перебазировать текущую ветку на указанную?

| Под темы   | Заметка                                                                     |
| ---------- | --------------------------------------------------------------------------- |
| git rebase | команда в Git, которая позволяет перебазировать текущую ветку на указанную. |

### Как можно временно сохранить изменения, не коммитя их, для последующего применения?

| Под темы | Заметка                                                                                                                                                                                                                                               |
| -------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Ответ    | **В Git изменения можно временно сохранить, не коммитя их, для последующего применения с помощью команды git stash**. Эта команда позволяет «сдать в архив» (или отложить) изменения, сделанные в рабочей копии, чтобы можно было применить их позже. |

### Какой командой можно просмотреть список всех веток, включая удаленные, и последние коммиты в них?

| Под темы | Заметка                                                                                                                                |
| -------- | -------------------------------------------------------------------------------------------------------------------------------------- |
| Ответ    | В системе Git для просмотра списка всех веток, включая удалённые, и последних коммитов в них используются команды git branch и git log |
