### Как работает метод put в HashMap?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| ----------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | Метод put в HashMap в Java используется для добавления или обновления пар «ключ — значение» в карте.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| Алгоритм    | Метод put принимает два аргумента:<br>1. **Ключ** — идентификатор, с которым будет связано значение.<br>2. **Значение** — значение, которое будет связано с ключом.<br>**Возвратное значение** зависит от того, существует ли ключ в карте:<br>- **Если ключ уже есть** — метод заменяет предыдущее значение, связанное с ключом, на новое, и возвращает предыдущее значение.<br>- **Если ключа ещё нет** — метод добавляет новую пару «ключ — значение» в карту и возвращает null.<br>**Важно**: если ключ уже есть в карте и значение, связанное с ним, — null, метод также возвращает null. |

### В чем разница между HashMap и TreeMap?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| ----------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **HashMap и TreeMap в Java — это реализации интерфейса Map, которые предоставляют аналогичный функционал по хранению пар «ключ-значение», но имеют ряд отличий**. Некоторые из них:<br>- **Упорядоченность элементов**. В HashMap порядок элементов не гарантируется, тогда как TreeMap автоматически упорядочивает элементы в соответствии с естественным порядком или с помощью компаратора.<br>- **Производительность**. Вставка, удаление и поиск элементов происходят быстрее в HashMap, чем в TreeMap, потому что HashMap использует хэш-таблицу для хранения элементов, в то время как TreeMap — красно-чёрное дерево.<br>- **Дополнительные методы**. TreeMap предоставляет дополнительные методы для работы с элементами в порядке их ключей, такие как firstKey(), lastKey() и subMap(). HashMap не имеет этих методов.<br>- **Использование памяти**. TreeMap имеет лучшую производительность в управлении памятью, так как не поддерживает массив для хранения пар «ключ-значение». В HashMap размер массива определяется при инициализации или изменении, что часто превышает необходимость. |
| HashMap     | **Лучше использовать**, если не требуется сохранять элементы в определённом порядке и когда требуется высокая скорость выполнения операций вставки, удаления и поиска элементов. Некоторые особенности HashMap:<br>- **Структура** — массив «корзин» (buckets), каждая из которых может содержать один или несколько элементов. Каждая корзина содержит связанный список элементов — узлов (Node), которые хранят ключ, значение и ссылку на следующий узел в списке.<br>- **Производительность** — в среднем случае операции put(), get() и remove() выполняются за константное время O(1). Однако производительность ухудшается при увеличении степени заполненности, особенно при высоком коэффициенте загрузки. При плотном заполнении коллекции увеличивается вероятность коллизий, что замедляет доступ к элементам.                                                                                                                                                                                                                                                                                |
| TreeMap     | **Лучше использовать**, когда необходимо сохранять элементы в отсортированном порядке или в порядке, заданном компаратором. Некоторые особенности TreeMap:<br>- **Структура** — красно-чёрное сбалансированное дерево. По-умолчанию TreeMap сортируется по ключам с использованием принципа натуральной сортировки, но это поведение может быть настроено под конкретную задачу при помощи объекта класса Comparator.<br>- **Производительность** — все операции — вставка, удаление и поиск — выполняются за логарифмическое время O(log(n)), где n — количество элементов в дереве.<br>- **Дополнительные методы** — TreeMap предоставляет методы для работы с элементами в порядке их ключей, такие как firstKey(), lastKey() и subMap().<br>- **Использование памяти** — TreeMap не поддерживает массив для хранения пар «ключ-значение», в то время как в HashMap размер массива определяется при инициализации или изменении, что часто превышает необходимость.                                                                                                                                    |

### Какой метод позволяет получить значение по ключу в HashMap?

| ----------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Метод get()** класса HashMap в Java позволяет получить значение по ключу.                                                                                                                                                                                                                                |

Если ключ отсутствует, может возвращаться специальное значение (например, null) либо бросаться исключение. |

### Как HashMap обрабатывает коллизии?

| Под темы | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| -------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Ответ    | HashMap обрабатывает коллизии (столкновения) с помощью **метода цепочек (external chaining)**. <br>При этом подходе в ячейке на самом деле содержится список (chain), а уже в списке может содержаться несколько значений (не обязательно с одинаковым хеш-кодом). Когда возникает коллизия, пара «ключ-значение» добавляется в связанный список по этому индексу. Чтобы получить значение, нужно перебрать связанный список, пока не будет найден желаемый ключ.<br>Также существует **метод открытой адресации (linear probing)**. При нём алгоритм ищет первую пустую ячейку после той, на которую указала хеш-функция.<br>HashMap умеет отличать коллизию от реальной перезаписи элемента. Когда структуре дают новую пару «ключ-значение», она проверяет, есть ли в массиве такие хэши и такие ключи. Результат проверки:<br>- если таких хэшей и ключей нет, в хэш-таблицу просто добавляется новая пара;<br>- если такой ключ есть, это перезапись — структура переписывает элемент с таким же ключом;<br>- если такого ключа нет, но хэш есть — это коллизия, новое значение записывается в ту же «корзинку» за предыдущим. |

### Как TreeMap сортирует элементы?

| Под темы                 | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| ------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение              | **Элементы в классе TreeMap в Java сортируются по естественному порядку ключей или с помощью заданного компаратора**. Это зависит от того, какой конструктор используется при создании карты.                                                                                                                                                                                                                                                                                                                                                                  |
| По естественному порядку | **По умолчанию** элементы в TreeMap сортируются в соответствии с естественным порядком ключей. Например: <br>- для целых чисел — по возрастанию;<br>- для строк — по алфавиту.<br>Естественный порядок определяется интерфейсом Comparable, который накладывает полный порядок на объекты каждого класса, его реализующего. Чтобы задать свой способ сравнения, нужно переопределить метод compareTo().<br>**Важно**: порядок, поддерживаемый в TreeMap, должен быть согласован с equals(), чтобы сортированная коллекция правильно реализовала интерфейс Map. |
| С помощью компаратора    | **Можно определить пользовательский компаратор** для сортировки ключей в соответствии с конкретными требованиями. Для этого компаратор передаётся в конструктор TreeMap.<br>**Comparator** — интерфейс в Java, который позволяет определить пользовательский порядок сортировки элементов в коллекции. Он предоставляет метод compare(), который сравнивает два объекта и возвращает целое значение, указывающее их относительный порядок.                                                                                                                     |
| Важно                    | **Важно**: все ключи, вставляемые в карту, должны быть взаимно сравнимы по заданному компаратору: comparator.compare(k1, k2) не должен вызывать ClassCastException для любых ключей k1 и k2 в карте.                                                                                                                                                                                                                                                                                                                                                           |
|                          | **Важно**: порядок, поддерживаемый в TreeMap, независимо от того, предоставлен ли явный компаратор, должен быть согласован с equals(), чтобы эта сортированная коллекция правильно реализовала интерфейс Map.                                                                                                                                                                                                                                                                                                                                                  |

### Какие интерфейсы реализуют HashMap и TreeMap?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **HashMap и TreeMap в Java реализуют интерфейс Map**, но при этом TreeMap также реализует дополнительные интерфейсы, которые расширяют функционал Map.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| HashMap     | **HashMap<K, V>** реализует **интерфейс Map<K, V>**. Он представляет структуру данных, где каждый элемент — пара «ключ-значение», при этом в рамках объекта Map все ключи уникальные.<br>Некоторые особенности реализации:<br>- Использует хэш-таблицу для хранения элементов.<br>- Обеспечивает быстрый поиск, добавление и удаление пар по ключу (в среднем O(1)), но не гарантирует порядка пар.<br>- Разрешает один null ключ и множество null значений.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| TreeMap     | **TreeMap<K, V>** реализует **интерфейс Map**, а также **интерфейсы SortedMap** и **NavigableMap**.<br>**SortedMap** расширяет Map и позволяет обеспечить полное упорядочение ключей. По умолчанию элементы упорядочиваются в соответствии с естественным порядком ключей (по возрастанию) или с помощью компаратора.<br>**NavigableMap** расширяет интерфейс SortedMap и обеспечивает возможность получения элементов коллекции относительно других элементов (условно говоря, навигацию).<br>Некоторые особенности реализации:<br>- Хранит пары, отсортированные по ключам (естественный порядок ключей или заданный Comparator).<br>- Использует красно-чёрное дерево.<br>- Операции медленнее (O(log n)), но позволяет получать пары в отсортированном по ключу виде.<br>- Не разрешает null ключи (если не задан специальный компаратор).<br>При этом порядок, поддерживаемый в TreeMap, должен быть согласован с equals(), чтобы эта сортированная коллекция правильно реализовала интерфейс Map. |

### Опишите процесс рехеширования в HashMap и как это влияет на производительность?

| Под темы                      | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| ----------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение                   | **Рехеширование (rehashing) в HashMap** — это процесс изменения размера хеш-таблицы и перераспределения её элементов при достижении определённого порога заполнения. Цель — обеспечить эффективную работу структуры данных, уменьшая вероятность коллизий.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| Принцип работы                | Рехеширование происходит, когда количество пар «ключ-значение» в HashMap превышает **пороговое значение**, определяемое коэффициентом загрузки (load factor). По умолчанию коэффициент загрузки равен 0,75, то есть изменение размера происходит, когда количество элементов превышает 75% от текущей ёмкости.<br>**Процесс рехеширования**:<br>1. **Расчёт новой ёмкости** — HashMap удваивает текущую ёмкость. Например, если текущая ёмкость равна 16, то новая ёмкость — 32.<br>2. **Создание нового массива** — HashMap создаёт новый массив с новой ёмкостью, все ячейки в котором изначально пусты.<br>3. **Перехэширование и перемещение элементов**:<br>    - Для каждой пары «ключ-значение» в старом массиве HashMap пересчитывает хэш-код ключа, используя новую ёмкость, и находит соответствующий сегмент в новом массиве.<br>    - Затем пара «ключ-значение» перемещается из старого блока в новый блок в массиве с изменённым размером.<br>Во время рехеширования хэш-коды ключей перефразируются на основе новой ёмкости, что может привести к другому распределению элементов по сравнению со старым массивом. |
| Влияние на производительность | - **Заранее задавать приблизительный размер хеш-таблицы** — если известно ожидаемое количество элементов, рекомендуется устанавливать начальную ёмкость как ожидаемое количество элементов / коэффициент загрузки. Это поможет избежать ненужных операций рехеширования.<br>- **Тщательно выбирать хеш-функцию** — особенно для пользовательских типов данных, чтобы минимизировать количество конфликтов. Плохая реализация hashCode() может привести к деградации производительности.<br>- **В современных версиях Java (начиная с Java 8)** — если записи в одном сегменте достигают порогового значения (TREEIFY_THRESHOLD, значение по умолчанию 8), то HashMap преобразует внутреннюю структуру сегмента из связанного списка в сбалансированное дерево (красно-чёрное дерево) во время повторного хэширования, обеспечивая более эффективный поиск (O(log n)) вместо линейного поиска (O(n)).                                                                                                                                                                                                                              |

### Каковы основные отличия между Red-Black деревом, используемым в TreeMap, и хеш-таблицей, используемой в HashMap?

| Под темы | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| TreeMap  | **Использует красно-чёрное дерево** для хранения пар «ключ-значение». Это самобалансирующееся бинарное дерево поиска с добавлением атрибута «цвет» для каждого узла, который может быть либо красным, либо чёрным.<br>**Особенности работы**:<br>- Ключи в TreeMap всегда отсортированы, что обеспечивает доступ к элементам в упорядоченном виде.<br>- Все операции — вставка, удаление и поиск — выполняются за логарифмическое время O(log(n)), где n — количество элементов в дереве.<br>- Красно-чёрное дерево обеспечивает быстрый поиск и удаление элементов.<br>**Внутреннее строение**: TreeMap состоит из узлов, каждый из которых содержит ключ, значение, ссылки на левого и правого потомков, а также цвет узла.                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| HashMap  | **Использует хеш-таблицу** для хранения элементов. Это ассоциативный массив, хранящий элементы в формате «ключ-значение», где каждому уникальному ключу соответствует определённое значение.<br>**Особенности работы**:<br>- Внутренняя реализация — это массив «корзин» (buckets), каждая из которых может содержать один или несколько элементов.<br>- Каждая корзина содержит связанный список элементов — узлов (Node), которые хранят ключ, значение и ссылку на следующий узел в списке.<br>- **Не гарантирует порядок элементов** — при итерации по HashMap порядок может быть произвольным и даже меняться после операций добавления или удаления.<br>- **Обеспечивает доступ к значению по ключу** за константное время O(1) в среднем случае.<br>**Начиная с Java 8**, когда количество элементов в одной корзине превышает определённый порог (обычно 8), связанный список трансформируется в красно-чёрное дерево. Это улучшает производительность в случаях высокой загруженности отдельных корзин.<br>**Вывод**: если нужно быстрое выполнение операций без учёта порядка ключей, следует использовать HashMap, если требуется хранение ключей в отсортированном порядке — TreeMap. |

### Какие стратегии можно использовать для оптимизации производительности TreeMap и HashMap в больших наборах данных?

| Под темы | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| -------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| TreeMap  | **TreeMap** — реализация интерфейса SortedMap, которая хранит пары «ключ-значение» в отсортированном порядке по ключам. В основе — красно-чёрное дерево, что обеспечивает порядок элементов. Некоторые стратегии оптимизации:<br>- **Поддержание баланса между деревьями**. Для достижения наилучших результатов красно-чёрное дерево следует регулярно балансировать.<br>- **Пакетная обработка**. Обрабатывая данные группами, а не по одному за раз, можно свести к минимуму нагрузку на память.<br>- **Выбор эффективных структур данных**. Например, можно уменьшить использование памяти, используя пользовательские объекты или примитивы внутри TreeMap.<br>- **Использование компаратора**. Это позволяет задать порядок хранения элементов, что улучшает производительность поиска и итерации.                                                                                                                                                                                                                                                                                                                                                                                                                    |
| HashMap  | **HashMap** обеспечивает быстрые операции вставки, удаления и поиска за счёт хеширования, но не гарантирует порядка элементов. Некоторые стратегии оптимизации:<br>- **Установка оптимальной начальной ёмкости**. Это может предотвратить дорогостоящие операции рехеширования.<br>- **Настройка коэффициента загрузки (load factor)**. Это влияет на компромисс между использованием памяти и вероятностью коллизий.<br>- **Правильная реализация методов hashCode() и equals()** для пользовательских классов ключей. Это критически важно для производительности HashMap.<br>- **Использование встроенной оптимизации** (с Java 8) — когда корзины (buckets) становятся слишком большими, они преобразуются в деревья, а не в связанные списки. Это улучшает производительность до O(log(n)) при доступе к элементам, значительно снижая время поиска и вставки. Для работы этой оптимизации ключи HashMap должны реализовывать интерфейс Comparable.<br>**Важно**: преждевременная оптимизация может привести к усложнению кода и снижению его читаемости. Рекомендуется начинать с написания понятного кода, а затем оптимизировать критичные для производительности участки на основе реальных данных профилирования. |
