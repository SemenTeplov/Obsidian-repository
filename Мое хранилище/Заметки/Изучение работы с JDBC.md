### Что такое JDBC?

| Под темы    | Заметка                                                                                                                                                |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Определение | JDBC (Java Database Connectivity) — платформенно независимый промышленный стандарт взаимодействия Java-приложений с реляционными базами данных (СУБД). |

### Как выполнить SQL-запрос в Java с использованием JDBC?

| Под темы | Заметка                                            |
| -------- | -------------------------------------------------- |
| Шаги     | Установить соединение с базой данных.              |
|          | Создать объект Statement.                          |
|          | Выполнить запрос с помощью метода executeQuery.    |
|          | Обработать результаты запроса в объекте ResultSet. |
|          | Закрыть соединение.                                |

### Какие преимущества использования JDBC перед другими методами доступа к базе данных?

| Под темы | Заметка                                                                                                                                                                                                                                                                                                                                                                                 |
| -------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Ответ    | **JDBC (Java Database Connectivity) имеет преимущества перед другими методами доступа к базе данных** благодаря универсальности, гибкости, высокой производительности и безопасности. Эти преимущества связаны с архитектурой JDBC, которая позволяет работать с базами данных через единый интерфейс, абстрагируя различия между различными системами управления базами данных (СУБД). |

### Какие основные интерфейсы используются в JDBC?

| Под темы   | Заметка                                                                                                                                                   |
| ---------- | --------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Connection | **Представляет соединение с базой данных**. Предоставляет методы для выполнения SQL-запросов, работы с транзакциями и получения метаданных о базе данных. |
| Statement  | **Предназначен для выполнения SQL-запросов**. Позволяет создавать объекты для выполнения запросов SQL на основе подключения к базе данных.                |
| ResultSet  | **Представляет результирующий набор базы данных**. Обеспечивает приложению построчный доступ к результатам запросов в базе данных.                        |

### Как управлять транзакциями в JDBC?

| Под темы | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                        |
| -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Ответ    | **В JDBC управление транзакциями (группой изменений в базе данных) осуществляется с помощью методов интерфейса java.sql.Connection**. По умолчанию соединение работает в режиме автоматической фиксации (autocommit = true) — каждый SQL-запрос обрабатывается как транзакция и автоматически фиксируется после выполнения. Если нужно выполнить несколько запросов в одной транзакции, необходимо отключить режим autocommit. |

### Как использовать PreparedStatement для защиты от SQL-инъекций?

| Под темы       | Заметка                                                                                                                                                                                                                                                                         |
| -------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение    | **PreparedStatement (параметризованный запрос) помогает защитить от SQL-инъекций** — метода атаки на базу данных, при котором злоумышленник вставляет вредоносный SQL-код через вводимые данные. PreparedStatement разделяет логику (SQL-код) и данные (пользовательский ввод). |
| Принцип работы | **Использование плейсхолдеров** (например, ?) для параметров, которые затем привязываются к конкретным значениям.                                                                                                                                                               |
|                | **Автоматическое экранирование специальных символов** в параметрах, чтобы вредоносный ввод не изменял структуру запроса.                                                                                                                                                        |
|                | **Прекомпиляция запроса** — сервер базы данных компилирует шаблон запроса с плейсхолдерами, чётко понимая его структуру.                                                                                                                                                        |
|                | **Передача данных для подстановки** — данные передаются отдельно, и сервер не интерпретирует их как часть SQL-команды.                                                                                                                                                          |

### Как реализовать пул соединений с базой данных в JDBC?

| Под темы       | Заметка                                                                                                                                                                                                                     |
| -------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Ответ          | **Реализовать пул соединений с базой данных (connection pool) в JDBC** можно с помощью **объектов типа DataSource**. Это позволяет повторно использовать существующие соединения вместо создания новых для каждого запроса. |
| Принцип работы | Когда новый пользователь запрашивает доступ к БД, ему выдаётся уже открытое соединение из пула.                                                                                                                             |
|                | Если все открытые соединения уже заняты, создаётся новое.                                                                                                                                                                   |
|                | Как только пользователь освобождает одно из уже существующих соединений, оно становится доступно для других пользователей.                                                                                                  |
|                | Если соединение долго не используется, оно закрывается.                                                                                                                                                                     |

### Какие шаблоны проектирования часто используются при работе с JDBC?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                         |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Factory     | **Шаблон Factory** позволяет создавать объекты, но позволяет подклассам выбирать тип создаваемых объектов. В JDBC этот шаблон реализуется, например, в классе **DriverManager**. Он предоставляет статический метод **getConnection()**, который служит фабрикой для получения объектов Connection. Метод абстрагирует сложность установления соединения, позволяя разработчикам не беспокоиться о деталях реализации драйвера. |
| Proxy       | **Шаблон Proxy** позволяет создавать посредника, который действует как интерфейс к другому ресурсу, но скрывает сложность компонента.                                                                                                                                                                                                                                                                                           |
| Transaction | **Шаблон Transaction** позволяет логически группировать разные действия в одно — транзакцию.                                                                                                                                                                                                                                                                                                                                    |
| Template    | **Шаблон Template** позволяет выполнять основной рабочий процесс JDBC, оставляя код приложения для предоставления SQL и извлечения результатов. В JDBC этот шаблон реализуется, например, в классе **JdbcTemplate**. Он выполняет SQL-запросы или обновления, инициирует итерацию по наборам результатов и перехватывает исключения JDBC, переводя их в общую иерархию исключений.                                              |

### Какие стратегии могут быть использованы для оптимизации производительности JDBC-приложений?

| Под темы             | Заметка                                                                                                                                                                                                                                                                                                                     |
| -------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Настройка соединения | **Использовать пул подключений**. Вместо создания нового подключения для каждого обращения к базе данных можно повторно использовать существующие подключения. Это снижает накладные расходы на создание и закрытие подключений.                                                                                            |
|                      | **Настроить параметры подключения**. Например, изменить таймауты подключения, максимальный размер пула подключений, размер выборки.                                                                                                                                                                                         |
|                      | **Использовать пакетную обработку (batch-запросы)**. Группировать несколько операций базы данных в один запрос, чтобы снизить количество обратных обращений между приложением и сервером базы данных.                                                                                                                       |
|                      | **Переиспользовать PreparedStatement**. Не создавать новый объект для каждого запроса — можно переиспользовать один и тот же объект для различных batch-пакетов.                                                                                                                                                            |
| Код                  | **Использовать параметризованные запросы (Prepared Statements)**. Они позволяют предварительно скомпилировать запросы и повторно использовать их с разными значениями параметров, что снижает необходимость в повторной компиляции и оптимизации.                                                                           |
|                      | **Ограничить количество возвращаемых данных**. Если необходимо получить только определённое количество записей из базы данных, использовать операторы LIMIT или TOP.                                                                                                                                                        |
|                      | **Использовать кэширование данных**. Это помогает избежать повторных запросов к базе данных.                                                                                                                                                                                                                                |
| База данных          | **Использовать подходящие индексы** для полей, по которым часто выполняются запросы. Индексы позволяют базе данных быстро находить нужные данные.                                                                                                                                                                           |
|                      | **Оптимизировать структуру таблиц** — разделить данные на отдельные таблицы, чтобы избежать дублирования.                                                                                                                                                                                                                   |
|                      | **Оптимизировать связи между таблицами** — использовать индексы и внешние ключи для улучшения производительности запросов, связанных с этими таблицами.                                                                                                                                                                     |
| Тестирование         | **Проводить нагрузочное тестирование** — проверять поведение приложения при различных нагрузках. Например, проводить тесты с пиковой нагрузкой (Spike Testing), длительным тестированием (Soak Testing) или стресс-тестированием (Stress Testing).                                                                          |
|                      | **Использовать профилирование** — динамический анализ программы, позволяющий замерить, сколько ресурсов потребляют разные части кода (время выполнения методов, объём памяти под объекты, частота вызовов функций и т. п.). Цель профилирования — помочь оптимизации программы, указав узкие места в коде или конфигурации. |
