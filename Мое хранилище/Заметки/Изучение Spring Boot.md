### Что такое Spring Boot и для чего он используется?

| Под темы               | Заметка                                                                                                                                                                                                                                                            |
| ---------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Определение            | Spring Boot — это фреймворк (инструмент) для создания  приложений на языке Java, входящий в экосистему Spring  Framework. Его главная цель — упростить и ускорить  разработку, развёртывание и поддержку Java‑приложений,  особенно веб‑сервисов и  микросервисов. |
| Особенности            | **Автоматическая конфигурация**  <br>Spring Boot «понимает» зависимости проекта и  автоматически настраивает компоненты Spring (например,  подключение к БД, веб‑сервер, безопасность),  минимизируя ручной код и XML‑конфигурации.                                |
|                        | **Встроенные серверы**    Включает готовые HTTP‑серверы (Tomcat, Jetty, Undertow).  Приложение можно запустить как исполняемый JAR‑файл  без отдельной установки сервера.                                                                                          |
|                        | **Стартеры (Starters)**  <br>Наборы предопределённых зависимостей для типовых  задач (например, `spring-boot-starter-web` для веб‑приложений, `spring-boot-starter-data-jpa` для работы с БД). Это упрощает управление  библиотеками.                              |
|                        | **Принцип «конвенция перед конфигурацией»**  <br>Использует разумные дефолтные настройки. Разработчику  нужно явно указывать только нестандартные параметры.                                                                                                       |
|                        | **Мониторинг и управление (Actuator)**  <br>Встроенные эндпоинты для проверки состояния  приложения (`/actuator/health`), метрик, логов и др.                                                                                                                      |
| Для чего  используется | **Веб‑приложения и REST API**  <br>Быстрое создание HTTP‑сервисов с обработкой запросов,  валидацией, сериализацией данных.                                                                                                                                        |
|                        | **Микросервисы**  <br>Лёгкое развёртывание независимых сервисов,  взаимодействующих через API.                                                                                                                                                                     |
|                        | **Работа с базами данных**  <br>Интеграция с SQL (JPA/Hibernate) и NoSQL‑СУБД через  готовые шаблоны.                                                                                                                                                              |
|                        | **Безопасность**  <br>Внедрение аутентификации и авторизации (например,  через Spring Security).                                                                                                                                                                   |
|                        | **Асинхронные задачи и планирование**  <br>Поддержка фоновых процессов и расписаний.                                                                                                                                                                               |
|                        | **Интеграция с облаками**   Совместимость с платформами вроде Kubernetes, AWS,  Google Cloud.                                                                                                                                                                      |

### Какие основные компоненты Spring Boot и как они взаимодействуют между собой?

| Под темы             | Заметка                                                                                                                                                                                                                                                                                                                                                                                        |
| -------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение          | Основные компоненты Spring Boot тесно интегрированы  между собой и работают как единый механизм, сокращая  рутину при разработке Java‑приложений. Разберём их  функции и взаимодействие.                                                                                                                                                                                                       |
| Ключевые  компоненты | **Spring Boot Starters** <br>**Что это**: наборы зависимостей («стартеры»),  упакованные по тематикам (web, JPA, security и др.).<br>**Примеры**: `spring-boot-starter-web`, `spring-boot-starter-data-jpa`, `spring-boot-starter-test`.<br>**Роль**: упрощают подключение библиотек — вместо десятка отдельных зависимостей добавляется  один стартер.                                        |
|                      | **Spring Boot AutoConfiguration**<br>**Что это**: механизм автоматической настройки  компонентов Spring на основе обнаруженных  зависимостей.<br>**Как работает**: анализирует classpath, находит стартеры  и активирует соответствующие конфигурации (например,  подключает Tomcat для `spring-boot-starter-web`).<br>**Роль**: избавляет от ручной настройки бо́льшей части  инфраструктуры. |
|                      | **Spring Boot Actuator**<br>**Что это**: модуль для мониторинга и управления приложением.<br>**Функции**:<br>HTTP‑эндпоинты (`/actuator/health`, `/actuator/metrics`);<br>сбор метрик, проверка работоспособности, логирование;<br>интеграция с системами мониторинга (Prometheus, Grafana).<br>**Роль**: даёт «обратную связь» о состоянии приложения в runtime.                              |
|                      | **Spring Boot CLI**<br>**Что это**: интерфейс командной строки для быстрого  создания и запуска приложений.<br>**Возможности**:<br>создание проектов через команды (`spring init`);<br>запуск Groovy‑скриптов без компиляции;<br>тестирование в консоли.<br>**Роль**: ускоряет прототипирование и эксперименты.                                                                                |
|                      | **Spring Initializr**<br>**Что это**: веб‑инструмент для генерации структуры проекта.<br>**Функции**:<br>выбор версий, зависимостей, языка (Java/Kotlin/Groovy);<br>генерация Maven/Gradle‑проекта с готовой конфигурацией.<br>**Роль**: стартовая точка для нового приложения.                                                                                                                |
|                      | **Embedded Servers**<br>**Что это**: встроенные HTTP‑серверы (Tomcat, Jetty, Undertow).<br>**Роль**: позволяют запускать приложение как  исполняемый JAR‑файл без внешней установки сервера.                                                                                                                                                                                                   |
|                      | **Spring Boot DevTools**<br>**Что это**: набор инструментов для разработки.<br>**Функции**:<br>автоматический перезапуск при изменении кода;<br>живая перезагрузка шаблонов (Thymeleaf, FreeMarker);<br>улучшенное логирование.<br>**Роль**: повышает продуктивность в процессе разработки.                                                                                                    |

### Что такое Spring Boot Starter и как он упрощает конфигурацию приложения?

| Под темы                             | Заметка                                                                                                                                                                                                                                                                                                                                                                                         |
| ------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение                          | **Spring Boot Starter** — это специализированный  модуль (набор зависимостей), упакованный по  тематическому принципу, который позволяет одним действием  подключить к проекту целый комплекс библиотек, конфигураций и  связанных ресурсов для решения определённой задачи.                                                                                                                    |
| Как устроен  стартер                 | По сути, стартер — это обычная зависимость для Maven  или Gradle, но с особой структурой и соглашениями:<br>- содержит список необходимых библиотек (прямых и  транзитивных);<br>- включает файлы автоматической настройки (например, `META‑INF/spring.factories`);<br>- задаёт ограничения версий для совместимости;<br>- может содержать исключения нежелательных  транзитивных зависимостей. |
| Как стартеры  упрощают  конфигурацию | **Сокращение ручного управления зависимостями**  <br>Вместо десятка отдельных библиотек указывается  один стартер. Система сборки (Maven/Gradle)  рекурсивно подтягивает всё необходимое.                                                                                                                                                                                                       |
|                                      | **Автоматическая настройка через AutoConfiguration**  <br>Стартер работает в связке с механизмом автоконфигурации Spring Boot:<br>- при обнаружении стартера в classpath Spring Boot  активирует соответствующие компоненты;<br>- например, `spring-boot-starter-data-jpa` автоматически настраивает:<br>- источник данных (DataSource);<br>- `EntityManagerFactory`;<br>- менеджер транзакций. |
|                                      | **Стандартизация настроек** <br>Стартеры задают разумные дефолтные значения для  параметров (порты, пути, таймауты), сокращая объём конфигурационных файлов.                                                                                                                                                                                                                                    |
|                                      | **Исключение конфликтов версий**  <br>В стартере уже согласованы версии всех входящих  библиотек, что предотвращает проблемы совместимости.                                                                                                                                                                                                                                                     |
|                                      | **Быстрая интеграция функционала**   Для добавления новой возможности (безопасность,  работа с БД, мониторинг) достаточно подключитьсоответствующий стартер:<br>- `spring-boot-starter-security` — базовая аутентификация/авторизация;<br>- `spring-boot-starter-actuator` — эндпоинты мониторинга;<br>- `spring-boot-starter-test` — инструменты для тестирования.                             |
|                                      | **Абстракция от низкоуровневых деталей**  Разработчик фокусируется на бизнес‑логике, а не на:<br>- выборе конкретных библиотек;<br>- управлении версиями;<br>- настройке интеграций между компонентами.                                                                                                                                                                                         |

### Как работает аннотация @SpringBootApplication?

| Под темы       | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| -------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Определение    | **Аннотация @SpringBootApplication в Spring Boot работает как точка входа в приложение**. Она объединяет несколько других аннотаций, чтобы упростить процесс конфигурации.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| Принцип работы | Аннотация @SpringBootApplication может принимать некоторые параметры, например:<br>- **exclude** — позволяет исключить конкретные классы автоконфигурации.<br>- **excludeName** — позволяет исключить имена классов автоконфигурации.<br>- **scanBasePackages** — указывает базовые пакеты для сканирования компонентов.<br>- **scanBasePackageClasses** — альтернатива параметру scanBasePackages для указания пакетов для сканирования компонентов.<br>**Важно**: @ComponentScan сканирует только пакет, где находится основной класс, и его подпакеты. Если компоненты или файлы конфигурации расположены вне этой иерархии, Spring Boot не обнаружит их. |

### Какие шаги необходимо выполнить, чтобы подключить базу данных к приложению на Spring Boot?

| Под темы      | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                      |
| ------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Нужно сделать | Добавьте зависимости в проект<br>В файл сборки (`pom.xml` для Maven или `build.gradle` для Gradle) включите:<br>- **Spring Data JPA** — для работы с ORM;<br>- **драйвер БД** — конкретный коннектор для вашей СУБД.                                                                                                                                                                                                         |
|               | Настройте подключение в `application.properties` или `application.yml`<br>Укажите URL, логин, пароль и другие параметры подключения. - `spring.datasource.url` — строка подключения (укажите хост, порт, имя БД);<br>- `spring.jpa.hibernate.ddl-auto` — стратегия генерации схемы:<br> - `create` — пересоздаёт БД при каждом запуске;<br> - `update` — обновляет схему, сохраняя данные;<br> - `none` — не изменяет схему. |
|               | Создайте модель (Entity)<br>Определите класс‑сущность, соответствующий таблице БД.                                                                                                                                                                                                                                                                                                                                           |
|               | Создайте репозиторий<br>Расширьте `JpaRepository` для работы с сущностью.                                                                                                                                                                                                                                                                                                                                                    |
|               | Используйте репозиторий в сервисе или контроллере                                                                                                                                                                                                                                                                                                                                                                            |
|               | Проверьте подключение<br>Запустите приложение. Spring Boot:<br>- автоматически подключится к БД;<br>- создаст таблицы (если `ddl-auto=create` или `update`);<br>- загрузит данные, если есть `data.sql` в `src/main/resources`.                                                                                                                                                                                              |

### Как использовать Spring Boot Actuator для мониторинга и управления вашим приложением?

| Под темы             | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| -------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение          | Spring Boot Actuator — это модуль, который добавляет  наблюдаемость в приложение, предоставляя метрики,  информацию о состоянии и возможности управления  через API или консоль. Он позволяет мониторить  работоспособность приложения, собирать метрики,  управлять логированием, а также выполнять другие  операции без перезапуска приложения.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| Подключение Actuator | Чтобы использовать Actuator в Spring Boot приложении, необходимо добавить соответствующую зависимость в проект. После добавления зависимости выполните `mvn install` или `gradle build`, чтобы подтянуть зависимости в проект.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| Встроенные эндпоинты | Actuator предоставляет набор готовых эндпоинтов для мониторинга и управления приложением. Некоторые из них:<br>- `/actuator/health` — показывает статус приложения (UP/DOWN). <br>- `/actuator/info` — предоставляет общую информацию о приложении (настраивается через `info` в `application.yml`).<br>- `/actuator/metrics` — метрики производительности приложения (например, использование памяти, CPU, количество запросов). <br>- `/actuator/env` — переменные окружения и их значения. <br>- `/actuator/loggers` — управление уровнями логирования. <br>- `/actuator/beans` — информация обо всех Spring Beans, зарегистрированных в приложении. <br>- `/actuator/shutdown` — позволяет корректно завершить работу приложения (по умолчанию отключён). <br>По умолчанию большинство эндпоинтов, кроме `/health` и `/info`, отключены. |

### Расскажите, как написать свой стартер?

| Под темы                  | Заметка                                                                                                                                                                                                                                                                                                                                                                     |
| ------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение               | Создание собственного Spring Boot Starter — способ  упаковать повторно используемую функциональность (зависимости + автоконфигурацию) для быстрого подключения к любым  Spring Boot‑приложениям.                                                                                                                                                                            |
| Общая структура  стартера | Обычно стартер состоит из двух модулей:<br>- **autoconfigure** — код автоконфигурации (классы `@Configuration`, `@Conditional*` и т. п.);<br>- **starter** — пустой JAR, содержащий зависимости (сам стартер + autoconfigure + сторонние библиотеки).                                                                                                                       |
| Пошаговая  реализация     | Создайте проект (Maven/Gradle)<br>Выберите имя по схеме:<br>- `ваш-проект-spring-boot-starter` (для объединённого модуля);<br>- `ваш-проект-spring-boot-autoconfigure` и `ваш-проект-spring-boot-starter` (для разделения).<br>**Важно:**<br>- не начинайте с `spring-boot` (это префикс официальных стартеров);<br>- используйте свой `groupId` (например, `com.example`). |
|                           | Добавьте обязательные зависимости                                                                                                                                                                                                                                                                                                                                           |
|                           | Создайте класс автоконфигурации                                                                                                                                                                                                                                                                                                                                             |
|                           | Зарегистрируйте автоконфигурацию                                                                                                                                                                                                                                                                                                                                            |
|                           | Создайте класс свойств (опционально)<br>Если стартер требует настройки, используйте `@ConfigurationProperties`                                                                                                                                                                                                                                                              |
|                           | Соберите стартер                                                                                                                                                                                                                                                                                                                                                            |
|                           | Используйте стартер в другом проекте<br>Добавьте зависимость в `pom.xml` целевого проекта                                                                                                                                                                                                                                                                                   |

### Как работает процесс автоматической конфигурации в Spring Boot и какие основные механизмы участвуют в этом процессе?
 
| Под темы                       | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| ------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение                    | Процесс автоматической конфигурации в Spring Boot —  это механизм, позволяющий **автоматически настраивать приложение на  основе зависимостей в classpath**, минимизируя ручное  конфигурирование.                                                                                                                                                                                                                                                                  |
| Ключевые  механизмы            | **`EnableAutoConfiguration`**<br>- Входит в состав `@SpringBootApplication`.<br>- Запускает сканирование classpath и активацию автоконфигураций.<br>- Работает на этапе запуска приложения.                                                                                                                                                                                                                                                                         |
|                                | **Условные аннотации (`@Conditional*`)**<br>    - Основа логики автоконфигурации.<br>    - Определяют, когда применять конфигурацию.<br>    - Основные виды:<br>        - `@ConditionalOnClass` — если класс есть в classpath;<br>        - `@ConditionalOnMissingBean` — если бин _не_ существует в контексте;<br>        - `@ConditionalOnProperty` — если свойство имеет заданное значение;<br>        - `@ConditionalOnBean` — если бин существует в контексте. |
|                                | **`META‑INF/spring.factories`**<br>- Файл в JAR‑библиотеках (например, `spring-boot-autoconfigure.jar`).<br>- Содержит список классов автоконфигурации под ключом  `org.springframework.boot.autoconfigure.EnableAutoConfiguration`.                                                                                                                                                                                                                                |
|                                | **`SpringFactoriesLoader`**<br>    - Внутренний механизм Spring для чтения `spring.factories`. <br>    - Загружает классы автоконфигурации из всех JAR в classpath.                                                                                                                                                                                                                                                                                                 |
|                                | **Порядок выполнения**<br>    - Spring Boot сканирует `spring.factories` во всех зависимостях.    <br>    - Для каждого класса автоконфигурации проверяет условия (`@Conditional*`).<br>    - Если условия выполнены — применяет конфигурацию (создаёт бины).                                                                                                                                                                                                       |
| Как это  работает на  практике | **Шаг 1. Добавление стартера**  <br>Вы подключаете, например, `spring-boot-starter-data-jpa`                                                                                                                                                                                                                                                                                                                                                                        |
|                                | **Шаг 2. Сканирование classpath**  <br>Spring Boot обнаруживает:<br>- классы Hibernate (`org.hibernate.SessionFactory`);<br>- интерфейс `JpaRepository`;<br>- драйвер БД (если есть в зависимостях).                                                                                                                                                                                                                                                                |
|                                | **Шаг 3. Активация автоконфигурации**<br>- `DataSourceAutoConfiguration` создаёт `DataSource` (на основе  `spring.datasource.*` из `application.properties`).<br>- `HibernateJpaAutoConfiguration` настраивает `EntityManagerFactory`.<br>- `JpaRepositoriesAutoConfiguration` сканирует репозитории.                                                                                                                                                               |
|                                | **Шаг 4.Создание бинов**  <br>Если все условия выполнены, Spring Boot:<br>- создаёт `DataSource`;<br>- настраивает транзакционный менеджер;<br>- регистрирует репозитории JPA.                                                                                                                                                                                                                                                                                      |

### Почему можно не указывать версии зависимостей в файле сборки при подключении большинства библиотек?

| Под темы                       | Заметка                                                                                                                                                                                                                                                                                                              |
| ------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Ответ                          | Причина в том, что Spring Boot  **управляет версиями зависимостей централизованно**  через механизм _parent‑проекта_ и _управляемые зависимости_ (dependency management).                                                                                                                                            |
| Ключевые  механизмы            | Родительский POM (spring-boot-starter-parent)<br>Когда вы указываете в `pom.xml`. Вы получаете:<br>- **готовые настройки сборки** (компилятор, кодировки, плагины);<br>- **управляемые версии** всех библиотек, совместимых с вашей версией Spring Boot;<br>- **согласованные свойства** (например, `java.version`). |
|                                | spring-boot-dependencies (управляемые зависимости)<br>Даже без `spring-boot-starter-parent` можно использовать управление версиями. Это даёт тот же эффект: все зависимости из экосистемы  Spring Boot автоматически получают корректные версии.                                                                     |
|                                | Стартеры (starters) как «упаковки»<br>Стартеры (`spring-boot-starter-*`) уже содержат:<br>- список необходимых библиотек;<br>- их согласованные версии;<br>- транзитивные зависимости.                                                                                                                               |
| Как это  работает на  практике | Вы указываете только артефакт. Версия **не нужна** — она берётся из `spring-boot-dependencies`.                                                                                                                                                                                                                      |
|                                | **Spring Boot подбирает совместимые версии**  <br>    Например, для Spring Boot 3.3.5:<br>    - Spring Framework — 6.1.13;<br>    - Tomcat — 10.1.30;<br>    - Jackson — 2.15.5.                                                                                                                                     |
|                                | **Транзитивные зависимости тоже управляются**  <br>Если стартер зависит от `jackson-databind`, его версия тоже будет согласована.                                                                                                                                                                                    |
|                                |                                                                                                                                                                                                                                                                                                                      |
