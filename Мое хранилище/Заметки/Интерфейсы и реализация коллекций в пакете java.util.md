### В чем разница между интерфейсами List и Set?

| Под темы | Заметка                                                                                                                                                                                                                                                  |
| -------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Различия | **Допущение дубликатов**. List допускает дубликаты, а Set — нет. Если вставить повторяющийся элемент в Set, существующее значение будет перезаписано.                                                                                                    |
|          | **Порядок**. List — упорядоченная коллекция, в которой поддерживается порядок вставки. Set — неупорядоченная коллекция, она не поддерживает никакого порядка. Существует несколько реализаций Set, которые поддерживают порядок, например LinkedHashSet. |
|          | **Доступ по индексу**. List поддерживает доступ по индексу, например, get(index). Set не поддерживает доступ на основе индекса.                                                                                                                          |
|          | **Нулевые элементы**. List допускает любое количество нулевых элементов, а Set может содержать не более одного нулевого элемента.                                                                                                                        |

Определяет основные методы, которые должны быть реализованы во всех коллекциях, например add(), remove() и contains().                                                                                                                                                                                                                                                                                                                                |

Элементы списка пронумерованы, начиная с нуля, и к конкретному элементу можно обратиться по целочисленному индексу.                                                                                                                                                                                                                                                                                                                                                                      |
|            | Интерфейс List — наследник интерфейса Collection, поэтому содержит все его методы и добавляет к ним несколько своих.                                                                                                                                                                                                                                                                                                                                                                                                         |
|            | Реализующие интерфейс List классы: ArrayList, LinkedList, Vector, Stack.                                                                                                                                                                                                                                                                                                                                                                                                                                                     |

Не допускает дублирующих значений.                                                                                                                                                                                                                                                                                                                                                                                                                                              |
|            | Интерфейс Set расширяет интерфейс Collection, поэтому любой класс, реализующий Set, реализует все методы, определённые в Collection.                                                                                                                                                                                                                                                                                                                                                                                         |

При этом в рамках объекта Map все ключи уникальные                                                                                                                                                                                                                                                                                                                                                                                  |
|            | Интерфейс Map не расширяет интерфейс Collection.                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
|            | Некоторые реализации интерфейса Map:<br>- **HashMap** — хранит пары «ключ-значение» с помощью хеширования для быстрого доступа, вставки и удаления.<br>- **LinkedHashMap** — похож на HashMap, но сохраняет порядок вставки пар «ключ-значение».<br>- **TreeMap** — хранит пары «ключ-значение» в отсортированном порядке по естественному порядку или указанному компаратору.<br>- **Hashtable** — синхронизированная реализация Map, которая не позволяет null-ключи или значения.                                         |

### Какие различия между ArrayList и LinkedList?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| ----------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **ArrayList и LinkedList — две различные реализации списка в языке программирования Java**, которые отличаются внутренней структурой данных и характеристиками производительности.                                                                                                                                                                                                                                                                                                                                                                                   |
| ArrayList   | **ArrayList** — **динамический массив**, который автоматически расширяется или уменьшается по мере добавления или удаления элементов.                                                                                                                                                                                                                                                                                                                                                                                                                                |
|             | - Элементы хранятся в последовательных ячейках памяти, к ним можно получить доступ по индексу.<br>- Это обеспечивает быстрый доступ к элементам по индексу, но может быть медленным при вставке или удалении элементов в середине списка.<br>- ArrayList требует меньше памяти, так как хранит только значения элементов и минимальную дополнительную информацию.<br>- Размер массива списка фиксирован и увеличивается автоматически при необходимости.                                                                                                             |
| LinkedList  | **LinkedList** реализован как **двусвязный список**, где каждый элемент содержит ссылки на предыдущий и следующий элементы.                                                                                                                                                                                                                                                                                                                                                                                                                                          |
|             | - Элементы хранятся в произвольных местах в памяти, и каждый элемент содержит указатель на следующий элемент в списке.<br>- Это обеспечивает эффективные операции вставки и удаления в середине списка, но требует дополнительной памяти для хранения указателей.<br>- В отличие от ArrayList, LinkedList не обеспечивает прямой доступ к элементам по индексу, и для доступа к элементам требуется проход по списку с начала или конца.<br>- Размер связанного списка может динамически изменяться, так как каждый элемент содержит указатель на следующий элемент. |

### Какие различия между Hashtable и Hashmap?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                     |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **HashMap и Hashtable** — две реализации интерфейса Map в Java, позволяющие хранить пары «ключ-значение». Обе структуры используют механизм хеширования для организации данных, что обеспечивает быстрый доступ к элементам со сложностью O(1) в среднем случае. Однако между ними существует ряд фундаментальных различий.                 |
| HashMap     | Хранит элементы в формате «ключ-значение», где каждому уникальному ключу соответствует определённое значение.                                                                                                                                                                                                                               |
|             | Внутренняя реализация — это массив «корзин» (buckets), каждая из которых может содержать один или несколько элементов.                                                                                                                                                                                                                      |
|             | Каждая корзина содержит связанный список элементов — узлов, которые хранят ключ, значение и ссылку на следующий узел в списке.                                                                                                                                                                                                              |
|             | С Java 8 при высокой загрузке бакетов связанный список трансформируется в красно-чёрное дерево, что улучшает производительность в случаях с большим количеством коллизий.                                                                                                                                                                   |
|             | **Синхронизация**: по умолчанию HashMap не синхронизирован, что позволяет одновременно обращаться к структуре нескольким потокам. Однако можно использовать методы Collections.synchronizedMap(map) или ConcurrentHashMap для создания потокобезопасной реализации HashMap.                                                                 |
|             | **Null-значения**: HashMap позволяет иметь один нулевой ключ и несколько нулевых значений.                                                                                                                                                                                                                                                  |
|             | **Итераторы**: итераторы HashMap — fail-fast, но не синхронизированы. Итераторы Hashtable также fail-fast и синхронизированы, что может привести к блокировкам при одновременной итерации в многопоточной среде.                                                                                                                            |
| Hashtable   | Обеспечивает синхронизацию на уровне всего объекта: каждый метод доступа или модификации (get(), put(), remove() и т. д.) синхронизирован целиком. В многопоточной среде это создаёт картину: только один поток может выполнять операцию с Hashtable в определённый момент времени, другие потоки блокируются и ждут освобождения монитора. |
|             | **Null-значения**: Hashtable не позволяет использовать null-ключи или null-значения в своей структуре. Вserting null может привести к NullPointerException.                                                                                                                                                                                 |
|             | **Размер**: в Hashtable размер фиксированный и установлен при создании объекта. Если количество элементов превышает размер Hashtable, происходит рехеширование (rehashing), что может замедлить выполнение программы. В HashMap же размер может меняться динамически при добавлении или удалении элементов.                                 |
|             | **Итераторы**: Hashtable не поддерживает fail-fast итераторы, которые позволяют обнаруживать изменения в структуре данных во время итерации, что может привести к ошибкам. HashMap же поддерживает fail-fast итераторы.                                                                                                                     |

### Как реализовать коллекцию, обеспечивающую быстрый доступ и поиск элементов?

| Под темы   | Заметка                                                                                                                                                                                                                                                                                                                                                      |
| ---------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| ArrayList  | **Обеспечивает быстрый доступ к элементам по индексу** (O(1)). Основан на динамическом массиве.                                                                                                                                                                                                                                                              |
|            | - Эффективная итерация по элементам.<br>- Минимальные накладные расходы на хранение.<br>- Однако при вставке или удалении элементов не в конце списка (особенно в начале) ArrayList требует сдвига всех последующих элементов, что приводит к сложности O(n).                                                                                                |
| LinkedList | **Обеспечивает быстрое добавление и удаление элементов** в начало, середину или конец списка (O(1), если есть ссылка на узел). Реализует двусвязный список, где каждый элемент хранит ссылки на предыдущий и следующий.                                                                                                                                      |
|            | - Быстрая вставка и удаление первого, последнего и элемента из середины списка.<br>- При этом доступ по индексу (get) медленнее, так как требует прохода по ссылкам.                                                                                                                                                                                         |
| TreeSet    | **Хранит элементы в отсортированном порядке** (естественном или заданном компаратором). Использует структуру красно-чёрного дерева.                                                                                                                                                                                                                          |
|            | - Операции имеют сложность O(log n).<br>- Поддерживает дополнительные операции с упорядоченными множествами (first, last, ceiling, floor).<br>- Элементы должны быть сравнимыми (Comparable или через Comparator).                                                                                                                                           |
| HashMap    | **Обеспечивает быстрый доступ к значению по ключу** за константное время O(1) в среднем случае. Внутренняя реализация — это массив «корзин» (buckets), каждая из которых может содержать один или несколько элементов. Каждая корзина содержит связанный список элементов — узлов (Node), которые хранят ключ, значение и ссылку на следующий узел в списке. |

### Что такое итератор и как он используется в коллекциях Java?

| Под темы    | Заметка                                                                                                                                                                                                            |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Определение | **Итератор (Iterator) в Java** — это объект, который позволяет последовательно перебирать элементы коллекции, не раскрывая её внутреннюю структуру. Он обеспечивает унифицированный способ обхода List, Set и Map. |
| Интерфейс   | Интерфейс Iterator предоставляет три ключевых метода для работы с элементами                                                                                                                                       |
|             | **E next()** — возвращает следующий элемент коллекции. Если элементов больше нет, метод выбрасывает исключение NoSuchElementException.                                                                             |
|             | **boolean hasNext()** — возвращает true, если в коллекции есть следующий элемент. Метод следует вызывать перед методом next(), так как при достижении конца коллекции метод next() выбрасывает исключение.         |
|             | **void remove()** — удаляет текущий элемент, который был получен последним вызовом next().                                                                                                                         |

Если добавить больше элементов, чем ёмкость массива, неявно вызывается метод grow() и пересоздаётся внутренний массив.<br>- **Удаление элементов** — можно реализовать методы для удаления элементов по индексу или по значению. При удалении по значению в цикле просматриваются все элементы списка, пока не будет найдено соответствие, — удалён будет лишь первый найденный элемент.<br>- **Поиск элементов** — можно реализовать метод для поиска элемента по индексу или по значению. |
| На основе связного списка (LinkedList) | **Добавление элементов** — можно использовать метод add(). Элементы хранятся в отсортированном порядке, дубликаты игнорируются, null-значения не допускаются.                                                                                                                                                                                                                                                                                                                                                                                                                                             |
|                                        | **Удаление элементов** — можно реализовать метод remove() для удаления указанного элемента из набора, если он присутствует.                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
|                                        | **Поиск элементов** — можно использовать методы для проверки, присутствует ли элемент в наборе (contains()).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |

### Как реализовать потокобезопасную коллекцию без использования стандартных синхронизированных коллекций?

| Под темы                               | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                  |
| -------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Определение                            | **В Java потокобезопасную коллекцию можно реализовать с помощью разных механизмов**, которые не требуют использования стандартных синхронизированных коллекций. Некоторые из них: атомарные операции (CAS), копирование при записи (Copy-on-Write), блокировки (ReentrantLock) или использование стримов (Stream API).                                                                                                   |
| Атомарные операции (CAS)               | **CAS (Compare-And-Swap)** — атомарная операция, которая используется для управления многопоточным доступом к данным без использования блокировок. Принцип работы: проверяется, соответствует ли текущее значение ожидаемому. Если да, происходит обновление значения. Если нет, операция повторяется.                                                                                                                   |
|                                        | Механизм CAS используется в классах из пакета java.util.concurrent.atomic, например, в потокобезопасных коллекциях (ConcurrentHashMap). С ним можно избежать использования явных блокировок и улучшить эффективность в средах с высоким параллелизмом.                                                                                                                                                                   |
| Копирование при записи (Copy-on-Write) | **CopyOnWriteArrayList** — потокобезопасный вариант ArrayList, который создаёт новую копию базовой структуры данных для каждой операции записи (добавление, установка, удаление). Эта копия затем становится доступной для чтения всеми потоками, в то время как исходный массив остаётся неизменным.                                                                                                                    |
|                                        | - Любые операции чтения всегда имеют согласованное представление данных, которые существовали во время последней операции записи, без необходимости использования блокировок для синхронизации.<br>- CopyOnWriteArrayList полезен только в определённых случаях, когда количество операций записи мало по сравнению с количеством операций чтения, иначе это может вызвать проблемы с памятью при большом объёме данных. |
| Блокировки (ReentrantLock)             | **ReentrantLock** — класс, который реализует то же поведение, что и обычные неявные блокировки. Блокировка осуществляется с помощью метода lock(), а освобождаются ресурсы — с помощью метода unlock().                                                                                                                                                                                                                  |
|                                        | **Особенности**:<br>- Если один поток вызвал lock(), и другой поток пытается получить доступ к методу до вызова unlock(), то второй поток будет простаивать до тех пор, пока метод не освободится.<br>- Только один поток может удерживать блокировку в каждый момент времени.                                                                                                                                           |
| Стримы (Stream API)                    | **Stream API** предоставляет функциональные методы для удобной и эффективной обработки коллекций, включая работу с потокобезопасными коллекциями. Например, можно использовать **параллельные потоки** (Parallel Stream), которые позволяют автоматически распараллеливать выполнение операций над элементами потока данных.                                                                                             |
|                                        | **Особенности**:<br>- Элементы разбиваются (если это возможно) на несколько групп и обрабатываются в каждом потоке отдельно.<br>- Затем на нужном этапе группы объединяются в одну для предоставления конечного результата.                                                                                                                                                                                              |

### Какие стратегии и подходы используются для оптимизации производительности при работе с большими коллекциями в Java?

| Под темы           | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| ------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение        | Для оптимизации производительности при работе с большими коллекциями в Java используются **стратегии, связанные с оптимизацией циклов, использованием Stream API, оптимизацией управления памятью и настройкой параметров виртуальной машины (JVM)**. Важно учитывать, что преждевременная оптимизация может привести к усложнению кода и снижению его читаемости. Рекомендуется начинать с написания понятного кода, а затем оптимизировать критичные для производительности участки на основе реальных данных профилирования.                                                                                                                                                                                                                                                                                                                                                                                                                            |
| Оптимизация циклов | **Вынесение инвариантных вычислений за пределы цикла**. Например, если в цикле вызывается метод size() коллекции, его можно вынести за пределы цикла, чтобы не вызывать метод с каждым телом цикла.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
|                    | **Использование расширенного цикла for** (цикла for-each) — он позволяет перебирать коллекции без ручного управления индексами, что упрощает код.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
|                    | **Избегание повторяющихся операций** в циклах, особенно с коллекциями. Например, не стоит проверять, содержит ли коллекция объект, на каждой итерации цикла — метод contains каждый раз сканирует весь список.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
|                    | **Использование StringBuilder** для эффективного объединения строк — это позволяет избежать создания ненужных объектов.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| Stream API         | Stream API (введён в Java 8) позволяет эффективно выполнять операции над коллекциями, такие как фильтрация, трансформация и агрегация.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
|                    | - **Замена сложных вложенных циклов** на простые цепочки операций с потоками.<br>- **Использование параллельных потоков** — они позволяют ускорить обработку больших объёмов данных, разделяя работу между несколькими ядрами процессора.<br>- **Снижение избыточных вычислений** — например, операции типа map и filter можно комбинировать, чтобы минимизировать количество проходов по данным.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| Управление памятью | Некоторые методы оптимизации управления памятью:<br>- **Объединение объектов** — повторное использование объектов вместо создания новых, когда это возможно. Например, поддержание «пула» объектов, который можно выделять и освобождать по мере необходимости.<br>- **Использование примитивных типов данных** вместо классов-обёрток — они имеют меньшие накладные расходы. Например, int вместо Integer при работе с целыми числами.<br>- **Оптимизация структур данных** — различные структуры данных имеют разные объёмы памяти и шаблоны доступа. Например, использование ArrayList вместо LinkedList, когда производительность произвольного доступа — приоритет.<br>- **Использование кэширования** — хранение результатов дорогостоящих вычислений или часто используемых ресурсов, что уменьшает необходимость в избыточной обработке.                                                                                                           |
| Настройка JVM      | Некоторые параметры JVM, которые можно настроить для повышения производительности:<br>- **Размер кучи** — слишком маленькая куча приведёт к частым сборкам мусора, очень большая — к длительным паузам GC. Рекомендуется настроить размер кучи, чтобы найти правильный баланс.<br>- **Соотношение поколений** — если в приложении много короткоживущих объектов, увеличение молодого поколения может повысить производительность. Например, параметр -XX:NewRatio=1 устанавливает соотношение Old:Young как 1:1, тогда как по умолчанию оно часто 2:1.<br>- **Настройка Thread Stack Size** — уменьшение размера стека с дефолтного 1MB до 256KB может снизить потребление памяти в приложениях с тысячами потоков.<br>- **Настройка компиляции switch-выражений** — параметр -XX:+OptimizeSwitchUsage заставляет JVM оптимизировать switch-операторы, преобразуя их в таблицы переходов или бинарный поиск в зависимости от характеристик входных данных. |
