### Что такое лямбда-выражение в Java? Какие преимущества лямбда-выражений вы можете назвать?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| ----------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Лямбда-выражение в Java** — это компактный способ описания анонимных функций, которые могут быть переданы как параметры или возвращены как результаты. Концепция появилась в версии Java 8 и стала важной частью функционального программирования в языке. <br>**Некоторые преимущества лямбда-выражений**:<br>- **Сокращение кода** — вместо создания анонимных классов с избыточным кодом можно использовать компактные лямбда-выражения.<br>- **Улучшение читаемости кода** — код становится более выразительным и проще для понимания, особенно при использовании функциональных интерфейсов и Stream API.<br>- **Поддержка функционального программирования** — лямбда-выражения способствуют использованию функционального стиля программирования, который поддерживает передачу функций как объектов и работу с неизменяемыми данными.<br>- **Упрощение работы с асинхронными задачами** — лямбда-выражения облегчают написание кода для асинхронного выполнения задач, таких как работа с потоками и обработка событий. |
| Суть        | Лямбда-выражение — это блок кода, который можно передать в другое место, как если бы это был объект. Некоторые особенности:<br>- **Отложенное выполнение** — лямбда-выражение не выполняется в момент объявления, а его выполнение откладывается до вызова метода, который использует эту лямбду.<br>- **Запоминание контекста** — если раньше в коде программист присвоил определённые значения переменным a и b, лямбда их помнит и использует для вычислений.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |

### Что такое функциональный интерфейс в Java?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| ----------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Функциональный интерфейс в Java** — это интерфейс, в котором определён **только один абстрактный метод** (метод без реализации). Также известен как интерфейс единого абстрактного метода (Single Abstract Method, SAM).<br>Функциональные интерфейсы появились в Java 8 и предназначены для использования в лямбда-выражениях или ссылках на методы.                                                                                                                                                                                                                |
| Структура   | Помимо одного абстрактного метода, функциональный интерфейс может содержать любое количество статических и дефолтных методов. Статические методы привязаны непосредственно к классу, и для вызова такого метода не нужен конкретный объект данного класса. <br>**Важно**: когда параметром метода является функциональный интерфейс, при вызове этого метода одним из аргументов должен быть блок кода. Передаваемый блок кода должен удовлетворять условию: его сигнатура должна совпадать с сигнатурой единственного абстрактного метода функционального интерфейса. |

### Может ли лямбда-выражение изменить значение локальной переменной?

| Под темы | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| -------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Ответ    | **В лямбда-выражениях Java прямые изменения локальных переменных невозможны**, так как они должны быть фактически неизменяемы.<br>Однако есть исключения:<br>- **Начиная с Java 11** можно использовать локальные переменные внутри лямбда-выражений, объявленные без ключевого слова final или var, если их значения не изменяются после присвоения. Это называется «неявно конечной переменной» (implicitly final variable) или «var-переменной, определённой с помощью ключевого слова val» (variably modified variable declared with the 'val' keyword).<br>- **Если локальная переменная объявлена как final** или не изменяемая по своей природе, то её можно использовать внутри лямбда-выражения. |

### Что такое ссылка на метод?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| ----------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Ссылка на метод (method reference)** — это сокращённая форма лямбда-выражения для вызова методов в языке Java. Это синтаксический сахар, который делает код короче и проще для чтения.<br>Ссылка на метод позволяет ссылаться на уже существующий метод и использовать его как функциональный интерфейс.<br>**Общий синтаксис ссылки на метод**: Name::methodName. Где Name может быть как объектом, так и классом, в зависимости от того, ссылку на что нужно получить и как это будет использовано потом.<br>**Некоторые способы передачи ссылки на метод**:<br>- **Ссылка на метод объекта**. Чтобы передать ссылку на метод объекта, нужно записать код вида объект::метод. Этот код эквивалентен коду x -> объект.метод(x). В качестве объекта могут фигурировать такие специальные переменные, как this и super.<br>- **Ссылка на метод класса**. Чтобы передать ссылку на статический метод, нужно записать код вида класс::метод. Этот код будет преобразован к коду вида x -> класс.метод(x).<br>- **Ссылка на конструктор**. Конструктор по своему поведению чем-то похож на статический метод класса, поэтому на него тоже можно передать ссылку. Выглядит это так: класс::new. |

В таких случаях может быть предпочтительнее использовать обычные методы или классы.<br>- **Не всегда подходят для всех задач** — лямбда-выражения не всегда подходят для всех сценариев. Например, в случаях, когда требуется сложное состояние или несколько методов в одном интерфейсе, анонимные классы или полные классы могут быть более подходящими. |

Это позволяет: <br>- **Сократить код** — вместо создания анонимных классов с избыточным кодом можно использовать компактные лямбда-выражения.<br>- **Передавать функциональность в метод** — лямбда-выражение может быть передано в метод как реализация единственного абстрактного метода функционального интерфейса.<br>- **Упростить работу с коллекциями** — лямбда-выражения можно использовать для фильтрации, преобразования или сортировки элементов коллекций, особенно в комбинации с Stream API.<br>Для обозначения функциональных интерфейсов в Java используется аннотация @FunctionalInterface, которая помогает компилятору и разработчикам понимать, что интерфейс предназначен для работы с лямбдами. |

### Что происходит при захвате переменной из внешнего контекста внутри лямбда-выражения?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| ----------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Лямбда-выражение в Java может обращаться к переменным, определённым вне его собственного блока**, — к переменным из окружающего метода или класса. Этот процесс называется **захватом переменных** (variable capture).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| Правила     | Лямбда-выражения могут захватывать три типа переменных из окружающей их области видимости:<br>1. **Переменные экземпляра**. Лямбда-выражение может напрямую обращаться к переменным экземпляра своего окружающего класса.<br>2. **Статические переменные**. Статические переменные принадлежат классу, а не какому-либо экземпляру, и лямбда-выражение может свободно обращаться к ним и изменять их.<br>3. **Локальные переменные**. Лямбда-выражение может фиксировать локальные переменные, объявленные во включающем методе, **но только если они фактически окончательные** — то есть не изменяются после назначения.<br>Это ограничение связано с тем, как лямбда-выражения реализованы в JVM — они могут быть выполнены в другом потоке или отложены во времени, что привело бы к проблемам с изменяемыми локальными переменными. |

### Какие стратегии и подходы используются для оптимизации производительности при работе с лямбда выражениями и потоками?

| Под темы            | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| ------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Лямбда выражения    | Некоторые стратегии оптимизации работы с лямбда-выражениями в Java:<br>- **Использование механизма invokedynamic**. JVM оптимизирует выполнение лямбда-выражений, создавая объект только один раз, если он не захватывает состояние из окружающего контекста. Если лямбда используется повторно в одном и том же контексте, JVM переиспользует уже созданный объект.<br>- **Отложенное создание**. Объекты лямбд создаются при первом вызове, а не во время компиляции. Это уменьшает время загрузки программы и экономит память, если лямбда не будет вызвана.<br>- **Инлайн-оптимизация**. JVM может встроить логику лямбды непосредственно в вызывающий код, уменьшая количество вызовов методов и повышая производительность программы.<br>- **Избегание побочных эффектов**. Лямбда-выражения не должны изменять внешнее состояние или иметь другие непреднамеренные последствия.                                                                                                                                                                                                                                                                                                                                                                                                |
| Потоки (Stream API) | Некоторые стратегии оптимизации работы с потоками (Stream API) в Java:<br>- **Фильтрация данных как можно раньше**. Это уменьшает объём данных для последующих операций.<br>- **Использование специализированных потоков для примитивов**. Это избегает накладных расходов на упаковку/распаковку.<br>- **Разумное применение параллельных потоков**. Они эффективны для больших коллекций и тяжёлых операций, но не всегда дают прирост производительности. Для запуска потоков в параллельном режиме можно использовать методы parallelStream() или parallel().<br>- **Минимизация использования stateful-операций**. Например, операции вроде sorted() требуют просмотра всех элементов и могут снизить эффективность параллелизма.<br>- **Выбор правильных коллекторов**. Некоторые коллекторы (например, Collectors.groupingByConcurrent()) оптимизированы для параллельного выполнения.<br>- **Использование эффективных структур данных**. Например, ArrayList и HashMap хорошо поддаются распараллеливанию, в отличие от LinkedList.<br>- **Рассмотрение возможности разбиения задач**. Иногда эффективнее разбить большую задачу на несколько меньших и обработать их параллельно.<br>Перед оптимизацией рекомендуется измерять производительность и определять узкие места. |

Передаваемый блок кода должен удовлетворять условию: его сигнатура должна совпадать с сигнатурой единственного абстрактного метода функционального интерфейса. |

### Какие есть способы обработки исключений в лямбда-выражениях и Stream API?

| Под темы            | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| ------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение         | **В лямбда-выражениях и Stream API в Java есть разные способы обработки исключений**. Это связано с тем, что функциональные интерфейсы Java 8 (Consumer, Function, Supplier и другие) не объявляют проверяемые исключения (checked exceptions) в своих методах.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| В лямбда-выражениях | Некоторые подходы к обработке исключений в лямбда-выражениях:<br>- **Обёртывание исключений**. Проверяемые исключения можно обернуть в непроверяемые. Это можно реализовать через вспомогательный статический метод.<br>- **Использование try-catch внутри лямбды**. Иногда этот подход является единственным решением. Однако он нарушает компактность лямбда-выражений.<br>- **Использование библиотеки Vavr**. Библиотека предоставляет функциональные интерфейсы, которые учитывают проверяемые исключения.<br>- **Создание собственных утилитных методов**. Можно создать специальные утилитные методы, которые управляют обработкой исключений для конкретных операций.<br>Также в лямбда-выражении можно генерировать исключения двух видов: заранее определённых типов Java (ArithmeticException, ArrayIndexOutOfBoundsException и т. д.) или собственноручно разработанных классов исключений. |
| В Stream API        | Некоторые способы обработки исключений в Stream API:<br>- **Выделение блока try-catch в отдельный метод**. Это позволяет обработать исключение и вернуть стандартное значение при его возникновении.<br>- **Использование FailableStream из Apache Commons Lang**. Это упрощённая версия Stream с сигнатурами методов, допускающими исключения.<br>- **Возврат экземпляра Optional** при возникновении исключения. Это предотвращает NullPointerException в дальнейшем в стеке вызовов.<br>**Важно**: Stream API не предоставляет декларативный способ обрабатывать исключения — в конвейере есть один канал для обработки данных, и нет отдельного канала для обработки исключений.                                                                                                                                                                                                                    |

### Объясните различия между методами andThen и compose в интерфейсе Function

| Под темы | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| andThen  | **Возвращает новую функцию, которая сначала применяет оригинальную функцию, а затем — функцию, переданную в качестве аргумента**.<br>**Особенности**:<br>- Метод принимает другую функцию в качестве параметра.<br>- Тип входного аргумента функции, которая применяется после оригинальной, должен соответствовать или быть супертипом типа возвращаемого значения текущей функции.<br>- Результат — новая функция, которая объединяет обе операции.<br>**Пример использования**: можно объединить функции, которые обрабатывают входные данные в определённом порядке, — сначала применить первую функцию к входу, а затем — вторую к результату первой функции. Это полезно, например, для создания цепочки функций, которые обрабатывают данные перед основной логикой (проверка, преобразование). |
| compose  | **Возвращает новую функцию, которая сначала применяет функцию, переданную в качестве аргумента, а затем — оригинальную функцию**.<br>**Особенности**:<br>- Метод принимает другую функцию в качестве аргумента.<br>- Тип входного аргумента составной функции — тип входного аргумента функции, которая применяется первой, а тип возвращаемого значения — тип возвращаемого значения текущей функции.<br>- Если оценка любой функции вызывает исключение, оно передаётся вызывающему объекту составной функции.                                                                                                                                                                                                                                                                                       |
