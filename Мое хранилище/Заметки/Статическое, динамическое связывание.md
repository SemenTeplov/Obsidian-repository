### Что такое статическое связывание в Java?

| Под темы                                      | Заметка                                                                                                                                                                                                                         |
| --------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение                                   | **Статическое связывание в Java** (также называется ранним связыванием) — это процесс, при котором **компилятор определяет, какой метод будет вызван, на основе типа переменной или ссылки на объект**, которая содержит метод. |
|                                               | Статическое связывание происходит **во время компиляции** кода.                                                                                                                                                                 |
| Принцип работы                                | Если тип переменной или ссылки заранее известен, то компилятор может точно определить, какой метод будет вызван, и связать его с этой переменной или ссылкой.                                                                   |
| Некоторые особенности статического связывания | **Для методов классов** (помечены модификатором static) — в контексте какого класса указано имя метода в исходном коде, на метод того класса в скомпилированном коде и ставится ссылка.                                         |
|                                               | **Для методов или полей с ключевым словом final** — они подлежат статическому связыванию, так как их реализация не может быть изменена.                                                                                         |
|                                               | **Для методов, объявленных с модификатором доступа private** — они также связываются статически, поскольку недоступны для переопределения в подклассах.                                                                         |
|                                               | **Для простых вызовов полей** — доступ к полям (переменным класса) также осуществляется через статическое связывание, например, если переменная является примитивным типом или объявлена как static.                            |

### Какие типы методов обычно используют статическое связывание в Java?

| Под темы       | Заметка                                                                                                                                                                                                                                                                       |
| -------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Типы методов   | **Статические методы** (объявлены с ключевым словом static). Связаны с классом, поэтому их связывание происходит на этапе компиляции.                                                                                                                                         |
|                | Методы или поля с ключевым словом final. Подлежат статическому связыванию, так как их реализация не может быть изменена.                                                                                                                                                      |
|                | Методы или поля внутри private классов. Связываются статически, поскольку недоступны для переопределения в подклассах.                                                                                                                                                        |
|                | **Перегрузка методов** (когда методы с одним именем имеют разные параметры (тип или количество)). Решение о том, какой метод вызвать, принимается на этапе компиляции.                                                                                                        |
| Принцип работы | Статическое связывание определяет, какой метод будет вызван, на основе **типа переменной или ссылки на объект**, которая содержит метод. Если тип заранее известен, компилятор может точно определить, какой метод будет вызван, и связать его с этой переменной или ссылкой. |

### Что такое динамическое связывание в Java?

| Под темы       | Заметка                                                                                                                                                                                                                                                                                                                |
| -------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение    | **Динамическое связывание (позднее связывание) в Java** — это процесс, при котором вызовы методов определяются **во время выполнения программы**, а не на этапе компиляции. Это позволяет реализовывать **полиморфизм**: один и тот же код работает по-разному в зависимости от того, объект какого типа его вызывает. |
| Принцип работы | Динамическое связывание обеспечивается **Java Virtual Machine (JVM) через таблицы виртуальных методов (vtable)**:                                                                                                                                                                                                      |
|                | - Когда создаётся объект, JVM строит таблицу, содержащую указатели на методы, которые могут быть вызваны для данного объекта.                                                                                                                                                                                          |
|                | - Если метод переопределён в подклассе, JVM заменяет соответствующую запись в таблице.                                                                                                                                                                                                                                 |
|                | - Когда вызывается метод через ссылку, JVM проверяет vtable для конкретного типа объекта. Если метод был переопределён, ссылка указывает на новый метод в vtable.                                                                                                                                                      |

### Приведите пример, когда используется динамическое связывание в Java.

| Под темы    | Заметка                                                                                                                                                                                                                                                       |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Динамическое связывание** (dynamic binding) в Java — это механизм, при котором определение метода происходит во время выполнения программы, а не на этапе компиляции. Это позволяет использовать один и тот же интерфейс для работы с разными реализациями. |

### Как работает механизм динамического связывания при наследовании?

| Под темы       | Заметка                                                                                                                                                                                                                                           |
| -------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение    | **Механизм динамического связывания (позднего связывания) работает при наследовании в объектно-ориентированном программировании**. Он позволяет на этапе выполнения программы выбирать подходящий метод, основываясь на фактическом типе объекта. |
| Принцип работы | Компилятор не знает тип объекта, но механизм вызова методов определяет его и вызывает соответствующее тело метода. Это происходит, когда:                                                                                                         |
|                | - **Метод базового класса переопределён в подклассе**. Виртуальная машина динамически находит и вызывает именно ту версию метода, которая определена в подклассе.                                                                                 |
|                | - **Есть переменная или ссылка на объект базового типа**, которая ссылается на объект производного класса. Вызов методов зависит от фактического типа объекта.                                                                                    |
|                | - **При работе с коллекциями** (особенно с элементами типа Object) динамическое связывание происходит при вызове методов объектов, которые содержатся в этих коллекциях.                                                                          |
|                | Программный код, основанный на вызове динамических методов, обладает свойством полиморфизма: один и тот же код работает по-разному в зависимости от того, объект какого типа его вызывает.                                                        |

### Чем отличается статическое связывание от динамического на примере методов классов?

| Под темы                | Заметка                                                                                                                                                                                                                                                                                                                            |
| ----------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Статическое связывание  | **Происходит во время компиляции кода**. Компилятор определяет, какой метод следует вызвать, исходя из типа переменной или ссылки на объект, которая содержит метод.                                                                                                                                                               |
| Пример                  | **Для методов классов, помеченных модификатором static**. В контексте какого класса указано имя метода в исходном коде, на метод того класса в скомпилированном коде и ставится ссылка. То есть осуществляется связывание имени метода в месте вызова с исполняемым кодом этого метода.                                            |
|                         | **Для перегруженных методов**. Компилятор на этапе компиляции определяет, какой из методов следует вызвать, исходя из переданных параметров.                                                                                                                                                                                       |
| Особенности             | - Используются не конкретные объекты, а информация о типе.                                                                                                                                                                                                                                                                         |
|                         | - Статическое связывание применяется ещё в одном случае — когда класс объявлен с модификатором final («финальный», «окончательный»).                                                                                                                                                                                               |
| Динамическое связывание | **Происходит во время выполнения программы**. Метод, который будет вызван, определяется только в момент выполнения, исходя из фактического типа объекта.                                                                                                                                                                           |
| Пример                  | - **Переопределение методов в классах-наследниках**. Когда метод базового класса переопределён в подклассе, JVM выбирает метод для вызова во время выполнения, основываясь на фактическом типе объекта.                                                                                                                            |
|                         | - **Вызов методов через типы-переменные родительского класса**. Когда есть переменная или ссылка на объект базового типа (например, через тип переменной Object или Animal), которая ссылается на объект производного класса, вызов методов будет зависеть от фактического типа объекта.                                           |
|                         | - **Работа с интерфейсами**. Когда класс реализует интерфейс, метод интерфейса может быть реализован по-разному в каждом классе. JVM динамически выбирает метод во время выполнения в зависимости от типа объекта.                                                                                                                 |
| Особенности             | - Если тип объекта не известен заранее, то компилятор не может точно определить, какой метод будет вызван, и связь происходит только во время выполнения программы.                                                                                                                                                                |
|                         | - Динамическое связывание в Java обеспечивается Java Virtual Machine через таблицы виртуальных методов (vtable) — когда создаётся объект, JVM строит таблицу, содержащую указатели на методы, которые могут быть вызваны для данного объекта. Если метод переопределён в подклассе, JVM заменяет соответствующую запись в таблице. |

### Как JVM определяет, какой метод вызывать при динамическом связывании?

| Под темы       | Заметка                                                                                                                                                                                                                     |
| -------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение    | **Java Virtual Machine (JVM) определяет, какой метод вызывать при динамическом связывании (поздней привязке) в Java, на основе фактического типа объекта**. Это происходит во время выполнения, а не во время компиляции.   |
| Принцип работы | Динамическое связывание используется для **виртуальных методов** — методов, которые могут быть переопределены в наследуемых классах. Выбор нужного метода происходит в зависимости от типа объекта, а не от ссылки на него. |
|                | Некоторые особенности динамического связывания:                                                                                                                                                                             |
|                | - **Заранее вычисляется таблица методов** для каждого класса (method table). В таблице перечисляются все сигнатуры методов (названия методов и типы их параметров) и сами вызываемые методы.                                |
|                | - **Если метод переопределён в подклассе**, JVM заменяет соответствующую запись в таблице.                                                                                                                                  |
|                | - **При фактическом вызове метода** виртуальная машина просто выполняет поиск в таблице.                                                                                                                                    |
|                | Динамическое связывание важно для реализации полиморфизма — одного из основных принципов объектно-ориентированного программирования (ООП).                                                                                  |

### Каковы особенности статического связывания при работе с перегруженными методами?

| Под темы       | Заметка                                                                                                                                                                                                                                                                                                                                                          |
| -------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение    | **Статическое связывание (раннее связывание) характерно для работы с перегруженными методами** — когда несколько методов имеют одно и то же имя, но принимают разные типы аргументов или разное количество аргументов. Это позволяет классу, структуре или интерфейсу объявлять несколько методов с одинаковыми именами и уникальными сигнатурами (параметрами). |
| Принцип работы | Компилятор **выбирает подходящий метод для вызова** на основе типов переданных аргументов. Некоторые особенности статического связывания:                                                                                                                                                                                                                        |
|                | **Использование не конкретных объектов, а информации о типе**. Компилятор проверяет тип ссылочной переменной, через которую вызывается метод, и проверяет, существует ли определение метода для неё в этом типе.                                                                                                                                                 |
|                | **Учёт приоритетов**. Выбирается метод с большим точным числом соответствий, при одинаковом количестве точных соответствий — рассматривается количество повышений точности, затем — число стандартных преобразований и т. д..                                                                                                                                    |
|                | **Невозможность создать перегруженный метод, который зависит только от типа возвращаемого значения**. Изменение типа возвращаемого значения не приводит к перегрузке.                                                                                                                                                                                            |

### Как влияет модификатор final на процесс динамического связывания методов?

| Под темы                   | Заметка                                                                                                                                                                                                                                                                                                                                                                                          |
| -------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Определение                | **Модификатор final в Java не влияет на процесс динамического связывания методов**. По умолчанию все нестатические методы в Java являются виртуальными, то есть могут быть переопределены в подклассах, и для них действует динамическое связывание. Исключение — методы, помеченные как final, которые **не могут быть переопределены** и, соответственно, не требуют динамического связывания. |
| Принцип работы             | Динамическое связывание происходит **во время выполнения** программы непосредственно во время вызова метода. На этапе написания метода заранее неизвестно, из какого класса будет проведён вызов. Это определяется типом объекта, для которого работает код: какому классу принадлежит объект, из того класса вызывается метод.                                                                  |
|                            | Динамическое связывание обеспечивается Java Virtual Machine через таблицы виртуальных методов (vtable). Когда создаётся объект, JVM строит таблицу, содержащую указатели на методы, которые могут быть вызваны для данного объекта. Если метод переопределён в подклассе, JVM заменяет соответствующую запись в таблице.                                                                         |
| Влияние модификатора final | Модификатор final **запрещает дальнейшее переопределение** метода. Это означает, что **метод не может быть переопределён** в подклассах. Это полезно, когда нужно предотвратить изменение исходной реализации.                                                                                                                                                                                   |
|                            | Например, если метод объявлен как final, **вызов метода может быть организован во время компиляции** — это называется статическим связыванием. Это быстрее, чем динамическое связывание методов, описанных без final.                                                                                                                                                                            |
