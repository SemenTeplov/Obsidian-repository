### Что такое первичный ключ в реляционной базе данных?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                     |
| ----------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Первичный ключ (primary key) в реляционной базе данных** — это атрибут (или набор атрибутов) таблицы, который **однозначно идентифицирует каждый кортеж (запись) в этой таблице**. Ключ гарантирует, что никакие две записи не могут иметь одинаковые значения для этого атрибута (или набора атрибутов). |
| Возможности | **Упростить поиск данных** — идентификатор позволяет быстро находить нужные записи.                                                                                                                                                                                                                         |
|             | **Ускорить выполнение пользовательских запросов** при больших объёмах данных — уникальные идентификаторы позволяют быстро искать информацию и формировать выборки по определённым критериям.                                                                                                                |
|             | **Обеспечивать согласованность данных** — с помощью первичных ключей можно выстраивать связи между таблицами.                                                                                                                                                                                               |
|             |                                                                                                                                                                                                                                                                                                             |

### Какое SQL-определение используется для установки ограничения NOT NULL на поле таблицы?

| Под темы    | Заметка                                                                                                                                                                                    |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Определение | **Ограничение NOT NULL** — определение в SQL, которое указывает, что **столбец не может принимать значения NULL**. Это полезно для обязательных полей, без которых запись не имеет смысла. |
|             | Ограничение NOT NULL предотвращает вставку или обновление NULL в указанный столбец.                                                                                                        |

### Как в SQL создать таблицу с автоматически инкрементируемым полем ID?

| Под темы   | Заметка                                                                                                                                   |
| ---------- | ----------------------------------------------------------------------------------------------------------------------------------------- |
| MySQL      | Ключевое слово  AUTO_INCREMENT определяет столбец как столбец с автоматическим инкрементом.                                               |
| SQL Server | Ключевое слово IDENTITY определяет столбец как столбец с автоматическим инкрементом.                                                      |
| PostgreSQL | PostgreSQL Типы данных SERIAL или BIGSERIAL автоматически создают числовую последовательность (SEQUENCE), которую будут инкрементировать. |

### Какое ограничение обеспечивает уникальность значений в столбце таблицы базы данных?

| Под темы    | Заметка                                                                                                                                                            |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Определение | **Ограничение UNIQUE** в SQL обеспечивает уникальность значений в столбце или группе столбцов. Это означает, что в столбце не может быть двух одинаковых значений. |
|             | Ограничение может применяться как к отдельным столбцам, так и к их комбинациям, формируя композитные ключи.                                                        |

### Как в SQL задать ограничение, чтобы значение поля таблицы соответствовало определенному диапазону чисел?

| Под темы    | Заметка                                                                                                                                                                                                                                                               |
| ----------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **В SQL для задания ограничения, чтобы значение поля таблицы соответствовало определённому диапазону чисел, используется оператор CHECK**. Он позволяет установить условие, которому должно удовлетворять значение, вводимое в таблицу, прежде чем оно будет принято. |
| SQL Server  | **После слова CHECK указывается в скобках условие**, которому должен соответствовать столбец.                                                                                                                                                                         |
| Oracle      | **Ограничение CHECK** позволяет задать логическое условие, которому должны соответствовать значения в столбцах таблицы.                                                                                                                                               |

### Как реализовать внешний ключ в SQL, указывающий на первичный ключ в другой таблице?

| Под темы    | Заметка                                                                                                                                                                            |
| ----------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Внешний ключ (FOREIGN KEY) в SQL реализуется как ссылка на первичный ключ в другой таблице**. Это позволяет установить связи между таблицами и обеспечить ссылочную целостность. |

### Как в SQL использовать ограничение CHECK для определения сложных условий на значения в столбце?

| Под темы    | Заметка                                                                                                                                            |
| ----------- | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | Ограничение CHECK состоит из ключевого слова **CHECK**, за которым следует логическое выражение в скобках.                                         |
| Особенности | Выражение должно включать столбец, для которого задаётся ограничение.                                                                              |
|             | Можно присвоить ограничению отдельное имя, чтобы улучшить сообщения об ошибках и позволить ссылаться на ограничение при изменении.                 |
|             | В одной таблице можно установить несколько ограничений CHECK для разных столбцов или для одного и того же столбца с применением различных условий. |

### Как реализовать составной первичный ключ в SQL?

| Под темы    | Заметка                                                                                                                                                                                                                                                                  |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Определение | **Составной первичный ключ (composite primary key) в SQL реализуется через конструкцию PRIMARY KEY (column1, column2, ...)**. Такой ключ необходим, когда уникальность строки определяется комбинацией нескольких полей, и одного числового идентификатора недостаточно. |
| Особенности | Все ключевые столбцы должны быть **NOT NULL**.                                                                                                                                                                                                                           |
|             | Отдельные столбцы, составляющие ключ, могут содержать повторяющиеся значения, но комбинация этих столбцов должна быть уникальной во всей таблице.                                                                                                                        |
|             | Порядок столбцов влияет на производительность базы данных: столбцы, которые часто используются в запросах, лучше расположить вначале.                                                                                                                                    |

### Как в SQL ограничить доступ к определенным строкам таблицы с использованием политик безопасности?

| Под темы      | Заметка                                                                                                                                                                                                                                                                           |
| ------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| MS SQL Server | **RLS** позволяет контролировать доступ к строкам в таблице в зависимости от характеристик пользователя, выполняющего запрос (например, идентификатора пользователя, роли).                                                                                                       |
| Определение   | **Определение функций-предикатов** — они возвращают логическое значение, указывающее, имеет ли текущий пользователь доступ к строке. Например, функция может проверять, соответствует ли отдел пользователя ID отдела, хранящемуся в строке таблицы.                              |
|               | **Применение политик безопасности** — можно указать функцию-предикат и определить, применяется ли она ко всем операциям или конкретным (SELECT, INSERT, UPDATE, DELETE).                                                                                                          |
|               | **Проверка конфигурации** — важно проверить, что каждый пользователь может только видеть строки, к которым он имеет доступ.                                                                                                                                                       |
| Oracle        | **RLS** реализован через **политики безопасности**, которые используют функции или процедуры для фильтрации строк в запросах. Политики могут быть назначены для конкретных таблиц или представлений.                                                                              |
| Определение   | **Создание политики** — вначале создаётся функция или процедура, которая будет выполнять проверку прав доступа, затем создаётся политика, которая использует эту функцию для фильтрации данных на уровне строк.                                                                   |
|               | **Применение политики** — политика автоматически применяется ко всем запросам на таблицу, например, возвращаются только те строки, где значение поля соответствует текущему пользователю.                                                                                         |
| PostgreSQL    | **RLS** позволяет определять **политики безопасности**, ограничивающие доступ пользователей к определённым строкам в зависимости от контекста выполнения запроса и атрибутов самих строк.                                                                                         |
| Определение   | **Автоматическая модификация запросов** — когда пользователь выполняет запрос к таблице с активированной политикой RLS, PostgreSQL автоматически добавляет условия фильтрации к запросу, сужая видимый набор данных только до тех строк, которые пользователь имеет право видеть. |
|               | **Создание политик** — можно указать политику для различных операций с данными (SELECT, INSERT, UPDATE, DELETE) и включить различные условия для каждой из них.                                                                                                                   |
|               | **Ограничения** — политики RLS не применяются к пользователям с привилегиями суперпользователя или к владельцам таблиц, если это специально не настроено.                                                                                                                         |
