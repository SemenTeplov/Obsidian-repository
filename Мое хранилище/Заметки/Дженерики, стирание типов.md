### Что такое дженерики в Java?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                   |
| ----------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Дженерики (generics) в Java** — это механизм для создания **универсальных классов, методов и интерфейсов**, которые работают с различными типами данных. Такие сущности создаются со специальным параметром, который позволяет указать, с каким типом данных они будут работать.                        |
|             | позволяют использовать один класс для разных типов данных                                                                                                                                                                                                                                                 |
|             | сохраняют типовую безопасность на этапе компиляции, что избавляет от явных приведений типов и предотвращает ошибки                                                                                                                                                                                        |
|             | позволяют создавать универсальные решения для работы с любыми видами данных, например, контейнеры или утилитные методы                                                                                                                                                                                    |
| Виды        | - **Дженерик-классы** (generic classes) — классы, «схемы» объектов с параметром. При создании объекта ему передаётся тип, с которым он будет работать.<br>- **Дженерик-методы** (generics methods) — методы, работающие по такому же принципу. Методу тип передаётся при вызове, сразу перед аргументами. |

### Для чего используются дженерики в Java?

| ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |

|             | **Избегать дублирования кода** — один обобщённый класс может работать с разными типами.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |

Это нужно, когда нужно работать с параметром метода, тип которого может отличаться от параметра типа класса. | ----------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

Это позволяет создавать классы, интерфейсы и методы, работающие с различными типами данных при сохранении строгой типизации. |

### Как работает механизм стирания типов в Java?

| Под темы       | Заметка                                                                                                                                                                                                                                                                                        |
| -------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение    | **Механизм стирания типов (type erasure) в Java** — это процесс, при котором компилятор удаляет информацию о параметризованных типах после проверки на этапе компиляции. Этот механизм был введён для обеспечения обратной совместимости с кодом, написанным до появления дженериков в Java 5. |
| Принцип работы | Компилятор заменяет все параметры типа их границами или типом Object, если границы не указаны.                                                                                                                                                                                                 |
|                | - **Для параметров типа без ограничений** (unbounded) — тип Object.<br>- **Для параметров типа с ограничениями** (bounded) — на эти границы.<br>- **Если для одного параметра типа установлено несколько ограничений**, то параметр заменяется на тип из первого ограничения.                  |
|                | Кроме стирания типов, компилятор может добавлять приведение (cast) к нужному типу и создавать переходные bridge-методы, чтобы сохранить полиморфизм в классах-наследниках.                                                                                                                     |

### Как ограничить типы, которые можно использовать в дженериках?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| ----------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **В дженериках (Generics) в Java можно ограничить типы, которые могут использоваться в качестве параметров**. Это позволяет создавать более специализированные классы и методы, сохраняя при этом гибкость обобщённого программирования.                                                                                                                                                                                                                                      |
|             | Для ограничения типов используются **ключевое слово extends** и **wildcard (подстановочные знаки)**.                                                                                                                                                                                                                                                                                                                                                                          |
| Синтаксис   | - **extends** — указывает, что параметр типа должен быть подклассом определённого класса или реализовывать определённый интерфейс.<br>- **wildcard** — позволяет работать с неизвестным типом. В Java используются два типа wildcard:<br>    - **? extends T** — верхний ограничитель (upper-bounded wildcard), где тип может быть наследником типа T или самим T.<br>    - **? super T** — нижний ограничитель (lower-bounded wildcard), где тип должен быть суперклассом T. |

### Какие проблемы решает механизм стирания типов и какие проблемы он создает?

| Под темы              | Заметка                                                                                                                                                                                                                                                                                                  |
| --------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Проблемы, решаемые    | **Корректно обрабатывать данные разного типа**. Переменная конкретного типа хранится в ячейке памяти общего типа, без меток типа, и обработка проводится в общем типе, если это можно.                                                                                                                   |
|                       | **Обеспечить обратную совместимость** с кодом, написанным до появления дженериков в языке. Компилятор удаляет информацию о параметризованных типах после проверки на этапе компиляции, чтобы не нарушать совместимость.                                                                                  |
|                       | **Упростить реализацию структур данных и алгоритмов** с помощью универсальных типов. Например, в Java дженерики (обобщённые типы) позволяют создавать классы, интерфейсы и методы, которые работают с любым типом данных, но при этом предоставляют типовую безопасность на этапе компиляции.            |
| Проблемы, создаваемые | **Ошибки времени выполнения**. После стирания типов не удаётся узнать конкретный тип, использующийся в дженериках, что может привести к ошибкам. Например, в Java нельзя создавать обобщённый массив напрямую, так как во время выполнения JVM не знает, какой конкретно тип представляет параметр типа. |
|                       | **Ограничения при работе с исключениями**. Дженерики не могут быть использованы с проверяемыми исключениями (checked exceptions), что ограничивает возможность создавать обобщённые классы, производящие проверяемые исключения.                                                                         |
|                       | **Сложности с созданием экземпляров типовых параметров**. Например, в Java недопустимо создавать экземпляры типовых параметров (new T()).                                                                                                                                                                |
|                       | **Разделение статических полей** между всеми экземплярами параметризованного класса независимо от их параметров типа.                                                                                                                                                                                    |

### Как реализовать дженерик метод, который принимает любой тип данных?

| Под темы    | Заметка                                                                                                                                                                                                                                     |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **В Java дженерик-метод, принимающий любой тип данных, реализуется с помощью параметра типа** (type parameter). Этот параметр заменяет конкретный тип данных, что позволяет создавать универсальные методы, работающие с различными типами. |
| Синтаксис   | Параметр типа указывается в объявлении метода **после всех модификаторов и перед типом возвращаемого значения**.                                                                                                                            |
|             | Параметр типа может быть ограничен (bounded). Например, можно указать, что метод принимает тип и все его подклассы (верхняя граница) или тип и все его суперклассы (нижняя граница). Для этого используется ключевое слово extends.         |
|             | В сигнатуре метода могут быть несколько параметров типа, разделённых запятыми.                                                                                                                                                              |
|             | При вызове метода перед его именем в угловых скобках указывается, какой тип будет передаваться на место универсального параметра.                                                                                                           |

### Каковы особенности использования дженериков при работе с исключениями и перегруженными методами?

| Под темы              | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                     |
| --------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Работа с исключениями | **Нельзя использовать тип, заданный в качестве параметра дженерика, в catch-выражениях**. Это ограничение связано с стиранием типов: во время выполнения информация о типе дженерика теряется, и компилятор не может определить, какой блок catch должен быть выполнен.                                                                                                                                                     |
|                       | **Можно сгенерировать исключение**, тип которого задаётся дженерик-параметром, **но экземпляр должен быть создан извне**. Это ограничение связано с ограничением дженериков: нельзя создать объект, используя оператор new, тип которого является параметром дженерика.                                                                                                                                                     |
|                       | **Дженерики инвариантны**: даже если A — подтип B, дженерик от A не является подтипом дженерика от B.                                                                                                                                                                                                                                                                                                                       |
|                       | **Нельзя определить недженерированный класс исключения внутри дженерик-класса** — это приводит к ошибке компиляции.                                                                                                                                                                                                                                                                                                         |
| Перегруженные методы  | **Определять, какую версию перегруженного метода использовать**, — задача уровня компиляции, а не рантайма. Компилятор подставляет наиболее подходящий метод, учитывая тип аргумента.                                                                                                                                                                                                                                       |
|                       | **Статические методы** не имеют доступа к параметрам типа классов-дженериков. Если такие методы должны использовать параметризацию, то это должно происходить на уровне метода, а не на уровне класса.                                                                                                                                                                                                                      |
|                       | **Трудности возникают при реализации или перегрузке методов**, содержащих в своей сигнатуре параметры, тип которых является параметром дженерика. Например, если в классе есть метод, содержащий параметр типа, во время компиляции создаётся специальный метод, реализующий метод из интерфейса, который принимает параметр типа. Несмотря на то, что такой код компилироваться не будет, в байт-коде он вполне корректен. |
