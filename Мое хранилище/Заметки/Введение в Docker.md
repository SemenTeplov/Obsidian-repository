### Что такое Docker и для чего он используется?

| Под темы       | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |     |
| -------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --- |
| Определение    | **Docker** — платформа для **контейнеризации приложений**. Она позволяет создавать, распространять и запускать приложения в контейнерах — изолированных средах, которые содержат всё необходимое для запуска приложения: код, библиотеки, зависимости, операционную систему.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |     |
| Принцип работы | Docker использует **виртуализацию на уровне операционной системы**. Виртуальная среда запускается прямо из ядра основной операционной системы и использует её ресурсы. Некоторые особенности работы:<br>- **Изоляция приложений** — каждый контейнер работает в своём собственном окружении, отделённом от других приложений и процессов на хост-системе. Это обеспечивает более безопасное и надёжное выполнение приложений.<br>- **Портативность** — контейнеры легко переносимы и могут быть запущены на любой системе, поддерживающей Docker.<br>- **Масштабируемость** — Docker позволяет масштабировать приложения горизонтально путём запуска нескольких контейнеров одновременно, что упрощает процесс обработки высоких нагрузок.<br>- **Управление версиями** — контейнеры позволяют управлять версиями приложений и их зависимостей, что упрощает процесс развёртывания и обновления.                                                                                                                      |     |
| Компоненты     | Некоторые компоненты платформы Docker:<br>- **Docker host** — компьютер, на котором работает Docker (может быть персональный компьютер, железный сервер или виртуальная машина в облаке).<br>- **Docker daemon** — фоновый процесс, который работает постоянно и ожидает команды. Все операции по управлению контейнерами отправляются именно в демон, например: запустить или остановить контейнер, скачать образ.<br>- **Docker client** — клиент, при помощи которого пользователи взаимодействуют с демоном и отправляют ему команды. Это может быть консоль, API или графический интерфейс.<br>- **Docker image** — неизменяемый образ, из которого разворачивается контейнер.<br>- **Docker container** — уже развёрнутое и запущенное приложение.<br>- **Docker registry** — репозиторий, в котором хранятся образы. Есть публичные репозитории, например Docker Hub, а можно создать свой репозиторий для использования внутри компании или команды.<br>- **Dockerfile** — файл-инструкция для сборки образа. |     |
| Инструменты    | Для работы с Docker используются, например:<br>- **Docker CLI** — интерфейс командной строки, через который передаются все команды: запустить контейнер, остановить его, посмотреть логи или собрать новый образ.<br>- **Docker Desktop** — официальное приложение с графическим интерфейсом для Windows, Linux и macOS.<br>- **Docker Compose** — инструмент для управления несколькими контейнерами, позволяет создавать контейнеры и задавать их конфигурацию.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |     |

### Каковы основные преимущества использования Docker?

| Под темы         | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| ---------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Портативность    | Контейнеры Docker легко переносимы и могут быть запущены на любой системе, поддерживающей Docker. Это позволяет:<br>- **Разрабатывать и тестировать приложения** на одной системе, а затем развернуть их на другой системе без проблем.<br>- **Запускать приложения** в различных окружениях: на локальном компьютере, в дата-центре или облаке. Поведение приложения при этом остаётся неизменным.<br>- **Упрощать перенос приложений** между средами, например, между производственным облаком и тестовыми облаками поставщиков облачных услуг.                                                                                                                                                                                                                                                                                                              |
| Скорость         | Запуск контейнера занимает секунды, поскольку не требуется загрузка ОС. Это ускоряет процессы тестирования и выпуска новых версий. Некоторые особенности скорости развёртывания:<br>- **Упрощение переноса приложений** — не нужно повторно настраивать зависимости и конфигурации на каждом новом сервере.<br>- **Возможность быстрого тестирования** — если выходит новое ПО или сервис, его можно запустить в контейнере и сразу начать тестировать.<br>- **Упрощение развёртывания зависимостей** — вместо ручной установки баз данных, брокеров сообщений, кешей и других компонентов можно запускать нужные сервисы одной командой.                                                                                                                                                                                                                      |
| Масштабируемость | Приложения, упакованные в контейнеры, можно легко масштабировать горизонтально — запускать дополнительные экземпляры в случае увеличения нагрузки. Это важно в микросервисной архитектуре. Некоторые особенности масштабируемости:<br>- **При росте нагрузки** можно быстро запустить дополнительные экземпляры контейнера и распределить между ними трафик. Когда нагрузка спадёт, лишние контейнеры легко остановить.<br>- **Docker подходит для реализации архитектуры на основе микросервисов** — каждый микросервис упаковывается в отдельный контейнер, который можно настроить и протестировать, запустить или остановить отдельно от других.<br>- **Инструменты для управления контейнерами** (например, Docker Swarm) могут равномерно распределять рабочую нагрузку, что позволяет приложениям быстро масштабироваться при резких изменениях спроса. |
| Безопасность     | **Изоляция приложений** обеспечивает безопасность и стабильность. Каждый контейнер работает в своём пространстве имён, не имея доступа к процессам других контейнеров. Если одно приложение даст сбой, остальные продолжат работать.<br>Также Docker позволяет:<br>- **Управлять версиями** приложений и их зависимостей, что упрощает процесс развёртывания и обновления.<br>- **Контролировать ресурсы** — неизолированные процессы могут конкурировать за память и вычислительные мощности процессора, а изолированные друг от друга программы не делают этого.<br>Однако важно учитывать, что контейнеры задействуют ресурсы ОС, и поэтому потенциально уязвимы. Также образы могут быть получены из разных источников, поэтому запускать следует только проверенные.                                                                                      |

### Чем контейнер Docker отличается от виртуальной машины?

| Под темы                | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| ----------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Виртуальная машина (VM) | **VM** — это полная эмуляция физической машины, которая включает в себя своё собственное ядро операционной системы и имитирует оборудование (процессор, память, диск и т. д.). Некоторые особенности VM:<br>- Каждая виртуальная машина работает на своей собственной операционной системе с отдельным ядром.<br>- Между виртуальной машиной и физическим оборудованием должна быть прослойка — специализированная программа (гипервизор). Она копирует (эмулирует) процессорное время, память, жёсткий диск, сеть и другие аппаратные ресурсы физического устройства, позволяя виртуальным машинам совместно ими пользоваться.<br>- На одном компьютере может функционировать несколько виртуальных машин (например, для имитации нескольких серверов на одном реальном сервере).<br>**Применение**: виртуальные машины предпочтительны, когда требуется строгая изоляция и возможность работы с различными операционными системами.                                                                                                                                                                                                                                                                          |
| Контейнер Docker        | **Контейнер Docker** — это изолированное пространство, которое позволяет запускать приложения с их зависимостями отдельно от основной системы. Некоторые особенности контейнера:<br>- Контейнер использует ядро хост-системы и изолирует только процессы приложения.<br>- Контейнер включает в себя код приложения, системные библиотеки, переменные окружения и конфигурационные файлы — всё, что нужно для работы программы.<br>- Контейнеры легковесные — занимают от десятка до сотен мегабайт, в то время как виртуальная машина требует гигабайты.<br>- Контейнеры изолированы друг от друга: падение одного контейнера не затрагивает остальные.<br>- Контейнеры переносимы — один и тот же образ запустится одинаково на локальном компьютере, тестовом сервере и в продакшн-окружении.<br>**Применение**: контейнеры Docker подходят для сценариев, где важны скорость и эффективность использования ресурсов, тогда как виртуальные машины — для задач, где требуется строгая изоляция и возможность работы с различными операционными системами.<br>**Важно**: Docker не требует выделять контейнеру конкретные ресурсы, как виртуальная машина, а работает прямо на ресурсах операционной системы. |

### Какой командой можно создать и запустить контейнер из образа в Docker?

| Под темы         | Заметка                                                                                                                                                                                                               |
| ---------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Ответ            | Основная команда для создания и запуска контейнера из образа в Docker — **`docker run`**.                                                                                                                             |
| Как это работает | 1. Docker проверяет, есть ли образ  локально. Если нет — автоматически загружает его из реестра (например, Docker Hub).<br>2. Создаёт контейнер на основе образа.<br>3. Запускает контейнер с указанными параметрами. |

### Каковы основные компоненты архитектуры Docker?

| Под темы                 | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| ------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Клиент (Docker Client)   | **Docker Client** — это интерфейс, через который пользователи взаимодействуют с Docker. Он может быть:<br>- инструментом командной строки (docker CLI);<br>- графическим пользовательским интерфейсом (GUI), например Docker Desktop;<br>- API, который общается с Docker Daemon.<br>Клиент позволяет пользователям выполнять команды для сборки, запуска и управления контейнерами. Например, команды:<br>- docker build — создаёт образ из Dockerfile;<br>- docker run  — запускает контейнер;<br>- docker ps  — выводит список запущенных контейнеров.<br>Клиент взаимодействует с Docker Daemon через REST API.                                                                                                                                                                                                                                                                                                                                                              |
| Демон (Docker Daemon)    | **Docker Daemon** (также известен как dockerd) — это фоновый процесс, который управляет объектами Docker: образами, контейнерами, сетями и томами.<br>**Некоторые задачи Docker Daemon**:<br>- управление контейнерами и образами: создание, запуск, остановка и удаление контейнеров и образов;<br>- сетевые операции: обеспечение связи между контейнерами как на локальном уровне, так и между различными машинами;<br>- управление хранилищами: обеспечение работы с дополнительными файловыми системами контейнеров.<br>Демон слушает Docker API-запросы от клиентов и выполняет их в системе. Для запуска используется команда dockerd.                                                                                                                                                                                                                                                                                                                                    |
| Образы (Docker Images)   | **Docker Images** — это неизменяемый шаблон, который включает операционную систему, зависимости, библиотеки и само приложение.<br>**Особенности образов**:<br>- Состоят из нескольких слоёв: каждый слой создаётся на основе предыдущего.<br>- Образы могут быть версионированы с помощью тегов: например, nginx:latest — nginx:1.19 — конкретная версия образа Nginx.<br>- Образы описаны в текстовом файле Dockerfile.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| Реестр (Docker Registry) | **Docker Registry** — это репозиторий для хранения Docker-образов.<br>**Некоторые особенности реестра**:<br>- Образы группируются в репозитории — наборы связанных образов с одинаковыми именами.<br>- Некоторые реестры — общедоступные, например Docker Hub, где хранятся миллионы официальных и пользовательских образов.<br>- Другие реестры — частные, которые используются для хранения корпоративных образов в облаке или локально.<br>**Взаимодействие с реестром**:<br>- Пользователи могут отправлять образы в реестр (команда docker push) и загружать их docker pull).<br>- Реестр поддерживает контроль доступа: например, команда docker login позволяет аутентифицироваться в публичном реестре.<br>**Пример взаимодействия**: при запуске контейнера из образа docker run nginx Docker Daemon проверяет наличие образа в локальном хранилище. Если образ отсутствует, демон обращается к реестру для загрузки и создаёт контейнер на основе загруженного образа. |

### Каким образом Docker обеспечивает изоляцию ресурсов для контейнеров?

| Под темы                              | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| ------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение                           | **Docker обеспечивает изоляцию ресурсов для контейнеров с помощью механизмов, основанных на функциях ядра Linux**. Эти механизмы позволяют каждому контейнеру работать в изолированном окружении, не взаимодействуя напрямую с другими контейнерами или с основной операционной системой.<br>Для изоляции используются пространства имён (namespaces) и группы управления ресурсами (cgroups). Также Docker обеспечивает изоляцию файловой системы и сети.                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| Пространства имён (namespaces)        | Создают отдельное представление системных ресурсов для каждого контейнера. Некоторые типы пространств имён, используемые в Docker:<br>- **PID** — изолирует процессы внутри контейнера: каждый контейнер имеет свою собственную нумерацию процессов, которая не пересекается с нумерацией процессов на хосте или в других контейнерах.<br>- **Network** — изолирует сетевые интерфейсы, маршруты и сетевые устройства (такие как виртуальные сетевые интерфейсы) для каждого контейнера.<br>- **UTS (UNIX Time-Sharing)** — изолирует имя хоста и доменное имя (hostname и domain name) для каждого контейнера.<br>- **User** — позволяет изолировать идентификаторы пользователей и групп внутри контейнера от хоста. Например, внутри контейнера можно запускать процессы от имени пользователя root, но на хосте этот пользователь будет отображаться как пользователь с более низкими привилегиями. |
| Группы управления ресурсами (cgroups) | Позволяют ограничивать и мониторить ресурсы (CPU, память, ввод-вывод на диск и т. д.), используемые контейнером. Некоторые способы использования cgroups в Docker:<br>- **Управление CPU** — Docker может ограничить использование CPU контейнера, чтобы один контейнер не мог монополизировать вычислительные ресурсы.<br>- **Управление памятью** — cgroups позволяют Docker устанавливать лимиты использования памяти для контейнеров. Если контейнер превышает выделенную память, он может быть ограничен или остановлен.<br>- **Управление блочным вводом-выводом** — Docker может контролировать пропускную способность дискового ввода-вывода, доступную для контейнера. Это предотвращает, что контейнеры, интенсивные в вводе-выводе, ухудшают производительность других контейнеров или хостовой системы.                                                                                     |

### Какие возможности предоставляет Dockerfile и как он используется для создания образов контейнеров?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| ----------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Dockerfile** — это текстовый файл с инструкциями по созданию образов контейнеров в экосистеме Docker. Он используется для автоматизации сборки образов, документирует процесс сборки и позволяет воспроизвести его.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| Возможности | Некоторые возможности Dockerfile:<br>- **Указание базового образа** — команды FROM. Чаще всего используется для образов с операционной системой и предустановленными компонентами. Если указанный образ не найден, Docker попытается найти и загрузить его из индекса образов.<br>- **Выполнение команд** внутри контейнера во время сборки образа. Так можно установить зависимости или обновить пакеты до нужной версии.<br>- **Копирование файлов** из локальной файловой системы в контейнер. Чаще всего копируют исходный код приложения.<br>- **Установка рабочей директории** для последующих инструкций. Так можно последовательно работать с файлами в разных директориях.<br>- **Определение аргументов по умолчанию** при запуске контейнера.<br>- **Задание команды**, которая будет выполнена при запуске контейнера.<br>Образ Docker — это набор слоёв файловой системы, наложенных друг на друга. Каждая инструкция в Dockerfile создаёт новый слой. При сборке Docker складывает слои в определённом порядке и объединяет их в единую файловую систему. |
| Синтаксис   | Некоторые особенности синтаксиса Dockerfile:<br>- **Инструкции** не чувствительны к регистру, но принято их писать заглавными буквами для визуального отделения от аргументов.<br>- **Аргументы** — это данные, передаваемые в инструкции, например команды оболочки командной строки или названия базовых образов.<br>- **Инструкции обрабатываются сверху вниз**.<br>Важно оптимизировать Dockerfile для эффективного использования кеша: инструкции, которые редко меняются, размещают в начале файла.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |

### Как можно реализовать коммуникацию между контейнерами, работающими на одном хосте в Docker?

| Под темы                                   | Заметка                                                                                                                                                                                                                                                                                                                                                                                       |
| ------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Bridge‑сеть  (по умолчанию)                | **Как работает:**  <br>При установке Docker автоматически создаётся сеть `bridge` (имя — `bridge`). Контейнеры, запущенные без явного указания сети, подключаются к ней.<br>**Особенности:**<br>- контейнеры видят друг друга по **IP‑адресам**;<br>- нет встроенного DNS — нельзя обращаться по именам контейнеров;<br>- подходит для простых сценариев, но не рекомендуется для продакшена. |
| Пользовательская  bridge‑сеть              | **Преимущества:**<br>- встроенный **DNS**: контейнеры доступны по именам;<br>- изоляция групп контейнеров;<br>- гибкие настройки (подсети, шлюзы).                                                                                                                                                                                                                                            |
| Сеть `host`                                | **Суть:** контейнер использует сетевой стек хоста (нет изоляции).<br>**Когда применять:**<br>- нужны высокие сетевые производительность;<br>- приложение слушает все интерфейсы хоста<br>**Ограничения:**<br>- нет изоляции между контейнерами и хостом;<br>- не работает на macOS/Windows (только Linux).                                                                                    |
| Сеть `none`                                | **Суть:** полное отключение сети для контейнера.  <br>**Применение:**<br>- изолированные задачи без сетевого взаимодействия;<br>- повышенная безопасность.                                                                                                                                                                                                                                    |
| Использование  Docker Compose              | **Преимущества:**<br>- автоматическое создание сети для сервисов;<br>- DNS‑имена по именам сервисов;<br>- простая настройка зависимостей.                                                                                                                                                                                                                                                     |
| Прямое подключение  контейнеров (`--link`) | **Проблемы:**<br>- нет DNS;<br>- жёсткая зависимость от порядка запуска;<br>- ограниченность функционала.                                                                                                                                                                                                                                                                                     |
| Многосетевая  конфигурация                 | **Применение:**<br>- разделение трафика (например, frontend/backend);<br>- доступ к разным группам сервисов.                                                                                                                                                                                                                                                                                  |

### Какие методы можно использовать для управления хранением данных в контейнерах Docker?

| Под темы                                           | Заметка                                                                                                                                                                                                                                                                                                                              |
| -------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Временное  хранилище  (по умолчанию)               | **Как работает:**<br>- при запуске контейнера  создаётся слой «чтение‑запись» поверх неизменяемых слоёв образа;<br>- все изменения файлов сохраняются в этом слое.<br>**Ограничения:**<br>- данные **теряются при остановке/удалении контейнера**;<br>- не подходит для баз данных, логов, пользовательского контента.               |
| Тома Docker  (Volumes)                             | **Особенности:**<br>- управляются Docker (хранятся в `/var/lib/docker/volumes/`);<br>- существуют независимо от контейнеров;<br>- поддерживаются в Linux и Windows;<br>- можно подключать к нескольким контейнерам.                                                                                                                  |
| Привязанные  монтирования  (Bind Mounts)           | **Как работает:**<br>- связывает каталог/файл хоста с путём в контейнере;<br>- данные хранятся **непосредственно на хосте**.<br>**Когда использовать:**<br>- для разработки (синхронизация кода между хостом и контейнером);<br>- доступ к конфигурационным файлам хоста;<br>- когда нужен прямой контроль над расположением данных. |
| tmpfs (временное  хранилище в оперативной  памяти) | **Особенности:**<br>- данные хранятся в RAM (не записываются на диск);<br>- исчезают при остановке контейнера;<br>- высокая скорость чтения/записи.<br>**Применение:**<br>- кэши;<br>- сессии;<br>- временные файлы.                                                                                                                 |
| Плагины хранилищ  (Storage Plugins)                | **Для продвинутых сценариев:**<br>- подключение сетевых файловых систем (NFS, CIFS);<br>- облачные хранилища (AWS EBS, Google Persistent Disk);<br>- специализированные драйверы (например, для баз данных).                                                                                                                         |
