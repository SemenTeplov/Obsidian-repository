### Что такое immutable (неизменяемые) коллекции в Java?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| ----------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Неизменяемые (immutable) коллекции в Java** — это коллекции, которые **не могут быть изменены после создания**. Они гарантируют, что ни один поток не может изменять их состояние, что увеличивает безопасность при многопоточной работе с коллекциями.<br>Важно различать термины «неизменяемый» (immutable) и «неизменяющийся» (unmodifiable). «Неизменяемый» подразумевает полное отсутствие возможности изменений, в то время как «неизменяющийся» не допускает изменение структуры списка, но позволяет модифицировать отдельные его элементы. |
| Виды        | В Java неизменяемые реализации доступны для разных типов коллекций, например:<br>- **List** — упорядоченная коллекция, допускающая дубликаты.<br>- **Set** — коллекция уникальных элементов.<br>- **Map** — хранит данные в виде пар «ключ-значение», причём ключи должны быть уникальными.<br>В Java 9 добавлен пакет java.util.immutable, который предоставляет неизменяемые реализации коллекций этих типов.                                                                                                                                       |

### Как можно создать immutable список в Java?

| Под темы  | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| --------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| До Java 9 | До Java 9 для создания неизменяемого списка использовали **метод Collections.unmodifiableList**. Он возвращал неизменяемый вид указанного списка, но исходный список всё ещё мог быть изменён.                                                                                                                                                                                                                                                                                                           |
| С Java 9  | С Java 9 для создания неизменяемого списка используют **статический фабричный метод List.of**. Он создаёт список, содержащий указанные элементы, и **неизменяемый** — его размер и элементы не могут быть изменены после создания.<br>**Некоторые особенности метода:**<br>- Не допускает нулевых элементов — если попытаться добавить null, появится исключение NullPointerException.<br>- Список, созданный List.of(), имеет фиксированный размер и не поддерживает добавление или удаление элементов. |

### Перечислите преимущества использования immutable коллекций

| Под темы           | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение        | **Использование immutable (неизменяемых) коллекций в Java имеет несколько преимуществ**, которые связаны с безопасностью, производительностью, параллелизмом и отладкой.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| Безопасность       | **Предотвращение изменения значений данных** после создания. Это помогает избежать ошибок и упрощает отладку программ. Например:<br>- **Потокобезопасность** — несколько потоков могут одновременно получать доступ к неизменяемой коллекции без необходимости внешней синхронизации, поскольку нет риска, что один поток изменит состояние, пока другой его читает.<br>- **Устойчивость к изменениям** — неизменяемые объекты устойчивы к манипуляциям, что важно при обработке потенциально ненадёжных входных данных. Например, неизменяемые строки (String) предотвращают изменение конфиденциальных данных (паролей, URL-адресов) вредоносным кодом.<br>- **Передача значений методами** — неизменяемые объекты можно передавать в методы без опасности изменения их значений, что позволяет создавать более безопасные и чистые интерфейсы. |
| Производительность | - **Возможность кэширования** — неизменяемые объекты можно легко кэшировать и повторно использовать, так как их состояние не меняется. Например:<br>    - **Кэширование хеш-кодов** в хэш-основанных коллекциях (HashMap, HashSet) — поскольку значение неизменяемого объекта никогда не изменяется, его hashCode() может быть вычислен один раз и закэширован. Последующие вызовы hashCode() будут возвращать закэшированное значение, что повышает производительность.<br>- **Экономия памяти** — неизменяемые коллекции обычно используют меньше памяти по сравнению с изменяемыми аналогами из-за создания новых экземпляров.<br>Однако есть и недостатки: для обновления неизменяемых коллекций не можно выполнять обновления на месте — вместо этого нужно создавать новые копии объектов, что увеличивает накладные расходы.               |
| Параллелизм        | - **Упрощение параллельного программирования** — потокобезопасность неизменяемых коллекций устраняет необходимость в явной синхронизации, что снижает риск условий гонки. Это также упрощает параллельную обработку, так как последовательность операций не имеет значения.<br>- **Возможность использовать многоядерную обработку** — поскольку последовательность операций не имеет значения, можно применять параллельную обработку.<br>Однако важно, чтобы изменяемые объекты, добавленные в неизменяемые коллекции, копировались защитным образом, чтобы предотвратить внешние модификации.                                                                                                                                                                                                                                                  |
| Отладка            | - **Предсказуемость кода** — постоянное состояние неизменяемых объектов упрощает анализ поведения кода. Например:<br>    - **Сравнение объектов** — неизменяемые объекты можно сравнивать просто по их значениям, а не по ссылкам, так как их значения всегда остаются неизменными.<br>    - **Устранение ошибок** — неизменяемые объекты предотвращают случайную или нежелательную модификацию значений данных, что помогает избежать ошибок.<br>Однако использование immutable коллекций не всегда оправдано — обычно нужно как создавать объекты, так и модифицировать их для отражения изменений, происходящих в системе.                                                                                                                                                                                                                     |

### Какие основные отличия между Collections.unmodifiableCollection и Collections.immutableCollection в Java?

| Под темы                           | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| ---------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение                        | **Collections.unmodifiableCollection** и **Collections.immutableCollection** в Java отличаются тем, что **Collections.unmodifiableCollection** возвращает **немодифицируемое представление** коллекции, а **Collections.immutableCollection** — **неизменяемую коллекцию**.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| Collections.unmodifiableCollection | **Возвращает обёртку над коллекцией**, которая предотвращает изменения через ссылку на обёртку. Некоторые особенности:<br>- Любая попытка модифицировать возвращённую коллекцию выдаёт исключение **UnsupportedOperationException**.<br>- Возвращённое представление — **живое**: изменения в оригинальной коллекции видны через него.<br>- Саму коллекцию можно менять, если есть прямой доступ к ней.<br>- Метод не заботится о конкретной реализации — он работает с любой коллекцией.<br>**Применение**: Collections.unmodifiableCollection полезен, когда нужно предоставить доступ к данным только для чтения, но при этом предотвратить случайные изменения. Например, метод может использоваться для возврата внутренних коллекций из методов, чтобы не допускать внешнего изменения.                                                                  |
| Collections.immutableCollection    | **Возвращает неизменяемую коллекцию**, которая не может быть изменена после создания. Некоторые особенности:<br>- Все методы мутатора (изменяющие методы) указаны для создания исключения **UnsupportedOperationException**.<br>- Коллекция не меняется по количеству и составу объектов, которые она содержит.<br>- Неизменяемые коллекции **безопасны для многопоточной работы**: ни один поток не может изменять их состояние.<br>- Для создания неизменяемых коллекций в Java 9 добавлены **фабричные методы** в интерфейсы List, Set и Map, например, List.of() или  Set.of().<br>**Применение**: Collections.immutableCollection используется, когда нужно гарантировать, что коллекция не может быть изменена даже через любую ссылку. Например, неизменяемые коллекции подходят для случаев, когда нужно, чтобы ни один код не мог изменить коллекцию. |

### Каковы особенности работы с immutable коллекциями в многопоточной среде?

| Под темы       | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| -------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение    | **Неизменяемые (immutable) коллекции в Java по своей сути потокобезопасны**. Это означает, что несколько потоков могут обращаться к коллекции одновременно, не опасаясь изменения её состояния.                                                                                                                                                                                                                                                                                                                                                                                              |
| Принцип работы | Неизменяемые коллекции **не требуют явной синхронизации** при многопоточном доступе. Это упрощает многопоточное программирование, так как не нужно беспокоиться о состоянии объектов.<br>Чтобы изменить данные, нужно лишь создать новый экземпляр неизменяемой коллекции. Попытки изменить коллекцию (добавить, удалить элементы) приводят к исключению **UnsupportedOperationException**.<br>**Важно**: неизменяемая коллекция потокобезопасна, только если элементы внутри неё неизменяемые. Если элементы изменяемые, изменение одного из них может привести к изменению всей коллекции. |

### Какие нововведения связанные с immutable коллекциями были введены в Java 9?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| ----------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **В Java 9 введены нововведения, связанные с неизменяемыми (immutable) коллекциями**, — добавлены статические фабричные методы для создания неизменяемых объектов List, Set и Map. Ранее для создания неизменяемой коллекции нужно было создать коллекцию, заполнить её данными и вызвать метод, например — Collections.unmodifiableList.                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| Методы      | В Java 9 в Collections API добавлены **статические фабричные методы**. Они обеспечивают удобный способ создания неизменяемых коллекций. Некоторые из них: <br>- **List.of** — создаёт пустой или непустой неизменяемый список.<br>- **Set.of** — создаёт пустой или непустой неизменяемый набор.<br>- **Map.of** — создаёт объект Immutable Map.<br>- **Map.ofEntries** — создаёт объект Immutable Map.Entry.<br>**Особенности неизменяемых коллекций, созданных с помощью фабричных методов**:<br>- Элементы не могут быть добавлены, удалены или заменены.<br>- Вызов любого изменяющего метода на коллекции всегда вызывает исключение UnsupportedOperationException.<br>- Если элементы в коллекции изменяемы, это может привести к тому, что содержимое коллекции будет выглядеть изменённым. |
| Классы      | В Java 9 добавлен **новый пакет java.util.immutable**, который предоставляет неизменяемые реализации коллекций, таких как List, Set и Map. Эти коллекции создаются с помощью методов из класса **java.util.ImmutableCollections**.<br>**Важно**: неизменяемые коллекции Java 9 — это настоящие неизменяемые коллекции, которые не могут быть изменены после создания. В отличие от обычных коллекций, которые при каждом изменении создают новый объект или копируют данные, неизменяемые коллекции Java 9 создаются только один раз.                                                                                                                                                                                                                                                              |

Важно различать термины «неизменяемый» (immutable) и «неизменяющийся» (unmodifiable): «неизменяемый» подразумевает полное отсутствие возможности изменений, в то время как «неизменяющийся» не допускает изменение структуры списка, но позволяет модифицировать отдельные его элементы. |
| С помощью JDK                 | **Метод Collections.unmodifiableList**. Создаёт из переданного списка его неизменяемый вариант: в него нельзя добавить или удалить элемент.                                                                                                                                                                                                                                                                            |

Создаёт полностью неизменяемый список: нужно вызвать статический метод List.of() и передать элементы списка.                                                                                                                                                                                                                                                                    |
|                               | **Важно**: метод Collections.unmodifiableList только предотвращает структурные изменения, а List.of создаёт полностью неизменяемый список.                                                                                                                                                                                                                                                                             |
| С помощью сторонних библиотек | **Библиотека Guava**. Содержит варианты различных коллекций, в частности, ImmutableList и ImmutableMap, которые специально сделаны неизменяемыми.                                                                                                                                                                                                                                                                      |
|                               | **Библиотека Apache Commons Collections**. Предоставляет API для создания неизменяемого списка: ListUtils.unmodifiableList(list).                                                                                                                                                                                                                                                                                      |

### Какие есть ограничения и недостатки при использовании immutable коллекций в Java?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| ----------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Использование неизменяемых (immutable) коллекций в Java имеет как ограничения, так и недостатки**. Неизменяемые коллекции не могут быть изменены после их создания, и любые операции, которые могли бы изменить данные, на самом деле создают новую версию структуры с изменениями, оставляя исходную версию нетронутой.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| Ограничения | - **Несовместимость с исходным кодом**. Например, попытка изменить параметр или возвращаемый тип с List на ImmutableList может привести к несовместимости с исходным кодом, так как эти типы не связаны друг с другом.<br>- **Локальность использования**. Например, если ImmutableList расширяет List, использовать его можно только локально, поскольку он передаёт границы API как List.<br>- **Возможность изменения базовой коллекции**. Например, метод Collections.unmodifiableList возвращает неизменяемое представление списка, но ссылка на список, которая была передана в этот метод, по-прежнему может быть изменена. Это может привести к ConcurrentModificationExceptions.<br>- **Неизменяемость объектов, содержащихся в коллекции**. Хотя коллекция не может быть изменена, сами объекты, которые она содержит, могут быть изменяемыми. Например, создание неизменяемой коллекции StringBuilder не «замораживает» эти объекты.                                                                               |
| Недостатки  | - **Переполнение памяти**. Создание новых версий структур может потребовать больше памяти по сравнению с изменяемыми структурами.<br>- **Производительность**. В некоторых случаях операции создания новых копий могут быть медленнее, чем изменения в изменяемых структурах данных.<br>- **Сложность отладки**. Из-за неизменяемости состояния легче отслеживать и воспроизводить ошибки, но при этом сложнее находить ошибки, связанные с изменением данных.<br>- **Необходимость защитного копирования**. Изменяемые объекты, добавляемые в неизменяемые коллекции, нужно копировать защитно, чтобы предотвратить внешние изменения.<br>- **Усложнение работы с фреймворками, основанными на отражении**. Неизменяемые объекты усложняют работу с фреймворками, так как требуют внедрения конструктора, и имена аргументов конструктора обычно недоступны через отражение.<br>При этом неизменяемые коллекции полезны в сценариях с частым чтением и редким изменением, так как не требуют синхронизации доступа к данным. |
