| Под темы            | Заметка                                                                                                                                                                                                                                                                                                                        |
| ------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Определение         | Apache Tomcat — это веб-сервер с открытым исходным кодом и контейнер сервлетов, разработанный Apache Software Foundation. Хотя его часто называют «сервером приложений», технически это веб-сервер, предназначенный специально для выполнения Java-сервлетов и отображения JavaServer Pages (JSP).                             |
|                     | Tomcat — это легковесный сервер, ориентированный исключительно на веб-уровень. Это делает его отраслевым стандартом для размещения приложений Spring Boot и микросервисов.                                                                                                                                                     |
| Ключевые компоненты | Catalina (контейнер сервлетов) — это движок, который фактически реализует спецификацию Java Servlet. Он управляет жизненным циклом ваших сервлетов (инициализация, обслуживание, уничтожение). При развертывании файла .war Catalina распаковывает его и загружает классы.                                                     |
|                     | Coyote (конектор) отвечает за сетевое взаимодействие. Он поддерживает протоколы HTTP/1.1, HTTP/2 и AJP. Он управляет пулом потоков, создавая их для эффективной обработки входящих запросов.                                                                                                                                   |
|                     | Jasper (движок JSP) Если ваше приложение использует файлы .jsp (JavaServer Pages), за их обработку отвечает Jasper. Он компилирует файл JSP в стандартный Java-сервлет (файл .class), чтобы Catalina могла его выполнить. Поэтому первая загрузка страницы JSP происходит медленно (из-за компиляции), а последующие — быстро. |
| Архитектура         | Сервер: компонент верхнего уровня. Представляет собой весь экземпляр Tomcat (JVM). Принимает команду завершения работы на определенном порту (по умолчанию 8005).                                                                                                                                                              |
|                     | Сервис: группа из одного или нескольких коннекторов, использующих один и тот же движок. Роль: обеспечивает связь внешнего мира (коннекторов) с внутренней логикой обработки (движком).                                                                                                                                         |
|                     | Коннектор (Coyote): компонент, который принимает входящие соединения. HTTP-коннектор: принимает стандартный веб-трафик через порт 8080 (по умолчанию). AJP-коннектор: принимает трафик от обратного прокси-сервера, такого как Apache HTTPD, через порт 8009.                                                                  |
|                     | Движок (Catalina): мозг всей системы. Представляет собой конвейер обработки запросов. Изучает заголовки входящих запросов, чтобы определить, какой хост и контекст должны их обработать.                                                                                                                                       |
|                     | Хост: представляет виртуальный хост (например, www.example.com. На одном сервере Tomcat может быть несколько хостов.                                                                                                                                                                                                           |
|                     | Контекст: представляет собой отдельное веб-приложение (например, /my-app).                                                                                                                                                                                                                                                     |
| Жизненный цикл      | Принять: Coyote Connector, прослушивающий порт 8080, принимает TCP-соединение.                                                                                                                                                                                                                                                 |
|                     | Parse: анализирует HTTP-заголовки и создает объекты HttpServletRequest и HttpServletResponse.                                                                                                                                                                                                                                  |
|                     | Маршрут (двигатель):Коннектор передает эти объекты в модуль Engine, который обращается к имени хоста (localhost).                                                                                                                                                                                                              |
|                     | Маршрут (хост): Механизм передает запрос соответствующему хосту.                                                                                                                                                                                                                                                               |
|                     | Маршрут (контекст): Хост анализирует путь URL (/my-app) и передает запрос в соответствующий контекст (ваше приложение).                                                                                                                                                                                                        |
|                     | Выполнить (сервлет):Контекст просматривает конкретное сопоставление (/hello) и вызывает соответствующий код Java-сервлета.                                                                                                                                                                                                     |
|                     | Ответ: Сервлет записывает данные в объект ответа, который возвращается по цепочке (Context -> Host -> Engine -> Connector) и отправляется обратно в браузер пользователя.                                                                                                                                                      |
