| Определение      | Spring Cloud предоставляет разработчикам инструменты для быстрого создания некоторых распространенных шаблонов для распределенных систем (например, управление конфигурацией, обнаружение сервисов, автоматические выключатели, интеллектуальная маршрутизация, микропрокси, шина управления, недолговечные микросервисы и тестирование контрактов). Координация распределенных систем приводит к появлению шаблонных решений, и с помощью Spring Cloud разработчики могут быстро создавать сервисы и приложения, реализующие эти шаблоны. Они будут хорошо работать в любой распределенной среде, включая собственный ноутбук разработчика, «голые» центры обработки данных и управляемые платформы, такие как Cloud Foundry. |
| Особенности      | - Распределенная/версионная конфигурация  <br>- Регистрация и обнаружение сервисов  <br>- Маршрутизация  <br>- Вызовы между сервисами  <br>- Балансировка нагрузки  <br>- Автоматические выключатели  <br>- Распределенная система обмена сообщениями  <br>- Краткоживущие микросервисы (задачи)  <br>- Контрактное тестирование на основе потребителя и производителя                                                                                                                                                                                                                                                                                                                                                         |
