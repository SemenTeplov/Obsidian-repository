| Под темы     | Заметка                                                                                                                                                                                                                                       |
| ------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение  | Антипаттерн - распространённый подход к решению класса часто встречаемых проблем, которые являются не эффективным, рискованным и непродуктивным.                                                                                              |
|              | Виды - архитектурные антипаттерны (возникают при проектировании), управленческие антипаттерны (возникаю при управлении), антипаттерны разработки (возникают при написании программы).                                                         |
| Антипаттерны | Аналитический паралич - чрезмерное анализирование ситуации при планировании что парализует какую-либо деятельность. Решение - определение долгосрочной цели, не концентрироваться на пустяках, установка сроков, не гнаться за совершенством. |
|              | Божественный объект - излишняя концентрация на одном объекте.                                                                                                                                                                                 |
|              | Одиночка - один экземпляр на всю программу с глобальным доступом.                                                                                                                                                                             |
|              | Полтергейст - класс, который не несет никакой пользы.                                                                                                                                                                                         |
|              | Жесткое кодирование - сильная привязка к конфигурациям или системному окружению.                                                                                                                                                              |
|              | Лодочный якорь - хранение не используемых частей системы.                                                                                                                                                                                     |
|              | Объектная выгребная яма - в пуле хранят объекты, которые нигде не используются.                                                                                                                                                               |
|              | Золотой молоток - уверенность в каком-либо решении и применение этого решения повсеместно.                                                                                                                                                    |
|              | Преждевременная оптимизация.                                                                                                                                                                                                                  |
|              | Спагетти код - код, который является плохо структурированным, запутанным и трудным для понимания.                                                                                                                                             |
|              | Магические числа - константы в программе без пояснения их цели и смысла.                                                                                                                                                                      |
|              | Программирование копированием и вставкой - бездумное копирование чужого кода.                                                                                                                                                                 |
|              | Изобретение колеса - реализация решений, которые уже есть.                                                                                                                                                                                    |
|              | Проблема йо йо - размытая структура приложения в связи с размытой фрагментацией.                                                                                                                                                              |
|              | Ненужная сложность - внесение ненужной сложности в решение.                                                                                                                                                                                   |
### Вывод
---
Антипатерны - ловушки, в которые разработчик часто встревает из за острой нехватки времени, невнимательности, неопытности или пинков со стороны менеджеров.
