### Какова основная идея хэш-таблиц и как они работают?

| Под темы       | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| -------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение    | **Основная идея хэш-таблиц** — **преобразование ключа в индекс с помощью специальной функции — хэш-функции**. Это позволяет обеспечить быстрый доступ к данным: если ключ для желаемого значения известен, ключ может быть хэширован, и местоположение значения найдено непосредственно.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| Принцип работы | Процесс работы хэш-таблицы включает несколько этапов:<br>1. **Хеширование ключа**. Ключ (например, «Алексей») пропускается через хеш-функцию, которая возвращает число (например, 42).<br>2. **Вычисление индекса**. Результат хеширования преобразуется в индекс массива с помощью операции остатка от деления: index = hash % array.length.<br>3. **Сохранение значения**. Значение сохраняется в ячейку массива с этим индексом<br>**Важно**: хорошая хеш-функция должна обладать несколькими свойствами: детерминированность (один и тот же ключ всегда даёт один и тот же хеш), равномерность (хеши должны равномерно распределяться по всему диапазону значений), быстрота (вычисление хеша должно быть быстрым) и минимизация коллизий (разные ключи должны давать разные хеши). |
| Структура      | В основе хэш-таблицы — **массив**, в котором хранятся пары «ключ-значение». **Ключ** — уникальный идентификатор элемента, который передаётся в хэш-функцию. **Значение** — информация, привязанная к ключу (например, число, строка или объект).<br>**Существует два основных вида хэш-таблиц**:<br>1. **С открытой адресацией** — элементы массива — пары.<br>2. **С цепочками** — элементы массива — списки пар.                                                                                                                                                                                                                                                                                                                                                                      |

### Каковы основные операции, которые можно выполнить с массивами?

| Под темы   | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| ---------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Чтение     | Элементы массива доступны по **индексам** — целочисленным значениям, указывающим позицию элемента. Индексация начинается с 0: первый элемент имеет индекс 0, второй — индекс 1 и т. д..                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| Добавление | Элементы можно добавлять в массив в разных местах: в начало, конец или на заданную позицию.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| Удаление   | В одних языках, например в Java, неиспользуемые массивы автоматически удаляет сборщик мусора. В других это приходится делать вручную.<br>- **пузырьковая сортировка (Bubble Sort)** — последовательное сравнение и обмен соседних элементов, если они стоят в неправильном порядке.<br>- **Сортировка вставками (Insertion Sort)** — массив условно делится на отсортированную и неотсортированную части, на каждом шаге алгоритм берёт один элемент из неотсортированной части и вставляет его в правильную позицию в отсортированной части.<br>- **Сортировка выбором (Selection Sort)** — на каждом шаге находит минимальный элемент из неотсортированной части массива и помещает его в начало.<br>- **Быстрая сортировка (QuickSort)** — выбирает опорный элемент и разделяет массив так, чтобы все элементы меньше опорного оказались слева, а больше — справа, затем рекурсивно сортирует левую и правую части. |

### Что такое двоичное дерево поиска и каковы его основные свойства?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| ----------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Двоичное дерево поиска** (англ. binary search tree, BST) — это двоичное дерево, для которого выполняются дополнительные условия, связанные с организацией хранения данных.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| Устройство  | Двоичное дерево поиска состоит из **узлов (вершин)** — записей вида (data, left, right), где:<br>- **data** — данные, привязанные к узлу;<br>- **left** и **right** — ссылки на узлы-потомки (левый и правый сыновья соответственно);<br>- **parent** — ссылка на родительский элемент (в каждом узле, кроме корневого).<br>**Корневой узел** — первый узел дерева, он не имеет родителя, при этом все остальные узлы — его потомки. **Листовой узел** — узел, у которого нет потомков: его правое и левое поддерево равно null.                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| Свойства    | - **Для каждого узла**:<br>    - Все узлы в левом поддереве имеют значения меньше значения узла.<br>    - Все узлы в правом поддереве имеют значения больше значения узла.<br>- **Оба поддерева** — левое и правое — являются двоичными деревьями поиска.<br>- **Данные в каждом узле** должны обладать ключами, на которых определена операция сравнения «меньше» либо «равно».<br>- **Эффективность поиска** связана с сбалансированностью дерева — максимальной разницей между глубиной левого и правого поддерева среди всех вершин. Есть два крайних случая: сбалансированное бинарное дерево (где каждый уровень имеет полный набор вершин) и вырожденное дерево, где на каждый уровень приходится по одной вершине. Вырожденное дерево эквивалентно связанному списку.<br>- **Время выполнения всех основных операций** пропорционально глубине дерева. Например, скоростные характеристики поиска в сбалансированном дереве — O(log n), в вырожденном — O(n). |

### Каковы преимущества и недостатки использования связанных списков по сравнению с массивами?

| Под темы       | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| -------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Массивы        | **Преимущества**:<br>- **Быстрый доступ к элементам** благодаря последовательному расположению элементов в памяти. Доступ к элементу по индексу выполняется за константное время (O(1)).<br>- **Простота реализации** — массивы легко реализовать и использовать в большинстве языков программирования.<br>- **Упорядоченность** — элементы массива располагаются в памяти последовательно, что позволяет быстро обращаться к ним по индексу.<br><br>**Недостатки**:<br>- **Фиксированный размер** — размер массива определяется при его создании и не может быть изменён в процессе выполнения программы. Расширение массива требует создания нового и копирования элементов, что может быть затратно по времени и памяти.<br>- **Отсутствие встроенных операций** — массивы не поддерживают такие операции, как вставка или удаление элементов, без дополнительной реализации.<br>- **Невозможность удаления или добавления элемента** без сдвига других при использовании статических массивов.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| Связные списки | **Преимущества**:<br>- **Гибкость** — связные списки могут быть изменены динамически в процессе выполнения программы. Можно добавлять, удалять и изменять элементы списка без необходимости перестраивать всю структуру данных.<br>- **Эффективность вставки и удаления** — вставка и удаление элементов в связном списке выполняется быстрее, чем в массиве. Это связано с тем, что для вставки или удаления элемента в массиве необходимо перемещать все элементы после него, что может занять много времени.<br>- **Динамическое выделение памяти** — связные списки позволяют динамически выделять память под элементы списка, что означает, что список может быть увеличен или уменьшен в зависимости от потребностей программы.<br>- **Универсальность** — связные списки могут использоваться для хранения различных типов данных, включая числа, строки, структуры и другие объекты.<br><br>**Недостатки**:<br>- **Отсутствие прямого доступа к элементам** — для доступа к элементу списка необходимо пройти весь список от начала до нужного элемента. Это может занять много времени, особенно если список содержит большое количество элементов.<br>- **Дополнительное использование памяти** — каждый элемент списка содержит ссылку на следующий элемент, что требует дополнительного использования памяти. Это может стать проблемой при работе с большими списками.<br>- **Неэффективность при поиске элементов** — поиск элемента в связном списке может занять много времени, особенно если список содержит много элементов.<br>- **Ограниченная поддержка параллелизма** — связные списки не подходят для параллельной обработки данных, так как каждый элемент списка должен быть обработан последовательно.<br>- **Сложность прямого доступа к элементу** — определение физического адреса по его индексу (порядковому номеру) в списке.<br>- **Соседние элементы списка** могут быть распределены в памяти нелокально, что снизит эффективность кэширования данных в процессоре. |

### Как реализуется алгоритм поиска в глубину на графе и для каких задач он обычно используется?

| Под темы       | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| -------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение    | **Алгоритм поиска в глубину (DFS, Depth-First Search)** — один из методов обхода графа, стратегия которого — как можно глубже исследовать выбранный путь до достижения пункта назначения или конца пути.                                                                                                                                                                                                                                  |
| Принцип работы | Алгоритм основывается на концепции «погружайся глубже, головой вперёд». Он начинает поиск с одной вершины и движется по графу, углубляясь в определённом направлении, пока не дойдёт до конца пути или тупика. Когда тупик или точка назначения найдены, алгоритм возвращается к предыдущей вершине, чтобы исследовать другие пути.                                                                                                       |
| Реализация     | Алгоритм DFS может быть реализован по-разному, например:<br>- **Рекурсивная реализация**. Алгоритм начинается с начальной вершины, помечает её как посещённую и затем рекурсивно вызывает себя для каждого непосещённого соседа.<br>- **Нерекурсивная версия**. Используется, когда глубина графа велика, использование рекурсии может привести к переполнению стека. В таких случаях алгоритм использует явный стек для хранения вершин. |

### Объясните, как работает алгоритм двоичного поиска и в каких условиях он наиболее эффективен.

| Под темы              | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| --------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Определение           | **Алгоритм двоичного (бинарного) поиска** (также известен как метод деления пополам или дихотомия) — алгоритм поиска элемента в отсортированном массиве (векторе). Используется в информатике, вычислительной математике и математическом программировании.                                                                                                                                                                                                                                                                                                                                                                                                        |
| Принцип работы        | 1. **Сортировать массив данных**.<br>2. **Разделить массив пополам** и найти середину.<br>3. **Сравнить срединный элемент** с заданным искомым элементом.<br>4. **Если искомое число больше среднего** — продолжать поиск в правой части массива (если он отсортирован по возрастанию): делить её пополам, повторяя пункт <br>5. **Если заданное число меньше** — алгоритм продолжит поиск в левой части массива, снова возвращаясь к пункту 3.<br>Процесс продолжается, пока не будет найден искомый элемент или не останется только один элемент для сравнения. Если искомый элемент не найден, возвращается информация об отсутствии такого элемента в массиве. |
| Условия эффективности | Алгоритм двоичного поиска **наиболее эффективен, когда нужно находить элемент в отсортированном массиве**. В таких случаях он позволяет значительно экономить время выполнения операций поиска, поскольку массив данных на каждом шаге разделяется надвое и одна половина сразу отбрасывается.<br>**Однако у бинарного поиска есть и ограничение**: он требует предварительной сортировки данных по возрастанию. Сложность сортировки — не менее O(n log n). Поэтому для коротких списков по-прежнему рекомендуется использовать линейный поиск.                                                                                                                   |

### Опишите, как реализуется и используется алгоритм Дейкстры для поиска кратчайшего пути на взвешенном графе.

| Под темы       | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| -------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение    | **Алгоритм Дейкстры** — классический алгоритм для поиска кратчайшего пути в взвешенном графе от одной вершины (начальной) до всех остальных вершин. Разработан голландским учёным Эдсгером Дейкстрой в 1959 году. <br>**Реализация** алгоритма включает несколько этапов: <br>1. **Инициализация**. Устанавливают расстояние до начальной вершины равным 0, а до всех остальных — бесконечности.<br>2. **Выбор вершины**. На каждом шаге выбирают непосещённую вершину с наименьшим известным расстоянием.<br>3. **Релаксация**. Обновляют расстояния до соседей выбранной вершины, если через неё путь короче.<br>4. **Повторение**. Продолжают процесс, пока не посетим все вершины или не достигнем целевой вершины.<br>**Ограничение**: алгоритм не работает корректно для графов с отрицательными весами рёбер. В таком случае лучше использовать алгоритм Беллмана-Форда. |
| Принцип работы | Алгоритм Дейкстры работает по принципу «жадного» алгоритма: на каждом шаге стремится минимизировать текущую общую стоимость пути. Некоторые особенности работы:<br>- **Использование очереди с приоритетом** вместо классической очереди. В очереди хранится не только номер вершины, но и вычисленное расстояние до неё, по которому ведётся сортировка.<br>- **При обработке выбранной вершины** все ещё не посещённые соседи добавляются в очередь для посещения (расстояние до каждой из них рассчитывается как расстояние до текущей вершины + длина ребра).<br>- **Алгоритм заканчивается**, когда на очередном шаге не останется неокрашенных вершин или если расстояние до всех неокрашенных вершин будет равно бесконечности (то есть эти вершины являются недостижимыми).                                                                                             |

### Какие существуют методы балансировки двоичных деревьев поиска и в чем заключается их необходимость?

| Под темы              | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| --------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение           | **Для балансировки двоичных деревьев поиска используются разные методы**, которые обеспечивают сбалансированность структуры, что важно для эффективности операций (поиска, вставки, удаления). Некоторые из них: АВЛ-деревья, красно-чёрные деревья и 2-3-деревья.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| АВЛ-деревья           | **Сбалансированность** определяется разностью высот правого и левого поддеревьев любого узла. Если эта разность по модулю не превышает 1, то дерево считается сбалансированным.<br>**Необходимость балансировки** возникает, если нарушается главное правило структуры: поддеревья-потомки одного узла начинают различаться больше чем на один уровень. Если разница в количестве уровней становится равна 2 или –2, запускается балансировка: связи между предками и потомками изменяются и перестраиваются так, чтобы сохранить правильную структуру.<br>**Преимущества**:<br>- Высота дерева логарифмически зависит от числа узлов, что гарантирует логарифмическую зависимость времени работы основных операций от числа ключей, хранимых в дереве.<br>- Вероятность получить сильно несбалансированное АВЛ-дерево крайне мала, а риск, что оно выродится, практически отсутствует. |
| Красно-чёрные деревья | **Сбалансированность** достигается за счёт введения дополнительного атрибута узла дерева — «цвета». Этот атрибут может принимать одно из двух возможных значений — «чёрный» или «красный».<br>**Необходимость балансировки** заключается в том, что благодаря ограничениям путь от корня до самого дальнего листа не более чем вдвое длиннее, чем до самого ближнего, и дерево примерно сбалансировано. Это позволяет быстро выполнять основные операции дерева поиска: добавление, удаление и поиск узла.<br>**Преимущества**:<br>- Операции вставки, удаления и поиска требуют в худшем случае времени, пропорционального длине дерева, что позволяет красно-чёрным деревьям быть более эффективными, чем обычные двоичные деревья поиска.                                                                                                                                            |
| 2-3-деревья           | **Сбалансированность** достигается за счёт того, что каждое левое, правое и центральное поддерево имеет одну и ту же высоту.<br>**Необходимость балансировки** заключается в том, что это позволяет содержать равные (или почти равные) объёмы данных. Также нелистовые вершины, которые содержат одно или два поля, указывающие на диапазон значений в их поддеревьях, используются для направления функции поиска к нужному поддереву и, в конечном итоге, к нужному листу.<br>**Преимущества**:<br>- Все листовые вершины находятся на одном уровне от корня дерева.<br>- Нелистовые вершины используются для направления функции поиска к нужному поддереву и, в конечном итоге, к нужному листу.                                                                                                                                                                                   |

### Каковы особенности реализации и применения хэш-таблиц с открытой адресацией по сравнению с методом цепочек?

| Под темы                 | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| ------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Определение              | **Хеш-таблицы с открытой адресацией (закрытое хеширование) и метод цепочек** — два основных метода разрешения коллизий (ситуаций, когда разным ключам соответствует одно значение хеш-функции) в хеш-таблицах. Оба метода имеют преимущества и недостатки, и выбор зависит от конкретных требований и условий использования.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| Метод цепочек            | **Принцип работы**: каждая ячейка хеш-таблицы содержит не отдельный элемент, а указатель на связный список, содержащий все элементы, ключи которых отображаются в данный индекс. При возникновении коллизии новый элемент просто добавляется в соответствующий связный список.<br>**Особенности**:<br>- Коллизии приводят к тому, что появляются цепочки длиной более одного элемента.<br>- Операции поиска или удаления данных требуют просмотра всех элементов соответствующей цепочки, чтобы найти в ней элемент с заданным ключом.<br>- Для добавления данных нужно добавить элемент в конец или начало соответствующего списка.<br>**Применение**: метод цепочек гибок и прост в реализации, подходит для работы с таблицами небольшой ёмкости. Однако эффективность поиска во многом зависит от выбора хеш-функции и от распределения ключей: неудачный выбор может привести к неравномерному распределению ключей по хеш-таблице, что увеличит длину связных списков и снизит производительность поиска.                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| Метод открытой адресации | **Принцип работы**: все элементы хранятся непосредственно в хеш-таблице, без использования связанных списков. Если ячейка пуста, в неё записывается какое-либо предопределённое значение, например, None.<br>**Особенности**:<br>- При коллизиях алгоритм ищет следующую свободную ячейку согласно определённому правилу. Например, используется метод линейного исследования: при возникновении коллизии следующие за текущей ячейки проверяются одна за другой, пока не найдётся пустая ячейка, куда и помещается элемент.<br>- Может возникнуть ситуация, когда хеш-таблица окажется полностью заполненной, так что будет невозможно добавлять в неё новые элементы. В этом случае решением может быть динамическое увеличение размера хеш-таблицы с одновременной её перестройкой.<br>- **Проблема**: формирование длинных последовательностей занятых ячеек, что негативно сказывается на скорости вставки, поиска и удаления ключей. Для снижения эффекта кластеризации используется другая стратегия разрешения коллизий — двойное хеширование: для определения шага смещения исследований при коллизии в ячейке используется другая хеш-функция, вместо линейного смещения на одну позицию.<br>**Применение**: метод открытой адресации используется, когда известно количество и частота вставляемых или удаляемых ключей. Метод цепочек в основном применяется, когда неизвестно, сколько и как часто будут вставляться или удаляться ключи. |
