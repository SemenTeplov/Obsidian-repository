### Что такое SQL?

| Под темы    | Заметка                                                                                                                                                                                                                    |
| ----------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **SQL (Structured Query Language)** — **язык структурированных запросов**. Предназначен для работы с **реляционными базами данных (БД)** — массивами информации, которые связаны между собой и представлены в виде таблиц. |
| Особенности | стандартизирован: одни и те же SQL-запросы работают с БД с разными системами управления;                                                                                                                                   |
|             | прост и понятен: запрос информации с его помощью выглядит как команда на английском;                                                                                                                                       |
|             | декларативен: пользователь запрашивает ожидаемый результат, а не прописывает шаги к его достижению.                                                                                                                        |

### Какие операции можно выполнять с помощью DML?

| Под темы | Заметка                                                                                                                                                                                                                                                                                                            |
| -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Ответ    | **С помощью команд DML (Data Manipulation Language) можно выполнять операции по манипулированию данными в базе данных**. К DML относятся команды **INSERT**, **UPDATE**, **DELETE** и **SELECT**. Эти команды позволяют добавлять, изменять, удалять и получать данные из таблиц, не затрагивая схему базы данных. |

### Чем отличается DDL от DML?

| Под темы                         | Заметка                                                                                                                             |
| -------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------- |
| DDL (Data Definition Language)   | **Управляет структурой базы данных**. Команды DDL создают, изменяют и удаляют объекты базы данных: таблицы, индексы, схемы и т. д.. |
| DML (Data Manipulation Language) | **Управляет данными внутри объектов базы данных**. Команды DML позволяют добавлять, изменять, удалять и получать данные из таблиц.  |

### Как можно изменить структуру таблицы, используя DDL?

| Под темы | Заметка                                                                                                       |
| -------- | ------------------------------------------------------------------------------------------------------------- |
| Ответ    | С помощью команды ALTER (Data Definition Language) в SQL (DDL) можно изменить структуру существующей таблицы. |
| Описание | добавлять или убирать столбцы;                                                                                |
|          | изменять тип данных;                                                                                          |
|          | задавать или убирать ограничения, например NOT NULL;                                                          |
|          | переименовывать таблицу или столбец.                                                                          |

### Приведите пример команды DCL.

| Под темы | Заметка                                                                  |
| -------- | ------------------------------------------------------------------------ |
| GRANT    | Используется для предоставления пользователю прав доступа к базе данных. |
| REVOKE   | Используется для отзыва разрешений у пользователя.                       |

### Какие транзакции поддерживает TCL и как их использовать?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                 |
| ----------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **TCL (Transaction Control Language) поддерживает управление транзакциями в базе данных**. Транзакция — это блок SQL-запросов, которые успешно завершаются как единое целое. Промежуточные результаты операции не видны для других транзакций, весь блок запросов будет выполнен или полностью отменён в случае ошибки. |
| Виды        | **COMMIT** — фиксирует все изменения, сделанные в текущей транзакции. Как только транзакция зафиксирована, она становится постоянной и не может быть отменена.                                                                                                                                                          |
|             | **ROLLBACK** — отменяет все изменения, сделанные в текущей транзакции, и восстанавливает базу данных в состояние, предшествующее началу транзакции.                                                                                                                                                                     |
|             | **SAVEPOINT** — создаёт точку сохранения внутри транзакции, к которой можно позже вернуться. Это позволяет откатывать изменения только до указанной точки, не затрагивая остальные части транзакции.                                                                                                                    |
|             | **SET TRANSACTION** — устанавливает свойства транзакции, например, уровень изоляции и режим доступа.                                                                                                                                                                                                                    |

### Какие стратегии оптимизации запросов вы знаете?

| Под темы | Заметка                                                                                                                                                                                                                                                |
| -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Ответ    | **Использование индексов** на часто используемых столбцах, чтобы ускорить поиск и фильтрацию данных. Например, B-Tree-индексы эффективны для точного поиска и диапазонных запросов, Hash-индексы — для точного поиска, но не для диапазонных запросов. |
|          | Избегание SELECT * — указание конкретных столбцов, которые нужны, уменьшает объём данных, которые нужно передать и обработать.                                                                                                                         |
|          | **Использование JOIN вместо подзапросов** — в большинстве случаев джойны эффективнее сложных вложенных запросов.                                                                                                                                       |
|          | **Кэширование запросов** — использование уже вычисленных результатов, что уменьшает нагрузку на базу данных.                                                                                                                                           |
|          | **Сбор статистики** по таблицам — это помогает оптимизатору выбирать наилучшие планы выполнения.                                                                                                                                                       |

### Как реализовать версионирование данных в реляционной базе данных?

| Под темы      | Заметка                                                                                                                                                                                                                                                                                                                                                                  |
| ------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Ручные методы | **Создание таблицы версий**. В ней фиксируются идентификатор версии и дата её создания. В таблицах, подлежащих версионированию, добавляется ссылка на соответствующий идентификатор из этой таблицы.                                                                                                                                                                     |
|               | **Метод «таблиц-двойников» (таблиц истории)**. Для каждой основной таблицы создаётся дублирующая таблица, включающая все поля из исходной таблицы, а также поле-тип операции над записью. При изменениях основной записи в таблице истории создаётся копия изменённой записи с отметкой, кто и когда её изменил.                                                         |
| Триггеры      | **Захват вставок и обновлений записей** с помощью триггеров для набора таблиц. Затем эти события записываются в таблицу или файл журнала изменений. Таким образом можно зафиксировать все изменения, включая удаление данных, а не только вставку и обновления.                                                                                                          |
|               | **Проверка данных** до вставки в таблицу на допустимость ограничениям целостности, а после — выполнение операций над только что вставленными данными.                                                                                                                                                                                                                    |
| Миграции      | **Использование скриптов миграций**. Они содержат инструкции для обновления схемы базы данных с одной версии на другую, а также могут включать откат изменений. Для упрощения отслеживания версии каждой миграции присваивается уникальный номер версии (например, числовая нумерация или использование меток времени).                                                  |
|               | **Автоматизация миграций** с помощью специализированных инструментов, например, Flyway или Liquibase. Они позволяют автоматически генерировать миграции, отслеживать их и проводить сравнение текущего состояния базы данных с желаемым.                                                                                                                                 |
| Журналы       | **Фиксация изменений** на основе журнала транзакций базы данных. По мере того, как в исходных отслеживаемых таблицах выполняются операции вставки, обновления и удаления, в журнал добавляются записи, описывающие эти изменения. Затем процесс отслеживания считывает журнал и добавляет сведения об изменениях в связанную с отслеживаемой таблицей таблицу изменений. |
|               | **Фиксация метаданных** об изменениях: характера изменения (вставка, обновление, удаление), времени изменения и конкретной транзакции, связанной с изменением.                                                                                                                                                                                                           |

### Приведите пример использования всех четырёх типов команд (DDL, DML, DCL, TCL) в рамках одного бизнесценария.

| Под темы                           | Заметка                                                                                                                                                                                                                                                                                                                                                                  |
| ---------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| DDL (Data Definition Language)     | `-- Создаем таблицу пользователей CREATE TABLE users ( id SERIAL PRIMARY KEY, username VARCHAR(50) NOT NULL, email VARCHAR(100) UNIQUE, password_hash VARCHAR(255) ); -- Создаем таблицу товаров CREATE TABLE products ( id SERIAL PRIMARY KEY, name VARCHAR(100), price DECIMAL(10,2), stock_quantity INTEGER );`                                                       |
| DML (Data Manipulation Language)   | `-- Добавляем нового пользователя INSERT INTO users (username, email, password_hash) VALUES ('user1', 'user1@example.com', 'hashed_password'); -- Добавляем товар INSERT INTO products (name, price, stock_quantity) VALUES ('Смартфон', 29999.99, 100); -- Обновляем количество товара на складе UPDATE products SET stock_quantity = stock_quantity - 1 WHERE id = 1;` |
| DCL (Data Control Language)        | `-- Предоставляем права на работу с таблицами GRANT SELECT, INSERT, UPDATE ON users TO sales_manager; -- Предоставляем права только на просмотр товаров GRANT SELECT ON products TO customer;`                                                                                                                                                                           |
| TCL (Transaction Control Language) | `BEGIN; -- Начинаем транзакцию покупки UPDATE products SET stock_quantity = stock_quantity - 1 WHERE id = 1; INSERT INTO orders (user_id, product_id, quantity) VALUES (1, 1, 1); COMMIT;`                                                                                                                                                                               |
