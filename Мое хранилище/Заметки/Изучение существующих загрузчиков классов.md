### Какие три основных загрузчика классов существуют в JVM?

| Под темы                 | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| ------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение              | В виртуальной машине Java (JVM) существуют **три основных загрузчика классов**: Bootstrap Class Loader, Extension Class Loader и Application Class Loader (System Class Loader).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| Bootstrap Class Loader   | **Корневой загрузчик классов**. Не является классом Java, а представляет собой платформозависимый машинный код.<br>**Функция**: загружает основные классы базовой среды выполнения (Java Runtime Environment, JRE), включая классы пакетов java.util и java.lang из пути jdk/jre/lib/.<br>**Особенности**:<br>- Загружает стандартные пакеты Java, такие как java.lang, java.net, java.util, java.io. <br>- Реализован в нативном коде, входит в состав JVM.<br>- Каждый загрузчик, за исключением базового, является потомком абстрактного класса java.lang.ClassLoader.                                                                                                                                                                                                                                                                                                                                                                                                                   |
| Extension Class Loader   | **Загрузчик классов расширений**. Подкласс загрузчика классов начальной загрузки и суперкласс загрузчика классов приложений.<br>**Функция**: загружает расширения стандартных библиотек Java, которые присутствуют в каталоге $JAVA_HOME/jre/lib/ext.<br>**Особенности**:<br>- По умолчанию находится в каталоге jre/lib/ext, но может быть задан системным свойством java.ext.dirs.<br>- Каждый загрузчик, за исключением базового, является потомком абстрактного класса java.lang.ClassLoader.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| Application Class Loader | **Загрузчик классов приложений** (System Class Loader). Дочерний по отношению к Extension Class Loader.<br>**Функция**: загружает специфические для приложения классы из каталогов и хранилищ, указанных переменной среды CLASSPATH или системным свойством java.class.path. В этом пути находятся классы, которые программист определяет для своего конкретного приложения. <br>**Особенности**:<br>- По умолчанию путь к классу устанавливается как текущий каталог приложения, но его можно изменить, добавив параметр командной строки -classpath или -cp.<br>- Это загрузчик по умолчанию для некоторых модулей JDK, которые содержат утилиты или экспортируют API утилит.<br>**Принцип иерархии делегирования**: когда загрузчик получает запрос на загрузку класса, он сначала пытается делегировать этот запрос своему родителю. Если родительскому загрузчику не удаётся загрузить класс, то он пытается загрузить его самостоятельно, отыскивая файл .class в определённых путях. |

### Что такое загрузчик классов в Java?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| ----------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Загрузчик классов (ClassLoader) в Java** — это компонент виртуальной машины Java (JVM), отвечающий за **динамическую загрузку классов в память во время выполнения программы**. В Java каждый класс должен быть загружен в память перед его использованием, и именно Classloader выполняет эту задачу.<br>Каждый загрузчик, за исключением базового, — потомок абстрактного класса java.lang.ClassLoader.                                                                                                                                                                                                                                                                                                                                                                       |
| Функции     | ClassLoader выполняет несколько функций:<br>- **Загрузка классов** — находит и загружает файлы классов (обычно с расширением .class) из classpath в память.<br>- **Проверка классов** — после загрузки Classloader проверяет байт-код класса для обеспечения его корректности и безопасности.<br>- **Связывание классов** — Classloader связывает загруженные классы, включая проверку ссылок и разрешение символов.<br>- **Инициализация классов** — Classloader инициализирует загруженные классы, вызывая их статические инициализаторы и инициализаторы переменных.<br>ClassLoader полезен в случаях, когда классы могут быть неизвестны на этапе компиляции и должны быть загружены из внешних источников, например, при создании плагинов или модулей, добавляемых на лету. |

### Какой загрузчик классов используется по умолчанию для загрузки пользовательских классов?

| Под темы | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| -------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Ответ    | **System ClassLoader** (Application ClassLoader) — загрузчик классов, который используется по умолчанию для загрузки **классов приложения** в Java.<br>Это следует из иерархии загрузчиков классов в JVM: Bootstrap ClassLoader, Extension ClassLoader и System ClassLoader. Определение того, какой загрузчик загружает тот или иной класс, осуществляется по принципу иерархии делегирования: когда загрузчик получает запрос на загрузку класса, он сначала пытается делегировать этот запрос своему родителю. Если родительскому загрузчику не удаётся загрузить класс, то он пытается загрузить его самостоятельно, отыскивая файл .class в определённых путях. |

### Какие задачи выполняет системный (или прикладной) загрузчик классов?

| Под темы | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| -------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Ответ    | **Системный (прикладной) загрузчик классов** (Application ClassLoader, также известен как System ClassLoader) в Java **загружает классы приложения из директории classpath**. Обычно это директория, указанная в переменной окружения CLASSPATH или параметре командной строки -cp.<br>В Java существует иерархия загрузчиков классов, где каждый тип имеет свою задачу. Кроме системного, в JVM есть встроенный (bootstrap) загрузчик классов и загрузчик классов платформы (platform class loader). |

### Как можно написать собственный загрузчик классов и для чего это может потребоваться?

| Под темы       | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| -------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Определение    | **В Java можно написать собственный загрузчик классов** — наследник абстрактного класса java.lang.ClassLoader. Это позволяет контролировать процесс загрузки классов и загружать их из различных источников.                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| Принцип работы | Загрузчики классов в Java работают по принципу **делегирования**. Когда JVM встречает класс, она проверяет, уже загружен ли он. Если нет, процесс загрузки передаётся по цепочке загрузчиков. Каждый загрузчик в иерархии ищет класс в своих определённых местах и делегирует поиск, если необходимо.<br>Некоторые принципы работы загрузчиков:<br>- **Видимость** — классы, загруженные родительскими загрузчиками, видны дочерним, но не наоборот.<br>- **Уникальность** — класс может быть загружен только однажды. Механизм делегирования позволяет убедиться, что загрузчик, инициирующий загрузку класса, не перегрузит загруженный ранее в JVM класс. |

### Как работает делегирование загрузки классов в Java?

| Под темы       | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| -------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение    | **Делегирование загрузки классов в Java** — это механизм, при котором запрос на загрузку класса передаётся **родительскому загрузчику**, и попытка загрузить класс самостоятельно выполняется, только если родительский загрузчик не смог найти и загрузить класс.<br>Загрузчики классов в Java построены в виде иерархии с отношениями «родитель-ребёнок». Каждый «дочерний» загрузчик классов имеет «родителя», за исключением корневого загрузчика.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| Принцип работы | Когда загрузчик получает запрос на загрузку класса, он сначала пытается делегировать этот запрос своему родителю. Родительский загрузчик также пытается найти желаемый класс у себя в кэше. Если класс уже был загружен и загрузчик знает о его местонахождении, то будет возвращён объект Class этого класса. Если нет, поиск продолжается, пока не дойдёт до базового загрузчика.<br>Если и в базовом загрузчике нет информации об искомом классе (т. е. он ещё не был загружен), будет выполнен поиск байт-кода этого класса по расположению классов, о котором знает данный загрузчик. Если загрузить класс не удастся, управление вернётся обратно загрузчику-потомку, который будет пытаться выполнить загрузку из известных ему источников.<br>**Цель делегирования** — загружать классы тем загрузчиком, который максимально близко находится к базовому, — так достигается максимальная область видимости классов. Также механизм позволяет убедиться, что загрузчик, инициирующий загрузку класса, не перегрузит загруженный ранее в JVM класс. |

### Как решается проблема разрешения циклических зависимостей между классами при загрузке?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| ----------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Проблема разрешения циклических зависимостей между классами при загрузке в Java решается по-разному в зависимости от фреймворка**. Циклические зависимости возникают, когда класс A зависит от B, а B — от A, и контейнер не может создать эти классы, потому что для создания каждого из них требуется, чтобы другой класс уже существовал.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| В Spring    | **Spring** может разрешать циклические зависимости, используя создание прокси и двухэтапное построение бинов. Некоторые способы решения:<br>- **Использование аннотации @Lazy**. Позволяет отложить создание бина до момента, когда он действительно понадобится. Spring не попытается сразу создать бин, позволяя другим бинам завершить инициализацию.<br>- **Применение метода для внедрения зависимости вместо конструктора** (Setter Injection). Spring создаёт бины с использованием сеттеров после инициализации основного объекта, что позволяет разорвать круг зависимостей.<br>- **Пересмотр архитектуры и разделение зависимостей**. Циклические зависимости часто являются признаком проблемной архитектуры. Если две сущности сильно зависят друг от друга, возможно, их стоит объединить или, наоборот, разделить на более мелкие компоненты.<br>**Важно**: в некоторых случаях Spring может разрешить циклические зависимости, если выставить свойство allow-circular-references в true, но это работает только с инжекцией через поля (field-based) или через сеттеры (setter-based). Constructor-based инжекция не поддерживается в этом случае. |

### Какие методы класса ClassLoader используются для загрузки классов?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| ----------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| loadClass   | **Загружает класс с указанным двоичным именем**. По умолчанию метод выполняет поиск классов в следующем порядке:<br>1. Вызывает  findLoadedClass(String) — проверяет, был ли класс уже загружен. Если класс уже загружен, метод возвращает его, иначе — null.<br>2. Вызывает  loadClass(String) в загрузчике родительского класса. Если родительское значение равно null, вместо него используется загрузчик классов, встроенный в виртуальную машину. <br>3. Вызывает findClass(String) — находит класс. Если класс был найден, и флаг resolve имеет значение true, метод вызывает resolveClass(Class) для результирующего объекта класса. Если класс не был найден, выбрасывается исключение ClassNotFoundException<br>**Особенности**:<br>- Метод служит точкой входа для загрузчика классов.<br>- Пользователь может настроить собственный ClassLoader для загрузки классов из определённого источника, переопределив метод  loadClass(). |
| findClass   | **Находит класс с указанным двоичным именем**. Метод может выбрасывать исключение ClassNotFoundException, если класс не существует.<br>**Особенности**:<br>- Нестатический метод, доступен только с объектом класса.<br>- Подклассы ClassLoader, которые следуют модели делегирования для загрузки классов, должны переопределить этот метод.<br>- Метод вызывается методом loadClass() после проверки родительского загрузчика на наличие запрошенного класса.                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| defineClass | **Преобразует массив байтов в экземпляр класса Class**. Экземпляры этого вновь определённого класса могут быть созданы с помощью Class.newInstance.<br>**Особенности**:<br>- Метод объявляется как final и не может быть переопределён.<br>- Сначала файл класса считывается в байтовый массив, метод defineClass принимает этот байтовый массив в качестве входных данных и возвращает эквивалентный объект Class.<br>- Если данные не содержали допустимого класса, метод выбрасывает ClassFormatError.                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| register    | **Регистрирует загрузчик классов как способный к параллельной загрузке**. Это необходимо, если загрузчик поддерживает параллельную работу. Метод называется ClassLoader.registerAsParallelCapable.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |

### Как JVM обрабатывает случай, когда два разных загрузчика классов загрузили классы с одинаковыми именами?

| Под темы       | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| -------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение    | **JVM обрабатывает случай, когда два разных загрузчика классов загрузили классы с одинаковыми именами, как разные классы**. Это связано с тем, что каждый загрузчик классов создаёт своё собственное пространство имён.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| Принцип работы | Когда JVM загружает класс, она использует иерархию загрузчиков классов. Каждый загрузчик отвечает за поиск и загрузку классов. Если два класса с одинаковым именем загружаются разными загрузчиками, **они считаются разными классами**, даже если их байт-код идентичен. <br>Это позволяет:<br>- **Изолировать классы** друг от друга, например, в сервлет-контейнерах и других серверных приложениях, где разные приложения могут использовать разные версии одной и той же библиотеки.<br>- **Предотвращать конфликты** при использовании нескольких версий одной библиотеки в JVM.<br>Каждый загрузчик делегирует свои полномочия родителю: перед поиском класса для загрузки он пытается узнать, не был ли загружен нужный класс раньше. |
