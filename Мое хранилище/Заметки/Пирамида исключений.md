### Что такое исключение в Java?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| ----------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Исключение (Exception) в Java** — это событие, которое нарушает нормальный ход выполнения программы. Проще говоря, это «сигнал» от JVM: что-то пошло не так во время выполнения кода.<br>Исключения могут быть вызваны различными причинами, такими как неправильный ввод данных пользователем, аппаратные сбои или недоступность ресурсов. <br>Если не обработать такое событие, программа завершится с ошибкой.                                                                                                                                                                                                                                                                                                                                                                       |
| Виды        | В Java исключения делятся на две фундаментальные категории: <br>1. **Проверяемые (checked)** — исключения, которые компилятор заставляет обрабатывать явно. Компилятор требует либо перехватить их в блоке try-catch, либо объявить в сигнатуре метода с помощью throws. Примеры: IOException, SQLException.<br>2. **Непроверяемые (unchecked)** — исключения, которые компилятор не заставляет обрабатывать. К ним относятся Error и подклассы RuntimeException. Их не нужно объявлять в сигнатуре метода, и компилятор не требует их явной обработки. Примеры: NullPointerException — попытка обращения к объекту через null-ссылку, ArrayIndexOutOfBoundsException — выход за пределы массива.                                                                                         |
| Обработка   | Для обработки исключений в Java используется **конструкция try-catch-finally**. Каждый блок в этой конструкции имеет свою специфическую функцию:<br>- **try** — очерчивает границы защищённого кода, который потенциально может выбросить исключение.<br>- **catch** — перехватывает конкретный тип исключения и содержит логику его обработки.<br>- **finally** — содержит код, который выполнится всегда, независимо от того, было ли выброшено исключение. Обычно используется для освобождения ресурсов.<br>**Особое внимание следует уделить порядку блоков catch**: исключения перехватываются в порядке их объявления, поэтому более специфические исключения должны обрабатываться перед более общими. В противном случае компилятор выдаст ошибку, так как код будет недостижим. |

### Какие существуют типы исключений в Java?

| Под темы                  | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| ------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение               | В Java исключения делятся на **проверяемые (checked)** и **непроверяемые (unchecked)**. Это разделение связано с тем, как компилятор проверяет код на наличие потенциальных исключений.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| Checked (проверяемые)     | **Проверяемые исключения** — это исключения, которые компилятор заставляет обрабатывать явно. Они являются подклассами Exception, но не относятся к RuntimeException.<br>- Компилятор требует либо перехватить исключение в блоке try-catch, либо объявить в сигнатуре метода с помощью ключевого слова throws.<br>- Чаще всего связаны с внешними ресурсами, такими как файлы, базы данных или сетевые соединения.<br>**Общее правило**: проверяемые исключения следует использовать для условий, которые вызывающий код может разумно ожидать и от которых может восстановиться, непроверяемые — для программных ошибок, некорректного использования API или условий, от которых восстановление маловероятно.                                                                                                      |
| Unchecked (непроверяемые) | **Непроверяемые исключения** — это исключения, которые компилятор не заставляет обрабатывать. К ним относятся Error и подклассы RuntimeException.<br>**Особенности**:<br>- Их не нужно объявлять в сигнатуре метода, и компилятор не требует их явной обработки.<br>- Обычно возникают из-за ошибок в логике программы или тестов.<br>**Общее правило**: непроверяемые исключения дают больше свободы, но требуют большей дисциплины — разработчик сам решает, когда и где обрабатывать ошибку, что может привести к неожиданным падениям программы, если не быть внимательным.<br>**Важно**: для разработки сложных систем часто требуется создание собственных типов исключений, отражающих специфику предметной области. Это позволяет сделать код более читаемым, а ошибки — более понятными и диагностируемыми. |

### Как отличается проверяемое исключение от непроверяемого?

| Под темы                 | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| ------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение              | Проверяемое (checked) и непроверяемое (unchecked) исключения в программировании (например, в Java) отличаются тем, как компилятор проверяет код на наличие потенциальных исключений.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| Проверяемые исключения   | **Проверяемые** — это исключения, которые **проверяются на этапе компиляции**. Компилятор заранее проверяет код на наличие потенциальных исключений и, если их найдёт, не скомпилирует код, пока разработчик не обработает их или не пробросит наверх.<br>Как правило, это ошибки, возникшие по вине внешних обстоятельств или пользователя приложения — например, неправильно указали имя файла.<br>Если метод может выбросить проверяемое исключение, он должен либо обработать его с помощью try-catch, либо объявить в сигнатуре с помощью ключевого слова throws.<br>Проверяемые исключения следует использовать в случаях:  <br>- восстановимые условия — когда вызывающий код может предпринять осмысленные действия для исправления ситуации;<br>- критические бизнес-процессы — когда пропуск обработки ошибки недопустим с точки зрения бизнес-логики;<br>- операции с внешними системами — взаимодействие с файлами, сетью, базами данных;<br>- API, требующие явной обработки ошибок — когда проектируется публичный API, требующий от клиентов обязательного рассмотрения исключительных случаев. |
| Непроверяемые исключения | **Непроверяемые** — это исключения, которые **не проверяются компилятором**. Они являются подклассами RuntimeException.<br>**Особенности**:<br>- Разработчик не обязан их объявлять или обрабатывать, хотя может это делать.<br>- Непроверяемые исключения чаще всего являются следствием ошибок программиста, и компилятор не в силах предусмотреть все возможные неправильные ситуации.<br>- Такие ошибки могут быть в любом месте программы, поэтому компилятор не требует указывать непроверяемые исключения в объявлении метода.<br>**Главное различие** — в обязательствах, накладываемых на разработчика:<br>- **Проверяемые исключения** заставляют явно прописывать стратегию обработки ошибки: «либо обработай, либо объяви, что она может произойти». Это делает код более надёжным, но иногда приводит к его «засорению» блоками try-catch и объявлениями throws.<br>- **Непроверяемые исключения** дают больше свободы, но требуют большей дисциплины. Разработчик сам решает, когда и где обрабатывать ошибку, что может привести к неожиданным падениям программы, если не быть внимательным.   |

### Как работает механизм обработки исключений в Java?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| ----------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Механизм обработки исключений в Java работает с помощью конструкции try-catch-finally**. Она позволяет структурировать код так, чтобы чётко разделить нормальный ход выполнения, обработку исключительных ситуаций и гарантированные действия.<br>**Процесс работы**:<br>- При возникновении исключения в блоке try обработчик исключения ищется в следующем за ним блоке catch. Если в catch есть обработчик данного типа исключения, управление переходит к нему. Если нет, JVM ищет обработчик этого типа исключения в цепочке вызовов методов, пока не будет найден подходящий catch.<br>- После выполнения блока catch управление передаётся в необязательный блок finally.<br>- Если подходящий блок catch не найден, JVM останавливает выполнение программы и выводит стек вызовов методов (stack trace), выполнив перед этим код блока finally при его наличии.<br>**Цели механизма исключений**:<br>- отделение нормальной логики от обработки ошибок;<br>- пропаганда ошибок вверх по стеку вызовов;<br>- группировка и классификация ошибок. |

### Перечислите и кратко опишите уровни иерархии исключений в Java.

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| ----------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Иерархия исключений в Java** начинается с класса **Throwable**, который является базовым для всех ошибок и исключений в языке. От Throwable наследуются два больших семейства: **Error** и **Exception**. Все остальные классы исключений — производные от этих двух классов.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| Error       | **Error** — это ошибки, связанные с самой виртуальной машиной Java или системными ресурсами. Обычно их не обрабатывают в коде, потому что они сигнализируют о серьёзных проблемах, от которых программа не сможет восстановиться.<br>При возникновении такой ошибки мало что можно сделать, кроме как уведомить пользователя и попытаться корректно завершить программу.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| Exception   | **Exception** — это исключения, которые можно и нужно обрабатывать. Они показывают, что во время работы программы возникла проблема, с которой можно что-то сделать.<br>**Исключения делятся на два типа**:<br>1. **Проверяемые исключения (checked)** — компилятор требует обязательно обработать или указать в сигнатуре метода. К ним относятся, например, IOException и SQLException. Чаще всего связаны с внешними ресурсами, такими как файлы, базы данных или сетевые соединения.<br>2. **Непроверяемые исключения (unchecked)** — наследуются от RuntimeException. Их можно не ловить, и компилятор не будет ругаться, но они часто приводят к падению программы, если их не предусмотреть. К таким исключениям относятся NullPointerException, IndexOutOfBoundsException и NoSuchElementException. Обычно они возникают из-за ошибок в логике программы или тестов. |

### Как можно использовать try-with-resources для автоматической очистки ресурсов в Java?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| ----------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Конструкция try-with-resources в Java позволяет автоматически очищать ресурсы** — объекты, которые необходимо закрывать после завершения программы. Это специализированная разновидность традиционного блока try, представленная в Java 7.<br>Ресурс — это любой объект, которым реализуется интерфейс AutoCloseable (включает интерфейс Closeable). Типичные примеры: файловые потоки, подключения баз данных, сетевые сокеты и другие объекты, связанные с вводом-выводом.                                                                                                                                                                                                                                                                                                   |
| Синтаксис   | Базовая структура оператора try-with-resources аналогична традиционному блоку try, но с отличием: ресурс объявляется и инициализируется в скобках сразу после ключевого слова try.<br>**Некоторые особенности синтаксиса:**<br>- Ресурс объявляется и инициализируется в скобках после ключевого слова try. Это объявление представляется обычно в виде объявления переменной, где ResourceType — это тип класса, а resource — имя переменной.<br>- Внутри блока try ресурс используется при необходимости выполнения операций: считывание файла, запрос к базе данных, отправка данных по сетевому сокету.<br>- При завершении выполнения блока try — в обычном режиме или из-за исключения — для корректного высвобождения ресурса автоматически вызывается его метод close(). |
| Правила     | - **Все объекты**, управляемые оператором try-with-resources, должны реализовывать интерфейс AutoCloseable.<br>- **Объекты, объявленные в операторе try-with-resources**, выполняются в блоке try, но не в блоках catch и finally.<br>- **Если в блоке try выбрасывается исключение и при закрытии ресурса — ещё одно**, второе исключение в Java не удаляется, а добавляется к первому как «подавленное исключение». Так не теряется важная информация о том, что пошло не так при управлении ресурсами.                                                                                                                                                                                                                                                                        |

### Какие стратегии и подходы используются для обработки исключений на разных уровнях абстракции в Java?

| Под темы          | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| ----------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Определение       | **В Java для обработки исключений на разных уровнях абстракции используются разные стратегии и подходы**. Это связано с иерархией исключений, которая позволяет структурировать код и обрабатывать ошибки на разных уровнях.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| Уровни абстракции | Некоторые уровни абстракции, где могут использоваться стратегии обработки исключений в Java:<br>- **Уровень приложения**. Создаётся иерархия исключений, например, базовое исключение для всего приложения, исключения уровня доступа к данным, исключения бизнес-логики.<br>- **Уровень сервиса**. Например, в REST API бизнес-ошибки пробрасываются до слоя контроллеров, где решается, как на них реагировать: показать ошибку, залогировать, вернуть код ответа.<br>- **Уровень инфраструктуры**. Например, контроллер на уровне инфраструктуры десериализует запрос и в случае успеха запрашивает результат у соответствующего сервиса на уровне сервиса.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| Стратегии         | Некоторые стратегии обработки исключений на разных уровнях абстракции в Java:<br>- **Для проверяемых исключений** (проверяются во время компиляции). Исключения должны быть либо перехвачены с помощью блоков try-catch, либо объявлены в сигнатуре метода с использованием ключевого слова throws. Рекомендация — использовать проверяемые исключения для исправимых ситуаций, где вызывающая сторона может предпринять значимые действия.<br>- **Для непроверяемых исключений** (не проверяются во время компиляции). Как правило, такие исключения указывают на ошибки программирования, и они не должны быть обнаружены при нормальных обстоятельствах. Рекомендация — использовать непроверяемые исключения для обозначения ошибок программирования, таких как логические ошибки или неправильное использование API.<br>- **Для кода в глубоком слое бизнес-логики**. Вместо лишней обработки исключений ошибка пробрасывается наружу, а там, ближе к пользователю (например, в контроллере или сервисе верхнего уровня), уже разбирается, как на неё реагировать. Это называется **пробросом исключения** — вызывающему методу говорится: «Я знаю, что тут может быть ошибка, но обрабатывать её должен ты». |

### Как исключения в Java влияют на производительность и как их можно оптимизировать?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Исключения в Java могут влиять на производительность**, но степень их влияния зависит от того, как они реализованы и где возникают. При разумном и правильном использовании логики исключений потери производительности минимальны, но когда исключения используются слишком часто или скрыты внутри критических путей, они могут стать узким местом.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| Влияние     | Некоторые факторы, которые влияют на производительность обработки исключений:<br>- **Генерация исключений**. Это запускает очистку стека, создание объектов и часто запуск механизмов журналирования. Даже в оптимизированных средах выполнения этот процесс потребляет ресурсы ЦП и памяти.<br>- **Перехват исключений**. Особенно негативно сказывается на производительности, если исключения перехватываются слишком широко. Широкие блоки перехвата могут скрывать множество ошибок, заставляя среду выполнения выполнять ненужную оценку условий.<br>- **Влияние на использование ЦП и памяти**. Обработка исключений увеличивает нагрузку на процессор из-за генерации трассировки стека и переключения контекста. Также она расходует память, создавая объекты исключений, особенно при их многократном возникновении в циклах.<br>В небольших приложениях влияние может быть незначительным, но в высокопроизводительных или устаревших системах обработка исключений может стать серьёзным узким местом.                                                                                                                                                                                                                                                                                                                                                                                                              |
| Оптимизация | Некоторые рекомендации по оптимизации работы с исключениями в Java:<br>- **Использовать исключения только для исключительных ситуаций**. Использование их для управления нормальным потоком выполнения программы — серьёзная ошибка проектирования.<br>- **Выбрасывать исключения раньше и обрабатывать их позже**. Создавать исключение всякий раз, когда в коде возникает условие исключения, не ждать выполнения каких-либо строк кода перед выходом из метода. При перехвате исключений функция должна располагаться в конце метода.<br>- **Использовать глобальный обработчик исключений**. Во время выполнения программы всегда будут исключения, которые ранее не были перехвачены в коде, — нужно добавить глобальный обработчик, чтобы логировать и обрабатывать исключение, а также предотвратить сбой приложения при возникновении исключения во время выполнения.<br>- **Разрешить JVM вызывать метод close() для закрываемых ресурсов**. Не пытаться закрыть их самостоятельно — это устранит возможность сложных и трудноустранимых утечек ресурсов. <br>- **Указывать точные исключения** при использовании оператора throw. Вместо общего класса исключений лучше использовать точные исключения, которые может генерировать данный метод.<br>- **Сначала обрабатывать наиболее очевидное исключение**. Разработчик должен сначала обрабатывать наиболее явное исключение, а наименее определённое — последним. |

### Какие лучшие практики существуют для обработки исключений в Java?

| Под темы        | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение     | **Некоторые лучшие практики для обработки исключений в Java** включают рекомендации по структуре кода, использованию различных типов исключений и логированию. Также важно избегать ошибок, которые могут привести к сложностям в отладке и поддержке кода.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| Структура кода  | - **Использовать конструкцию try-catch-finally**. В блоке try находится потенциально опасный код, в catch — код для обработки ошибки, а finally выполняется всегда, даже если произошло исключение (например, для закрытия ресурсов). Некоторые рекомендации:<br>    - **Сохранять код в блоке try** минимальным и специфичным для операций, которые могут вызывать исключения, чтобы избежать перехвата непреднамеренных исключений.<br>    - **Сначала перехватывать наиболее конкретное исключение**, а затем более общие исключения.<br>    - **Убедиться, что блок finally сам по себе не генерирует исключение**, которое потенциально может скрыть исключения, созданные в блоке try.<br>- **Использовать функцию multi-catch** в Java 7, которая позволяет перехватывать несколько исключений в одном блоке catch, уменьшая дублирование кода.<br>- **Учитывать порядок блоков catch** — исключения перехватываются в порядке их объявления, поэтому более специфические исключения должны обрабатываться перед более общими. |
| Типы исключений | - **Соблюдать баланс между проверяемыми (checked) и непроверяемыми (unchecked) исключениями**. Например:<br>    - **Checked исключения** (например, IOException) требуют обязательной обработки через try-catch или объявление в сигнатуре метода с помощью throws.<br>    - **Unchecked исключения** (NullPointerException, RuntimeException) можно не обрабатывать, но лучше это делать.<br>- **Использовать проверяемые исключения** в следующих случаях:  <br>- - восстановимые условия, когда вызывающий код может предпринять осмысленные действия для исправления ситуации;<br>- - критические бизнес-процессы, когда пропуск обработки ошибки недопустим с точки зрения бизнес-логики;<br>- - операции с внешними системами (взаимодействие с файлами, сетью, базами данных).<br>- **Создавать собственные типы исключений** для представления конкретных условий возникновения ошибок в приложении. Это делает код более читаемым.                                                                                           |
| Логирование     | - **Записывать информацию об исключении** для последующего отладки и устранения неполадок. Можно использовать фреймворк логирования (java.util.logging) или стороннюю библиотеку (log4j, Logback). Некоторые рекомендации:<br>    - **Регистрировать не только сообщение разработчика, но и весь объект исключения** — это обеспечивает полное представление об исключении, включая вызываемые методы и трассировку стека.<br>    - **Указывать тип исключения** в логе — это помогает идентифицировать природу проблемы.<br>    - **Генерировать исключения с описательными сообщениями** — они должны как можно точнее описать проблему. Например, для NumberFormatException сообщение должно содержать строку ввода, которая вызвала проблему.                                                                                                                                                                                                                                                                                     |
