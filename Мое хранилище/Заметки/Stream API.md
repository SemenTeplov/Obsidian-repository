### Что такое Stream API в Java?

| Под темы       | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| -------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение    | **Stream API в Java** — это набор инструментов для работы с последовательностями элементов в функциональном стиле. Поток (Stream) не хранит данные, а представляет собой конвейер операций над элементами источника данных.<br>Основная функциональность Stream API сосредоточена в пакете java.util.stream.                                                                                                                                                                                                                                                                                                                                                      |
| Принцип работы | Данные обрабатываются с помощью цепочек промежуточных и терминальных операций. **Особенность**: каждый элемент потока проходит через весь конвейер поэтапно — операции не применяются к коллекции целиком, а работают над каждым элементом поочерёдно.<br>**Некоторые свойства Stream API**:<br>- **Декларативность** — разработчик может описать, что нужно сделать с данными, а не как это должно быть реализовано.<br>- **Ленивость** — операции в потоке не выполняются сразу — они «откладываются» до тех пор, пока не будет вызвана терминальная операция. Это позволяет Java оптимизировать выполнение операций, обрабатывая данные по мере необходимости. |
| Операции       | Методы Stream API делятся на две категории:<br>1. **Промежуточные (intermediate)** — преобразуют поток и возвращают новый поток. Могут быть цепными, не выполняются до вызова терминальной операции. Примеры: filter(), map(), flatMap(), distinct(), sorted(), limit(), skip().<br>2. **Терминальные (terminal)** — завершают работу с потоком и выдают результат. После их вызова поток становится недоступным. Примеры: forEach(), collect(), reduce(), count(), min(), max(), findFirst(), anyMatch(), allMatch().                                                                                                                                            |

### Как создать поток из коллекции в Java с использованием Stream API?

| Под темы              | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| --------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Определение           | **В Java с использованием Stream API поток (Stream) можно создать из коллекции, массива или последовательности элементов**. Для этого используются разные методы, например, .stream(),  Arrays.stream(), Stream.of(). Также поток можно создать из файла с помощью метода Files.lines().<br>**Важно**: Stream API не изменяет исходную коллекцию, и любые операции над потоком (например, фильтрация, маппинг, сортировка) не влияют на сам исходный набор данных. |
| Метод .stream()       | Создаёт поток из коллекции.                                                                                                                                                                                                                                                                                                                                                                                                                                        |
|                       | **Особенности**:<br>- Поток можно использовать только один раз — после выполнения терминальной операции поток закрывается.<br>- Промежуточные операции не выполняются до вызова терминальной операции.                                                                                                                                                                                                                                                             |
| Метод Arrays.stream() | Создаёт поток из существующего массива.                                                                                                                                                                                                                                                                                                                                                                                                                            |
|                       | **Особенности**:<br>- В случае примитивных типов данных (например, int) поток можно дополнительно преобразовать в объектный тип с помощью метода boxed().                                                                                                                                                                                                                                                                                                          |
| Метод Stream.of()     | Создаёт последовательный поток из одного или нескольких указанных элементов.                                                                                                                                                                                                                                                                                                                                                                                       |
|                       | **Особенности**:<br>- По умолчанию возвращает последовательный (непараллельный) поток.<br>- Потоки, созданные с помощью Stream.of(),  по умолчанию не параллельны — для параллельной обработки используется метод stream.parallel().                                                                                                                                                                                                                               |
| Метод Files.lines()   | Создаёт поток из строк файла.                                                                                                                                                                                                                                                                                                                                                                                                                                      |
|                       | **Особенности**:<br>- Метод не читает все строки в список, а заполняет поток по мере потребления, что позволяет эффективно использовать память.<br>- Если файл закодирован в UTF-8, в аргумент метода можно передать StandardCharsets.UTF_8 — это декодирует байты файла в символы с указанной кодировкой.                                                                                                                                                         |

### Для чего используются лямбда-выражения в контексте Stream API?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | Лямбда-выражения в контексте Stream API используются для **упрощения обработки коллекций**. Они позволяют описывать поведение в виде анонимных функций, что делает код более лаконичным и выразительным.<br>Некоторые операции, в которых применяются лямбда-выражения:<br>- **Filter**. Применяет предикат к элементам потока, фильтруя несовпадающие элементы.<br>- **Map**. Преобразует каждый элемент потока с помощью предоставленной функции.<br>- **Sorted**. Сортирует элементы потока на основе компаратора, который можно кратко реализовать с помощью лямбда-выражения.<br>- **ForEach**. Обходит каждый элемент потока, лямбда-выражение определяет действие, выполняемое над каждым элементом.<br>- **Reduce**. Выполняет сокращение элементов потока с помощью заданной функции для получения единого совокупного результата.<br>Также лямбда-выражения облегчают параллельную обработку, поскольку позволяют кратко, но ясно определять операции, выполняемые с элементами потока. |

### Что такое конвеерные и терминальные операторы?

| Под темы                            | Заметка                                                                                                                                                                                                                                                                                                                                                                                            |
| ----------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Конвейерные (intermediate) операции | **Преобразуют поток и возвращают новый поток**. Могут быть цепными, не выполняются до вызова терминальной операции.<br>**Особенности**:<br>- Описываются декларативно, часто с помощью лямбда-выражений.<br>- Возвращают новый поток, что позволяет строить «цепочки» операций (pipeline).<br>- Не выполняются немедленно — откладываются до тех пор, пока не будет вызвана терминальная операция. |
| Терминальные (terminal) операции    | **Завершают работу с потоком и выдают результат**. Могут включать такие действия, как подсчет, сбор в коллекцию, поиск или перебор элементов.<br>**Особенности**:<br>- Инициируют всю цепочку преобразований, закрывают поток и возвращают модифицированные данные.<br>- После вызова терминальной операции поток больше нельзя использовать повторно — это приведёт к ошибке.                     |

### Можно ли использовать один и тот же поток дважды?

| Под темы | Заметка                                                                                                                                                                                                                                                                                                                       |
| -------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Ответ    | **Нет, в Java один и тот же поток нельзя использовать дважды**.<br>Это связано с жизненным циклом потока: после запуска и выполнения потока он переходит в состояние «остановленный» (известное также как «мёртвый»). Если попытаться запустить поток дважды, он выдаст исключение **java.lang.IllegalThreadStateException**. |

### Объясните разницу между методами findFirst() и findAny()

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| ----------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| findFirst() | **Возвращает первый элемент потока**, удовлетворяющий заданному условию, если такой элемент существует. Если поток пуст, метод возвращает пустой объект Optional.<br>**Особенности**:<br>- В последовательных потоках возвращает первый элемент по порядку, так как обработка идёт последовательно.<br>- Для параллельных потоков порядок элементов не определён, поэтому findFirst() может вернуть любой из подходящих элементов.<br>- Метод полезен, когда важно получить именно первый элемент (например, при упорядоченном потоке), обычно в сочетании с фильтрацией.                                                                             |
| findAny()   | **Возвращает любой элемент потока**, удовлетворяющий заданному условию, если такой элемент существует. Если поток пуст, метод возвращает пустой объект Optional.<br>**Особенности**:<br>- В однопоточных сценариях поведение findAny() идентично findFirst().<br>- При параллельной обработке потока метод может вернуть любой элемент, потому что каждый поток обрабатывает свой сегмент данных независимо. Это позволяет завершить работу быстрее, не дожидаясь обработки всех элементов, как в последовательных потоках.<br>- findAny() может быть полезен, когда порядок элементов не важен, а нужно просто получить результат как можно быстрее. |

### Как преобразовать каждый элемент коллекции в другой тип данных с использованием Stream API?

| Под темы        | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| --------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Ответ           | В Stream API в Java для преобразования элементов коллекции в другой тип данных используются методы map(), flatMap() и collect().                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| Метод map()     | **Позволяет применить заданную функцию к каждому элементу потока**, создавая новый поток с преобразованными элементами. В качестве аргумента метод принимает **Function** — функцию, которая определяет, как будет преобразован каждый элемент.<br>**Особенности**:<br>- Является нетерминальной операцией, то есть возвращает новый поток, а не конечный результат.<br>- Преобразование может включать изменение типа элементов, изменение их состояния и даже фильтрацию.                                                                                                                        |
| Метод flatMap() | **Применяется, когда каждый элемент преобразуется в поток или коллекцию**, а затем все эти потоки объединяются в один. Это особенно полезно для работы с вложенными структурами данных, такими как списки списков.<br>**Принцип работы**: flatMap() применяет функцию-маппер к каждому элементу потока, функция-маппер возвращает не просто значение, а поток значений. Все полученные потоки «сплющиваются» в один результирующий поток.                                                                                                                                                          |
| Метод collect() | **Используется для преобразования потока в коллекцию или другой тип данных**. Чаще всего применяется для сбора элементов в список, множество или строку.<br>**Особенности**:<br>- Принимает объект типа **Collector**, который определяет, как именно будут собраны элементы. <br>- Для большинства распространённых операций уже созданы предопределённые коллекторы, к ним можно получить доступ с помощью типа **Collectors**. Например:<br>    - **toList** — преобразует поток в список;<br>    - **toSet** — преобразует поток во множество;<br>    - **toMap** — преобразует поток в карту. |

### Что делает метод collect() в Stream API, и какие существуют варианты его использования?

| Под темы       | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| -------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение    | **Метод collect() в Stream API позволяет собрать элементы потока в различные коллекции или другие структуры данных**. Это терминальная операция, которая завершает обработку потока и возвращает конечный результат.                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| Принцип работы | Метод collect() принимает **объект типа Collector**, который определяет, как будут обрабатываться элементы. Интерфейс Collector содержит пять ключевых методов:<br>- **supplier()** — создаёт новый контейнер результатов;<br>- **accumulator()** — добавляет элемент в контейнер результатов;<br>- **combiner()** — объединяет два контейнера результатов (важно для параллельного исполнения);<br>- **finisher()** — преобразует контейнер в итоговый результат;<br>- **characteristics()** — возвращает характеристики коллектора (влияют на оптимизации).<br>Класс Collectors предоставляет множество готовых реализаций Collector для различных задач агрегации. |

### Что такое Stateless и Statefull операции?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Определение | **Stateless** — «без состояния». Так называют операции, которые обрабатывают каждый элемент потока **независимо от других**. Они не требуют информации о предыдущих или последующих элементах для своей работы, что делает их подходящими для параллельной обработки. <br>Примеры операций без состояния: map() и filter(). Например, в методе filter() каждый элемент проверяется по заданному условию отдельно, и его результат не зависит от других элементов. <br>**Stateful** — «с состоянием». Так называют операции, которые **требуют информации о других элементах потока**. Эти операции не могут начать возвращать результаты, пока не обработают часть или весь поток.<br>Примеры операций с состоянием: sorted(), distinct() или limit(). Например, sorted() должна сначала собрать все элементы, чтобы их отсортировать, а затем уже передать их на последующие этапы. |

### Как реализовать параллельную обработку данных с использованием Stream API?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| ----------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Параллельную обработку данных с использованием Stream API в Java 8 можно реализовать с помощью методов parallel() и parallelStream()**. По умолчанию потоки выполняются последовательно, но с явным вызовом одного из этих методов поток переключается в параллельный режим.<br>Параллельные потоки автоматически разделяют задачу на подзадачи, которые выполняются на разных ядрах процессора, а затем объединяют результаты. Это особенно эффективно для CPU-bound задач с большими объёмами данных.                                                                                                                                                                                                                                           |
| Методы      | - **parallel()** — используется для преобразования последовательного потока в параллельный. Его можно вызвать для любого объекта последовательного потока, чтобы включить параллельную обработку.<br>- **parallelStream()** — вызывается непосредственно для объекта коллекции для создания параллельного потока. Возвращает параллельный поток, позволяющий выполнять параллельную обработку элементов коллекции.<br>**Важно**: не все операции подходят для распараллеливания, так как некоторые могут иметь зависимости или побочные эффекты, которые могут привести к неправильным результатам. Перед использованием параллельных потоков рекомендуется понимать характеристики выполняемых операций и учитывать последствия распараллеливания. |

### Какие особенности работы с состоянием в лямбда-выражениях и Stream API следует учитывать для избегания проблем?

| Под темы         | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| ---------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Определение      | **Для избежания проблем при работе с состоянием в лямбда-выражениях и Stream API в Java важно учитывать особенности, связанные с неизменяемостью данных и типом операций**. Эти особенности влияют на работу с состоянием в разных контекстах, и их неправильное использование может привести к ошибкам.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| Лямбда-выражения | - **Переменные внутри лямбда-выражения должны быть effectively final** — то есть их значение не должно изменяться после инициализации. Это связано с тем, как лямбда-выражения реализованы в JVM — они могут быть выполнены в другом потоке или отложены во времени, что привело бы к проблемам с изменяемыми локальными переменными.<br>- **Лямбда-выражения не могут содержать поля или изменяемые внутренние переменные**. Это ограничивает их в ситуациях, где требуется сохранять данные внутри объекта.<br>- **Лямбда-выражения должны быть свободны от побочных эффектов** — не должны изменять какое-либо внешнее состояние или иметь другие непреднамеренные последствия. Например, не следует изменять внешнюю переменную в лямбда-выражении.                                                                                                                  |
| Stream API       | - **Операции в Stream API делятся на два типа**: stateless (без состояния) и stateful (с состоянием). **Операции без состояния** (map(), filter()) обрабатывают каждый элемент потока независимо от других, не требуют информации о предыдущих или последующих элементах для своей работы. **Операции с состоянием** (sorted(), distinct(), limit()) требуют информации о других элементах потока, не могут начать возвращать результаты, пока не обработают часть или весь поток.<br>- **Если в пайплайне используются только операции без состояния**, то поток может быть обработан «в один проход». **Однако при добавлении операций с состоянием** поток делится на секции, и каждая секция должна завершить свою обработку перед началом следующей.<br>- **После вызова терминальной операции** поток больше нельзя использовать повторно — это приведёт к ошибке. |

### Как использовать метод reduce() в Stream API для агрегации данных, и какие подводные камни могут возникнуть при его использовании?

| Под темы        | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| --------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение     | **Метод reduce() в Stream API используется для агрегации данных** — объединения всех элементов потока в одно итоговое значение. Это может быть сумма всех чисел, объединение строк или любая другая операция, сводящая все элементы к одному результату.<br>Метод принимает на вход два параметра:<br>1. **Начальное значение** — в случае суммы это ноль, в случае умножения — единица и так далее.<br>2. **Лямбда, выполняющая операцию свёртки**. В эту лямбду приходит два параметра: результат предыдущего вычисления и текущий элемент. Накапливаемый результат в процессе вычисления принято называть аккумулятором.<br>Метод возвращает Optional, так как поток может быть пустым, и это позволяет избежать ошибок при работе с отсутствующими элементами. Для упрощения работы можно использовать версию reduce() с начальными значениями, что позволяет избежать работы с Optional.                                                                                                   |
| Принцип работы  | Метод repeatedly применяет логику функции, выполняющей операцию свёртки, к элементам потока. Новое значение аккумулятора всегда равно тому, что было возвращено из предыдущего вызова лямбды.<br>**Важно**: reduce() требует использования аккумулятора без состояния и без вмешательства — в идеале он должен быть неизменяемым. Большинство аккумуляторов создают новые объекты для хранения значения следующего свёртка, и если в потоке много элементов, операция reduce() может создавать много объектов, что может влиять на производительность.                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| Подводные камни | - **Нарушение принципа ассоциативности**. Например, нельзя использовать reduce() для операций вычитания — это нарушит ассоциативность.<br>- **Проблемы с компиляцией**. Например, если в потоке есть поток пользовательских объектов, а типами аргументов накопителя являются Integer и User, но реализация накопителя — сумма целых чисел, компилятор не может определить тип пользовательского параметра. В таких случаях нужно использовать функцию для объединения результатов подпотоков в один (комбайнер).<br>- **Неэффективность для больших потоков**. Например, если использовать reduce() для конкатенации элементов строк, операция reduce() может создавать много объектов строк, что может влиять на производительность.<br>- **Непредсказуемые результаты** при использовании reduce() в распараллеленных потоках. Нужно убедиться, что операция reduce() ассоциативна (порядок следования операндов не влияет на результат), не влияет на источник данных и не имеет состояния. |

### Можно ли использовать Stream для чтения большого файла?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| ----------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Да, Stream API в Java можно использовать для чтения большого файла**. Для этого в Java 8 и более поздних версиях есть метод **Files.lines(),** который возвращает поток строковых элементов.<br>Этот подход позволяет читать файл **построчно**, загружая в память только ту часть файла, с которой работает в данный момент. Стандартный метод — загрузка всего файла в память — не рекомендуется, так как создаёт проблемы: избыточное потребление памяти, риск OutOfMemoryError, невозможность параллельной обработки. |

### Объясните, как работает терминальный метод reduce()

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Определение | **Метод reduce() в Stream API (Java) — терминальная операция, которая обрабатывает элементы потока и возвращает одно значение**. Это может быть сумма всех чисел, объединение строк или любая другая операция, сводящая все элементы к одному результату. <br>После выполнения терминальной операции поток считается использованным и не может быть использован повторно.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| Алгоритм    | Метод reduce() принимает на вход два параметра:<br>1. **Начальное значение** (identity) — предоставляет начальное значение для функции из второго параметра, а также значение по умолчанию, если поток не имеет элементов.<br>2. **Лямбда, выполняющая операцию свёртки** (accumulator). В эту лямбду приходит два параметра: результат предыдущего вычисления и текущий элемент. Накапливаемый результат в процессе вычисления принято называть аккумулятором, так как он аккумулирует значение. <br>**Особенности работы**:<br>- Новое значение аккумулятора всегда равно тому, что было возвращено из предыдущего вызова лямбды.<br>- Если поток выполняется параллельно, метод reduce() может использовать **комбайнер** — функцию, которая собирает результаты промежуточных вычислений в один результат. В случае числовых аккумуляторов обычно комбайнер — функция их суммирования, в случае списков — слияние списков. |
