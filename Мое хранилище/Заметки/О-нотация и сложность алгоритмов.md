### Что такое О-нотация?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **O-нотация (Big O notation) в программировании** — это математическая нотация, которая описывает, как быстро растёт время работы алгоритма при увеличении объёма данных. <br>**Описывает верхнюю границу сложности** — алгоритм точно не будет работать медленнее, чем указано. Также нотация может использоваться для оценки сложности алгоритма по памяти. <br>**Зачем нужна**:<br>- сравнивать эффективность различных алгоритмов для решения одной и той же задачи;<br>- прогнозировать поведение алгоритма при увеличении размера входных данных;<br>- оптимизировать код путём идентификации и улучшения сложных алгоритмов.                                                                     |
| Правила     | - **Учитывается худший случай**. Если в большинстве ситуаций алгоритм выполняется быстро, но существуют случаи, когда эффективность снижается, описываться будет именно этот случай.<br>- **Константные множители не учитываются** — объём входных данных оказывает несущественное влияние на эти компоненты формулы. Например, запись O(n² + 5n + 3) приравнивается к временной сложности O(n²).<br>- **Если выражение содержит сумму**, учитывается то слагаемое, которое растёт быстрее других, например, O(n² + 2n) = O(n²).                                                                                                                                                                        |
| Примеры     | - **O(n)** — линейная сложность. Например, алгоритм поиска наибольшего элемента в не отсортированном массиве — придётся пройтись по всем n элементам, чтобы понять, какой из них максимальный.<br>- **O(log n)** — логарифмическая сложность. Простейший пример — бинарный поиск: если массив отсортирован, можно проверить, есть ли в нём какое-то конкретное значение, методом деления пополам.<br>- **O(n²)** — квадратичная сложность. Такую сложность имеет, например, алгоритм сортировки вставками: в канонической реализации он представляет собой два вложенных цикла: один, чтобы проходить по всему массиву, а второй, чтобы находить место очередному элементу в уже отсортированной части. |

### Какова временная сложность алгоритма линейного поиска в худшем случае?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **O(n)** — временная сложность алгоритма линейного поиска в худшем случае, где **n** — количество элементов в массиве.<br>Это означает, что время выполнения алгоритма пропорционально количеству элементов в массиве. <br>В худшем случае алгоритму может понадобиться проверить **каждый элемент** массива, начиная с первого, и сравнить его с искомым значением. Если элемент совпадает, поиск завершается, и возвращается индекс найденного элемента. Если элемент не совпадает, алгоритм переходит к следующему элементу и повторяет процесс, пока не найдёт совпадение или не достигнет конца массива.<br>В худшем сценарии искомый элемент может оказаться **последним** в массиве, в этом случае потребуется **n итераций** для нахождения элемента. |

### Приведите пример временной сложности алгоритма, которая является линейной.

| Под темы   | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| ---------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Объяснение | Линейная сложность означает, что **время выполнения алгоритма растёт прямо пропорционально размеру входных данных**. Например:<br>- Если размер входных данных увеличивается в 6 раз, время выполнения алгоритма также увеличится в 6 раз.<br>- При увеличении размера входных данных вдвое время выполнения алгоритма также увеличится примерно вдвое.<br>- Если увеличить размер данных в 10 раз, то время выполнения увеличится приблизительно в 10 раз, и так далее.<br>Линейный рост характерен для алгоритмов, которые выполняют постоянное количество операций для каждого элемента входных данных. |
| Примеры    | - **Алгоритм поиска наибольшего элемента в неотсортированном массиве**. Придётся пройтись по всем n элементам массива, чтобы понять, какой из них максимальный.<br>- **Обход массива, списка, коллекции**.<br>- **Простая агрегация данных** (например, вычисление суммы или среднего).<br>**Важно**: линейная сложность не значит «быстрая» — если n — это десятки или сотни миллионов записей, даже линейный проход займёт секунды или минуты. Здесь важно помнить, что асимптотика описывает характер роста, но не абсолютное время.                                                                    |

### Как изменится временная сложность алгоритма сортировки массива, если количество элементов в массиве удвоится?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| ----------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Временная сложность алгоритма сортировки массива может измениться по-разному в зависимости от типа сложности**. Некоторые варианты:<br>**Линейная сложность (O(n))** — время выполнения растёт прямо пропорционально количеству элементов. Удвоение размера входных данных **удвоит и необходимое время** для выполнения алгоритма.<br>**Линеарифметическая или линеаризованная сложность (O(n × log n))** — удвоение размера входных данных увеличит время выполнения **чуть более, чем вдвое**.<br>**Квадратичная сложность (O(n²), O(n^2))** — удвоение размера входных данных увеличивает время выполнения **в 4 раза**. <br>Для оценки временной сложности используется асимптотическая нотация, чаще всего — Big O. Она показывает, как сложность алгоритма растёт с увеличением входных данных, при этом игнорирует конкретные коэффициенты и фокусируется на темпе роста. |

### Что такое амортизационный анализ временной сложности алгоритма?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Определение | **Амортизационный анализ временной сложности алгоритма** — это метод анализа вычислительной сложности алгоритма, который учитывает **среднюю стоимость операций в последовательности**, даже если отдельные операции могут быть дорогими. В отличие от анализа наихудшей временной сложности (который даёт верхнюю оценку времени выполнения одной операции), амортизационный анализ рассматривает стоимость нескольких операций, выполняемых со временем.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| Идея        | Основная идея амортизационного анализа — **распределить стоимость дорогих операций на несколько менее дорогих**. Это позволяет показать, что даже если некоторые из операций последовательности являются дорогостоящими, то при усреднении по всем операциям средняя их стоимость будет небольшой за счёт низкой частоты встречаемости.<br>При амортизационном анализе каждой операции присваивается **учётная стоимость**, которая может быть больше или меньше реальной длительности операции. При этом должно выполняться условие: если исполняется некоторая последовательность операций над структурой данных (причём до этого структура данных находилась в начальном состоянии), то фактическая суммарная длительность всех операций не превосходит суммы их учётных стоимостей.<br>**Важно**: оценка, даваемая амортизационным анализом, не является вероятностной — это оценка среднего времени выполнения операций для худшего случая. |
| Методы      | Есть три основных метода амортизационного анализа:<br>1. **Агрегирующий анализ**. Вычисляется оценка сверху T(n) на время исполнения n операций, затем амортизационная сложность одной операции приравнивается к T(n)/n.<br>2. **Метод предоплаты**. Каждой операции заранее приписывается амортизированная стоимость, которая может отличаться от её реальной стоимости. При этом более «дешёвые» операции обычно имеют амортизированную стоимость выше реальной, а более «дорогие» — ниже реальной. За счёт этого при исполнении дешёвых операций накапливается некоторая сумма, которую можно «потратить» на то, чтобы выполнить операцию, чья амортизированная стоимость ниже реальной.<br>3. **Метод потенциалов**. Накопленная сумма вычисляется как функция («потенциал») от состояния структуры данных. Амортизированная стоимость при этом равна сумме реальной стоимости и изменения потенциала.                                       |

### Чем отличается временная сложность в худшем случае от средней временной сложности?

| Под темы       | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| -------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение    | Временная сложность алгоритма в худшем случае и средняя временная сложность отличаются тем, что худший случай описывает верхнюю границу времени выполнения алгоритма, а средний случай — ожидаемое поведение на типичных данных.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| Худший случай  | **Временная сложность алгоритма в худшем случае** — это функция от размера входных данных, равная **максимальному количеству элементарных операций**, проделываемых алгоритмом для решения экземпляра задачи указанного размера.<br>Некоторые особенности худшего случая:<br>- Указывает на наибольшее время выполнения алгоритма при любых входных данных размера n.<br>- Гарантирует, что алгоритм завершится за указанный промежуток времени.<br>- Чаще используется при анализе: он даёт верхнюю границу, обеспечивая предсказуемость.<br>Пример: для сортировки по вставке для n чисел худший случай — когда числа сортируются в обратном порядке, и для их сортировки требуется O(n²) шагов.                                                                     |
| Средний случай | **Средняя временная сложность** — это математическое ожидание времени работы алгоритма по всем возможным входам.<br>Некоторые особенности среднего случая:<br>- Учитывает ожидаемое поведение алгоритма на типичных данных.<br>- еже, чем худший случай, вычисляется средняя сложность. <br>Пример: для задачи поиска элемента в массиве при полном переборе слева направо: наилучшая оценка — O(1), если искомый элемент окажется в начале списка, худшая оценка — O(n), если искомый элемент окажется в конце списка, средняя оценка — O(n²) = O(n). <br>**Важно**: для многих алгоритмов средняя и наихудшая временная сложность могут существенно различаться (например, O(n log n) против O(n²)). Это требует дополнительного анализа для практических сценариев. |

### Как можно оценить пространственную сложность алгоритма?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| ----------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Пространственную сложность алгоритма оценивают, анализируя объём памяти, необходимый для его выполнения, в зависимости от размера входных данных**. Пространственная сложность учитывает не только явное использование памяти (например, переменные, массивы), но и скрытые затраты, такие как стек вызовов функций или кэширование данных.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| Методы      | Для анализа пространственной сложности алгоритма можно использовать следующие подходы:<br>- **Аналитический метод** — вычисление объёма памяти, необходимого для каждой операции алгоритма, и суммирование этих значений.<br>- **Экспериментальный метод** — измерение фактического использования памяти при выполнении алгоритма на различных входных данных.<br>- **Моделирование** — использование специализированных инструментов для моделирования поведения алгоритма в условиях ограниченной памяти.<br>Пространственная сложность алгоритма может быть разделена на две основные составляющие:<br>1. **Фиксированная часть** — объём памяти, который не зависит от размера входных данных (например, переменные для управления циклами, константы).<br>2. **Переменная часть** — объём памяти, который изменяется в зависимости от размера входных данных (это могут быть динамические структуры данных, такие как массивы, списки или деревья). |
| Нотации     | Для описания пространственной сложности алгоритмов используют **асимптотическую нотацию** — **Big O notation**. Она описывает верхнюю границу роста объёма памяти, используемого алгоритмом, в худшем случае. Нотация позволяет сравнивать алгоритмы по их потреблению памяти, игнорируя константы и незначительные члены, которые могут быть важны для небольших размеров входных данных.<br>**Важно**: пространственная сложность описывает только асимптотическое поведение алгоритма, то есть его поведение при больших размерах входных данных. Фактическое потребление памяти алгоритмом может зависеть от многих факторов, таких как аппаратное обеспечение, операционная система и язык программирования.                                                                                                                                                                                                                                        |

### Каковы основные различия между сложностью О(n log n) и О(n^2)? Приведите примеры алгоритмов.

| Под темы                               | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| -------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Сравнение сложности O(n log n) и O(n²) | **Временная сложность** — это функция, показывающая зависимость времени выполнения алгоритма от размера входных данных.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| Основные различия                      | Рост функции:<br>- O(n log n): растёт медленнее, чем квадратичная функция.<br>- O(n²): растёт быстрее, чем логарифмическая функция.<br>Эффективность:<br>- Алгоритмы с O(n log n) более эффективны при больших значениях n.<br>- Алгоритмы O(n²) менее эффективны, особенно при больших входных данных.<br>Практическое применение:<br>- Сложность O(n log n) характерна для эффективных алгоритмов сортировки (например, быстрая сортировка).<br>- Сложность O(n²) типична для простых алгоритмов (например, пузырьковая сортировка).                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| Примеры алгоритмов                     | O(n log n):<br>- **Быстрая сортировка** (QuickSort): делит массив на части и рекурсивно сортирует их.<br>- **Сортировка слиянием** (MergeSort): разделяет массив на части, сортирует их и сливает обратно.<br>- **Двоичный поиск**: находит элемент в отсортированном массиве за O(log n).<br>O(n²):<br>- **Пузырьковая сортировка**: сравнивает соседние элементы и меняет их местами, если они стоят в неправильном порядке.<br>- **Сортировка выбором**: находит минимальный элемент в неотсортированной части массива.<br>- **Вставка**: вставляет каждый элемент на его правильное место в отсортированной части.<br>Пример: если у нас есть массив из 1000 элементов, то:<br>- Быстрая сортировка (O(n log n)) потребует примерно 13 000 операций.<br>- Пузырьковая сортировка (O(n²)) потребует около 1 000 000 операций, что значительно медленнее.<br>Таким образом, алгоритмы с сложностью O(n log n) значительно эффективнее при работе с большими наборами данных по сравнению с алгоритмами O(n²). |

### Как влияет структура данных на временную сложность алгоритма? Приведите пример.

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Структура данных** напрямую влияет на временную сложность алгоритма, поскольку определяет, как данные хранятся и обрабатываются в памяти компьютера. Разные структуры данных обеспечивают различные операции с разной эффективностью<br>Основные факторы влияния:<br>- **Доступ к элементам**: некоторые структуры позволяют быстрый доступ (например, массивы), другие требуют обхода (связные списки).<br>- **Вставка и удаление**: в некоторых структурах эти операции выполняются быстро (деревья, хеш-таблицы), в других — медленно (массивы).<br>- **Поиск**: в хеш-таблицах поиск обычно выполняется за константное время, а в массивах — за линейное.<br>- **Сортировка**: разные структуры данных требуют разных алгоритмов сортировки с различной сложностью. |
