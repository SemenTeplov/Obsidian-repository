| Определение    | **Аннотация @Transactional** в Spring обеспечивает **декларативное управление транзакциями**. Её можно пометить метод или класс сервиса, чтобы выполнить его логику внутри транзакции. Разработчику не нужно вручную открывать, коммитить или откатывать транзакцию — за него это сделает Spring.<br>Spring использует концепцию **TransactionManager** — центрального компонента, который координирует начало, завершение и откат транзакций. Некоторые реализации:<br>- **DataSourceTransactionManager** — работает с JDBC;<br>- **JpaTransactionManager** — для JPA/Hibernate;<br>- **JtaTransactionManager** — подходит для распределённых транзакций через JTA (Java Transaction API).<br>В контексте Spring Boot (начиная с версий Spring Boot 3 и Spring Data JPA 3.5+ на базе Spring Framework 6) практически никакой дополнительной настройки для использования @Transactional не требуется — если в проекте есть зависимость spring-boot-starter-data-jpa, то поддержка транзакций включается автоматически. |
| Принцип работы | Spring при старте приложения создаёт специальную прокси-обёртку вокруг каждого бина, в котором обнаружена аннотация @Transactional. Этот прокси перехватывает вызовы методов и выполняет дополнительный код до и после метода. Упрощённо можно представить, что для транзакционного метода выполнение идёт так:<br>1. **До вызова реального метода** — прокси обращается к менеджеру транзакций и открывает новую транзакцию (или присоединяется к существующей).<br>2. **Вызов метода** — бизнес-код выполняется внутри открытой транзакции. Если в процессе выполнения произойдёт необработанное исключение, Spring пометит транзакцию как требующую отката.<br>3. **После возврата из метода** — прокси перехватывает результат или исключение.                                                                                                                                                                                                                                                                     |
| Параметры      | В аннотации @Transactional можно указывать, например:<br>- **Propagation** — определяет, что делать, когда вызов метода происходит в контексте уже существующей транзакции. Spring поддерживает несколько режимов propagation (значения перечисления org.springframework.transaction.annotation.Propagation).<br>- **Isolation** — позволяет объявить уровень изоляции транзакции (опции: DEFAULT, READ_UNCOMMITTED, READ_COMMITTED, REPEATABLE_READ, SERIALIZABLE).<br>- **RollbackFor** — указывает, какие типы исключений должны вызывать откат транзакции. По умолчанию откат происходит только при RuntimeException и Error.<br>- **NoRollbackFor** — позволяет указать, какие исключения не требуют отката.                                                                                                                                                                                                                                                                                                      |
