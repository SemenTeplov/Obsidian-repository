### Какая агрегатная функция используется для вычисления среднего значения столбца?

| Под темы    | Заметка                                                                                                          |
| ----------- | ---------------------------------------------------------------------------------------------------------------- |
| Определение | **AVG()** — агрегатная функция в SQL, которая вычисляет **среднее значение числового столбца**.                  |
|             | Функция суммирует все значения столбца, а затем делит полученное число на количество ненулевых значений столбца. |

### Для чего используется агрегатная функция COUNT?

| Под темы     | Заметка                                                                            |
| ------------ | ---------------------------------------------------------------------------------- |
| Используется | **Получить общее количество записей** в таблице или результирующем наборе запроса. |
|              | **Подсчитать строки с непустыми значениями** в конкретном столбце.                 |
|              | **Подсчитать уникальные значения** в столбце, игнорируя NULL.                      |

### Как рассчитать сумму значений по определенному столбцу?

| Под темы | Заметка                                    |
| -------- | ------------------------------------------ |
| Пример   | `SELECT SUM(имя_столбца) FROM имя_таблицы` |

### Как применить агрегатную функцию к столбцу с условием?

| Под темы | Заметка                                                                                                                                                                              |
| -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| WHERE    | **Агрегатные функции не могут быть использованы непосредственно в предложении WHERE**. WHERE фильтрует строки до группировки, а агрегатные функции применяются внутри каждой группы. |
| HAVING   | **Оператор HAVING** фильтрует уже сгруппированные данные, используя условия на агрегатные функции. HAVING можно применять только после агрегации.                                    |

### Как объединить результаты агрегатных функций с GROUP BY?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                   |
| ----------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Результаты агрегатных функций (COUNT, SUM, AVG, MAX, MIN) можно объединить с оператором GROUP BY** в зависимости от контекста — SQL или Pandas (Python). Это позволяет агрегировать данные внутри сгруппированных строк, что полезно для анализа больших объёмов информации и создания сводных отчётов. |

### Как использовать агрегатную функцию для вычисления общего количества уникальных значений в столбце?

| Под темы    | Заметка                                                                                                                                                       |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Функция COUNT(DISTINCT column)**. Указывает, что для вычислений применяются только уникальные значения столбца.                                             |
| Пример      | Подсчёт количества уникальных клиентов, сделавших покупки: `SELECT COUNT(DISTINCT customer_id) FROM orders`.                                                  |
|             | Подсчёт количества различных товарных категорий по городам: `SELECT city, COUNT(DISTINCT category) as category_diversity FROM store_inventory GROUP BY city`. |
| Особенности | Учитывает только непустые значения (NULL игнорируются).                                                                                                       |
|             | Работает с текстовыми, числовыми, логическими типами данных и датами.                                                                                         |

### Как реализовать оконные функции с использованием агрегатных функций?

| Под темы           | Заметка                                                                                                                                                                                                                                             |
| ------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение        | **В SQL** оконные функции определяются с помощью инструкции OVER(). В скобках указываются параметры окна — как его формировать и упорядочивать.                                                                                                     |
| Примеры            | **PARTITION BY** — разделяет данные на группы для независимых вычислений.                                                                                                                                                                           |
|                    | **ORDER BY** — определяет порядок строк внутри окна.                                                                                                                                                                                                |
|                    | **ROWS/RANGE** — задаёт точные границы окна относительно текущей строки.                                                                                                                                                                            |
| Агрегатные функции | SUM, COUNT, AVG, MAX, MIN                                                                                                                                                                                                                           |
| Особенности        | Оконные функции выполняются после обработки WHERE, GROUP BY и HAVING, но до final ORDER BY и LIMIT. Это означает, что оконную функцию нельзя использовать в условиях фильтрации WHERE, но можно применить её результаты для последующей сортировки. |
|                    | В одном запросе можно использовать несколько оконных функций.                                                                                                                                                                                       |

### Какие стратегии можно использовать для оптимизации запросов с агрегатными функциями на больших объемах данных?

| Под темы                        | Заметка                                                                                                                                                                                                                                                                                                                                                    |
| ------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Индексирование                  | **Создавать индексы для столбцов, которые часто используются в агрегатных функциях** (COUNT, SUM, AVG и др.). Это обеспечивает более быстрый доступ к данным.                                                                                                                                                                                              |
|                                 | **Использовать Bitmap-индексы** для столбцов с низкой кардинальностью (количество уникальных значений относительно общего числа строк) — они ускоряют агрегатные функции без сканирования всей таблицы. Однако Bitmap-индексы не универсальны — не подходят для столбцов с высокой кардинальностью или для таблиц с частыми операциями вставки/обновления. |
| WHERE                           | **Фильтровать данные как можно раньше** — применять условия отбора в предложениях WHERE как можно раньше. Чем меньше данных будет обработано на последующих этапах (соединения, группировки, сортировки), тем быстрее выполнится запрос.                                                                                                                   |
|                                 | **Использовать индексы в условиях отбора** — запросы с условиями по индексированным полям выполняются быстрее, чем по неиндексированным.                                                                                                                                                                                                                   |
|                                 | **Оптимизировать диапазоны и сравнения** — предпочтительнее использовать точные сравнения (=) и диапазоны (МЕЖДУ, БОЛЬШЕ ИЛИ РАВНО, МЕНЬШЕ ИЛИ РАВНО) по индексированным полям.                                                                                                                                                                            |
| Порядок операций                | **Размещать вычисления с агрегатными функциями после фильтрации** — например, применять условия отбора (WHERE) до предложения GROUP BY, чтобы уменьшить количество записей, которые нужно будет группировать и агрегировать.                                                                                                                               |
|                                 | **Использовать предложение HAVING для фильтрации по результатам агрегации** — если нужно отфильтровать результаты после агрегации (например, выбрать только те номенклатурные позиции, у которых общая сумма продаж превышает определённое значение), условия в HAVING применяются к агрегированным данным, а не к исходным записям.                       |
| Материализованные представления | **Хранить результаты агрегированных данных** — например, суммы продаж по регионам или времени. Это позволяет избегать многократного пересчёта сложных агрегатных функций при каждом запросе.                                                                                                                                                               |
|                                 | **Использовать инкрементные материализованные представления** — они автоматически обновляются в реальном времени, когда исходные таблицы получают новые данные. Поскольку результаты вычисляются инкрементно во время вставки, запросы выполняются над значительно меньшими подмножествами данных.                                                         |

### Как использовать агрегатные функции в сочетании с подзапросами для решения сложных задач анализа данных?

| Под темы           | Заметка                                                                                                                                                                             |
| ------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение        | Агрегатные функции и подзапросы в SQL можно использовать для решения сложных задач анализа данных, комбинируя эти инструменты. Это позволяет:                                       |
|                    | **Обобщать данные** с помощью агрегатных функций, которые обрабатывают набор значений и возвращают одно значение.                                                                   |
|                    | **Выполнять сложные операции** с помощью подзапросов, которые вложены внутрь другого запроса.                                                                                       |
| Агрегатные функции | **Агрегатные функции** (COUNT, SUM, AVG, MAX, MIN) позволяют выполнять вычисления над группами строк, возвращая единственное значение.                                              |
| Подзапросы         | **Подзапрос** (вложенный запрос) — это SELECT-запрос, помещённый внутри другого SQL-оператора. Внешний запрос использует результат выполнения внутреннего запроса для своей работы. |
