### Что такое JOIN в контексте SQL?

| Под темы    | Заметка                                                                                                         |
| ----------- | --------------------------------------------------------------------------------------------------------------- |
| Определение | **JOIN в контексте SQL** — **оператор, который объединяет данные из разных таблиц** по общему полю или условию. |
| Позволяет   | собрать целостную картину данных из нескольких таблиц                                                           |
|             | посчитать метрики из разных источников                                                                          |
|             | строить витрины и отчёты без денормализации исходных таблиц                                                     |
|             | находить аномалии — заказы без клиентов, клиентов без активностей                                               |
|             |                                                                                                                 |

### Какова основная разница между INNER JOIN и OUTER JOIN?

| Под темы   | Заметка                                                                                                                                       |
| ---------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| INNER JOIN | возвращает такие строки, по которым фигурируют совпадения в обеих связываемых таблицах.                                                       |
| OUTER JOIN | позволит получить все строки из одной таблицы и соответствующие им строки из второй, или даже все строки из обеих таблиц, если такие имеются. |

### Для чего используется LEFT JOIN?

| Под темы    | Заметка                                                                                                                                             |
| ----------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | Оператор LEFT JOIN (или LEFT OUTER JOIN) в SQL используется для объединения данных из разных таблиц, даже когда соответствующие записи отсутствуют. |

### Как можно объединить таблицы без использования JOIN, используя другие SQL конструкции?

| Под темы  | Заметка                                                                                                                                                                                                                                                                                                      |
| --------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| WHERE     | **Указывает условие соединения**. Например, если нужно объединить таблицы Orders и Customers, но каждый заказ из Orders связан с конкретным покупателем из Customers, можно использовать выражение WHERE и фильтровать строки при условии, что поле CustomerId из Orders соответствует полю Id из Customers. |
| UNION     | **Объединяет результаты двух запросов** в один результирующий набор. По умолчанию UNION удаляет повторяющиеся строки, если указан параметр ALL — результат содержит все строки, в том числе повторяющиеся.                                                                                                   |
| INTERSECT | **Возвращает только строки, которые есть в обоих результирующих наборах**. Это полезно, например, когда нужно найти строки, которые есть и в первой таблице, и во второй.                                                                                                                                    |
| EXCEPT    | **Выводит только те строки из первого набора строк, которых нет во втором наборе**. Это полезно, когда необходимо сравнить две таблицы и вывести только те строки первой таблицы, которых нет в другой таблице.                                                                                              |

### Объясните, как работает RIGHT JOIN и в каких случаях его лучше использовать?

| Под темы     | Заметка                                                                                                                                                                                                                                            |
| ------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Объяснение   | **RIGHT JOIN** — это операция объединения, которая возвращает **все строки из правой таблицы** и только соответствующие строки из левой таблицы. Если в левой таблице нет соответствующей строки, то возвращается NULL для значений левой таблицы. |
| Используется | **Поиск записей без соответствия**. Когда необходимо найти все элементы правой таблицы, которые не имеют соответствия в левой.                                                                                                                     |
|              | **Аудит целостности данных**. Для выявления несоответствий между связанными таблицами.                                                                                                                                                             |
|              | **Отчёты по полным категориям**. Когда требуется отобразить все категории, даже если в некоторых нет элементов.                                                                                                                                    |
|              | **Проверка полноты покрытия**. Например, чтобы убедиться, что все клиенты получили необходимые услуги.                                                                                                                                             |

### Какие типы OUTER JOIN существуют в SQL?

| Под темы         | Заметка                                                                                                                                                                                |
| ---------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| LEFT OUTER JOIN  | **Возвращает все строки из левой таблицы** и совпадающие строки из правой таблицы. Если совпадений нет, для столбцов правой таблицы используются значения NULL.                        |
| RIGHT OUTER JOIN | **Возвращает все строки из правой таблицы** и совпадающие строки из левой таблицы. Если совпадения отсутствуют, вместо значений левой таблицы возвращается NULL.                       |
| FULL OUTER JOIN  | **Возвращает все строки**, если есть совпадение либо в левой, либо в правой таблице. Если совпадения нет, то в столбцах таблицы, в которой нет совпадения, используются значения NULL. |

### Как реализовать FULL OUTER JOIN, если этот тип JOIN не поддерживается напрямую?

| Под темы | Заметка                                                                               |
| -------- | ------------------------------------------------------------------------------------- |
| Способы  | **LEFT JOIN** возвращает все строки из левой таблицы и совпадающие строки из правой.  |
|          | **RIGHT JOIN** возвращает все строки из правой таблицы и совпадающие строки из левой. |
|          | **UNION** объединяет результаты двух запросов SELECT.                                 |

### Что такое NATURAL JOIN в SQL и как он отличается от INNER JOIN?

| Под темы    | Заметка                                                                                                                                  |
| ----------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **NATURAL JOIN в SQL** — это тип соединения, который автоматически сопоставляет столбцы с одинаковыми именами в двух или более таблицах. |
| Особенности | Столбцы, по которым соединяются таблицы, должны иметь одинаковое имя и тип данных в обеих таблицах.                                      |
|             | В результирующей таблице столбцы с одинаковыми именами появляются только один раз.                                                       |
|             | Не используются предложения ON или USING для указания столбцов соединения — SQL автоматически обнаруживает общие столбцы.                |
| Отличия     | NATURAL JOIN не требует явно указывать столбцы для соединения, а INNER JOIN требует явно указать столбцы в предложении ON или USING.     |

### Как использовать JOIN для оптимизации запросов с подзапросами?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                |
| ----------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **JOIN в SQL можно использовать для оптимизации запросов с подзапросами**, чтобы избежать накладных расходов на вложенные запросы. Подзапросы (SUBQUERY) часто менее производительны, чем JOIN, поскольку выполняются для каждой строки основной выборки. JOIN позволяет СУБД лучше оптимизировать выполнение запроса, особенно при работе с большими наборами данных и многочисленными таблицами, особенно когда применяются индексы. |
| Правила     | **Выбирать правильный тип JOIN**. Например, INNER JOIN обычно быстрее, чем OUTER JOIN, поскольку возвращает только совпадающие строки из обеих таблиц. По возможности избегать CROSS JOIN, поскольку он создаёт большие декартовы продукты, которые могут быть ресурсоёмкими.                                                                                                                                                          |
|             | **Уменьшать объём данных, обрабатываемых JOIN**, с помощью дополнительных условий фильтрации в предложениях WHERE или JOIN ON запроса. Это проталкивает условия фильтрации как можно глубже в план запроса, обычно до выполнения JOIN.                                                                                                                                                                                                 |
|             | **Создавать индексы** на столбцы, используемые в условиях JOIN, — это ускоряет операции соединения таблиц.                                                                                                                                                                                                                                                                                                                             |
|             | **Ограничивать объём данных в подзапросах** — использовать WHERE в подзапросе для фильтрации ненужных строк на раннем этапе.                                                                                                                                                                                                                                                                                                           |
