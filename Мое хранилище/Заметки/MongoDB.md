### Что такое MongoDB и какие особенности отличают её от реляционных СУБД?

| Под темы    | Заметка                                                                                                                                                                                                                                                                       |
| ----------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **MongoDB** — **документо-ориентированная нереляционная система управления базами данных (СУБД)**, классифицированная как база данных NoSQL. В отличие от традиционных реляционных СУБД, MongoDB использует **гибкую структуру данных** вместо таблиц с фиксированной схемой. |
| Особенности | **Гибкость схемы данных** — в отличие от реляционных баз, где изменение схемы требует модификации всех существующих записей, MongoDB позволяет динамически расширять структуру документов, добавляя новые атрибуты без реорганизации всего хранилища.                         |
|             | **Горизонтальное масштабирование** — данные могут распределяться по нескольким серверам с использованием механизма шардинга, что повышает отказоустойчивость и скорость обработки запросов.                                                                                   |
|             | **Отсутствие сложных соединений** — в MongoDB не используются конструкции вроде JOIN, которые в реляционных базах соединяют данные из разных таблиц.                                                                                                                          |
|             | **Поддержка специальных запросов** — данные можно искать по полям и диапазонам, поддерживаются регулярные выражения, можно возвращать пользовательские функции.                                                                                                               |

### В чем преимущества хранения данных в формате BSON по сравнению с JSON?

| Под темы | Заметка                                                                                                                                                                                                                                                                                                                             |
| -------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Ответ    | **Преимущества хранения данных в формате BSON (Binary JSON) по сравнению с JSON** заключаются в **эффективности хранения и обработки данных**, особенно для больших и сложных наборов данных. BSON расширяет JSON, добавляя поддержку дополнительных типов данных и обеспечивая более эффективное кодирование для сложных структур. |
| JSON     | **JSON (JavaScript Object Notation)** — текстовый формат, который легко читается и понимается человеком и машиной.                                                                                                                                                                                                                  |
| BSON     | **BSON (Binary JSON)** — двоичный формат, который занимает меньше места, чем JSON, особенно при обработке больших объёмов данных.                                                                                                                                                                                                   |

### Какие типы данных поддерживает MongoDB?

| Под темы | Заметка                                                                                                                                                                                                             |
| -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Ответ    | **MongoDB поддерживает несколько типов данных**, которые используются для хранения разных типов данных. Некоторые из них: строки (String), целые числа (Integer), логические значения (Boolean) и массивы (Arrays). |

### Что такое коллекции и документы в MongoDB?

| Под темы  | Заметка                                                                                                                                                                                                                               |
| --------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Ответ     | **В MongoDB данные хранятся в виде коллекций и документов** — иерархических структур, содержащих пары «ключ — значение» (поля). Если проводить аналогии с реляционной базой, коллекции соответствуют таблицам, а документы — строкам. |
| Коллекции | **Коллекция** — это группа документов, эквивалент таблицы в реляционной базе данных.                                                                                                                                                  |
| Документы | **Документ** — это основная единица данных, аналогичная строке в реляционной базе данных.                                                                                                                                             |

### Как подключиться к MongoDB из Java-приложения с помощью MongoDB Java Driver или Spring Data MongoDB?

| Под темы            | Заметка                                                                                                                                                                                              |
| ------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| MongoDB Java Driver | **Подходит для простых задач** — подключения к базе данных MongoDB, выполнения запросов и обработки данных.                                                                                          |
| Особенности         | **Указание параметров соединения** — хоста, порта и, если требуется, учётных данных для аутентификации.                                                                                              |
|                     | **Использование класса MongoClient** для установления соединения.                                                                                                                                    |
|                     | **Поддержка операций** — вставки документов, запроса данных, обновления документов, удаления документов, создания индексов и выполнения агрегаций.                                                   |
| Spring Data MongoDB | **Подходит для проектов Spring Boot** — автоматически устанавливает клиент MongoDB, который подключается к серверу при запуске приложения.                                                           |
| Особенности         | **Настройка соединения** — можно указать строку подключения (URI) или отдельные свойства spring.data.mongodb. Например, `spring.data.mongodb.uri=mongodb: //user123:secret@localhost:27017/sandbox`. |
|                     | **Указание свойств приложения** — можно указать хост, порт, имя базы данных, имя пользователя и пароль для аутентификации.                                                                           |
|                     | **Автоконфигурация** — Spring Boot автоматически создаёт клиент MongoDB, если не заданы свойства.                                                                                                    |

### Как выполняются базовые CRUD-операции с помощью MongoTemplate и/или репозиториев Spring Data?

| Под темы        | Заметка                                                                                                                              |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| MongoTemplate   | **MongoTemplate** — класс, который обеспечивает высокий уровень абстракции для выполнения общих операций MongoDB, включая CRUD.      |
| Методы          | **Вставка новых документов** — метод insert(). Если документ с указанным идентификатором уже присутствует, операция вызывает ошибку. |
|                 | **Обновление существующих документов** — методы updateFirst() или updateMulti().                                                     |
|                 | **Выполнение запросов MongoDB** — методы find(), findOne() или aggregate().                                                          |
| MongoRepository | **Интерфейс MongoRepository** предоставляет набор методов для выполнения CRUD-операций в коллекциях MongoDB.                         |
| Методы          | save() — сохранение объекта в коллекцию.                                                                                             |
|                 | findById() — извлечение сохранённого документа по идентификатору.                                                                    |
|                 | findAll() — возврат всех документов коллекции.                                                                                       |
|                 | deleteById() — удаление документа по идентификатору.                                                                                 |

### Что такое индексы в MongoDB и как они влияют на производительность?

| Под темы | Заметка                                                                                                                                                                         |
| -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Ответ    | **Индексы в MongoDB** — это структуры данных, которые ускоряют выполнение запросов, позволяя базе данных быстро находить нужные документы, минуя полное сканирование коллекции. |
| Влияние  | **Ускорение запросов** — MongoDB может напрямую переходить к соответствующим данным, что ускоряет выполнение запросов.                                                          |
|          | **Оптимизация сортировки** — индексы предоставляют упорядоченную структуру данных, что позволяет MongoDB более эффективно возвращать отсортированные результаты.                |
|          | **Эффективные запросы по диапазону** — индекс позволяет MongoDB пропускать ненужные части данных, что ускоряет процесс.                                                         |

### Как устроены запросы с фильтрацией и агрегацией в MongoDB?

| Под темы | Заметка                                                                                                                                                                                                                                                                            |
| -------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Ответ    | **Запросы с фильтрацией и агрегацией в MongoDB устроены по принципу конвейера (pipeline)**. Каждый этап конвейера получает документы от предыдущего этапа, обрабатывает их и передаёт результат дальше. Это позволяет создавать сложные запросы из простых операций.               |
|          | Основные компоненты агрегационного framework: \$match — фильтрация документов, \$group — группировка данных, \$sort — сортировка результатов, \$project — выбор и трансформация полей, \$limit и \$skip — пагинация, \$lookup — JOIN-операции, \$unwind — разворачивание массивов. |

### Как работает репликация в MongoDB и зачем она нужна?

| Под темы       | Заметка                                                                                                                                                                                                |
| -------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Определение    | **Репликация в MongoDB** — это процесс синхронизации данных на нескольких серверах, который обеспечивает **избыточность данных, отказоустойчивость и высокую доступность**.                            |
| Принцип работы | Репликация в MongoDB реализуется с помощью **наборов реплик** — группы серверов, которые поддерживают один и тот же набор данных.                                                                      |
| Процесс        | **Операции записи на основном узле** — когда пользователь отправляет операцию записи (вставку, обновление, удаление) на основной узел, он обрабатывает её и записывает в свой oplog (журнал операций). |
|                | **Репликация oplog на вторичные узлы** — вторичные узлы регулярно проверяют oplog основного узла и применяют те же операции к своим данным в том же порядке, что и на основном узле.                   |
|                | **Автоматическое переключение при отказе** — если основной узел выходит из строя, один из вторичных узлов автоматически становится новым основным.                                                     |

### Что такое шардинг и в каких ситуациях он применяется?

| Под темы    | Заметка                                                                                                                                                                                                                                                                       |
| ----------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Шардинг (шардирование) — метод распределения данных в базах данных**, при котором общая база разбивается на более мелкие независимые фрагменты — шарды. Каждый шард хранит подмножество общей информации и зачастую располагается на отдельном сервере или группе серверов. |
|             | Шардинг применяется в ситуациях, когда один сервер не может обеспечить достаточную производительность и устойчивость в условиях растущего объёма операций и увеличения количества данных.                                                                                     |

### Как реализовать транзакции в MongoDB и какие ограничения у них есть?

| Под темы    | Заметка                                                                                                                                                                                                                                                                                                                 |
| ----------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Транзакции в MongoDB** — это логические единицы работы, которые объединяют одну или несколько операций (вставки, обновления, удаления) и выполняют их как одну неразделимую единицу. Это обеспечивает целостность данных: либо все операции в транзакции завершены, либо ни одна из них не применяется к базе данных. |
| Реализация  | **Запустить сеанс** — каждый раз, когда нужно выполнить транзакцию, необходимо запустить новую сессию.                                                                                                                                                                                                                  |
|             | **Инициировать транзакцию** — после запуска сеанса можно начать транзакцию внутри него.                                                                                                                                                                                                                                 |
|             | **Выполнить операции** — выполнить желаемые операции (вставки, обновления, удаления) внутри транзакции.                                                                                                                                                                                                                 |
|             | **Завершить или откатить транзакцию.**                                                                                                                                                                                                                                                                                  |
| Ограничения | **Ограниченный срок действия** — по умолчанию транзакция живёт 60 секунд, потом откатывается.                                                                                                                                                                                                                           |
|             | **Конфликты записи** — одновременная запись в один и тот же документ может вызвать конфликты.                                                                                                                                                                                                                           |
|             | **Ограничения механизма хранения** — транзакции доступны только в наборах реплик и разделённых кластерах, на standalone транзакции не поддерживаются.                                                                                                                                                                   |
|             | **Ограничения на размер документов** — по умолчанию лимит — 16 МБ для всех операций в транзакции.                                                                                                                                                                                                                       |
|             | **Ограничения на операции** — не все операции MongoDB можно выполнять внутри транзакции, например, операции DDL (Data Definition Language), административные команды.                                                                                                                                                   |

### Как MongoDB обрабатывает конкурентные запросы и обеспечивает согласованность данных?

| Под темы                           | Заметка                                                                                                                                                                                                                                                                                                                                  |
| ---------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Обработка конкурентных запросов    | **Репликация**. Данные поддерживаются на нескольких серверах, что обеспечивает избыточность и высокую доступность. В наборе реплик один узел обозначается как основной, а остальные — как второстепенные. Основной узел принимает все операции записи, и эти операции затем реплицируются на второстепенные узлы.                        |
|                                    | **Шардинг (sharding)**. Данные распределяются по нескольким машинам (шардам) для поддержки развёртываний с большими наборами данных и операциями с высокой пропускной способностью. Это позволяет параллельно обрабатывать больше информации, преодолевая ограничения одного сервера.                                                    |
| Обеспечение согласованности данных | **Настройка согласованности**. По умолчанию MongoDB использует **итоговую согласованность**: первичный узел подтверждает запись, а затем асинхронно реплицирует её на вторичные узлы. Хотя первичный узел обеспечивает долговечность записи, он не ждёт, пока все вторичные узлы синхронизируются, прежде чем подтвердить успех клиенту. |
|                                    | **Механизмы Read Concerns (требования к чтению) и Write Concerns (требования к записи)**. Они позволяют разработчикам настраивать уровень согласованности на уровне каждой операции.                                                                                                                                                     |
