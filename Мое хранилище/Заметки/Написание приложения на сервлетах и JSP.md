### Что такое сервлет в контексте Java EE?

| Под темы    | Заметка                                                                                                                                                                                                                        |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Определение | Servlet в контексте Java EE (Enterprise Edition) — Java-класс, который используется для создания динамических веб-приложений. Он обрабатывает запросы от клиентов и генерирует ответы в формате HTML, XML или других форматах. |
|             | Сервлеты — основные строительные блоки веб-приложений на платформе Java.                                                                                                                                                       |

### Как настроить Tomcat для запуска веб-приложения?

| Под темы                       | Заметка                                                                                                                                                                                                                                                   |
| ------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Установка                      | **Скачать дистрибутив Tomcat** с официального сайта. Можно использовать пакетный менеджер или скачать версию в виде архива.                                                                                                                               |
|                                | **Распаковать архив** в нужный каталог.                                                                                                                                                                                                                   |
|                                | **Для корректного функционирования** на сервере должен быть предварительно установлен пакет JDK (Java Development Kit).                                                                                                                                   |
|                                | **По умолчанию** Tomcat запускается на порту 8080, но можно изменить порт в файле server.xml.                                                                                                                                                             |
| Настройка переменных окружения | Установить переменную CATALINA_HOME — она указывает на каталог установки Tomcat.                                                                                                                                                                          |
|                                | Настроить переменную JAVA_HOME — она указывает на местоположение установки JDK.                                                                                                                                                                           |
|                                | Создать файл setenv.sh (или setenv.bat) — в нём можно указать значения переменных, связанных с Tomcat. Этот файл автоматически вызывается при запуске Tomcat.                                                                                             |
| Развертывание                  | **Скопировать файл веб-приложения (WAR)** в каталог webapps. Если Tomcat был запущен, он сам развернёт приложение из WAR-файла через несколько секунд (появится одноимённый каталог).                                                                     |
|                                | **Использовать менеджер приложений (Tomcat Web Application Manager)**. В нём можно загрузить и развернуть WAR-файлы через веб-интерфейс.                                                                                                                  |
|                                | **Настроить Tomcat на автоматическое развёртывание** — приложения можно размещать в каталоге webapps.                                                                                                                                                     |
| Отладка                        | **Использовать встроенную отладку** — Tomcat предоставляет возможности для отслеживания потока приложений и выявления проблем. Можно добавлять логи в ключевые точки кода, чтобы контролировать выполнение приложения и определять местоположение ошибок. |
|                                | **Устанавливать точки останова** в коде — это позволяет останавливать выполнение приложения в определённых точках и проверять состояние переменных, объектов и потоков.                                                                                   |
|                                | **Настроить приложение на удалённую отладку** — это позволяет подключать IDE к удалённому серверу и отлаживать код в реальном времени.                                                                                                                    |

### Каковы основные отличия между сервлетами и JSP?

| Под темы | Заметка                                                                                                                                                                                                 |
| -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Servlet  | **Сервлеты** — это Java-классы, которые расширяют функциональность веб-сервера. Они обрабатывают HTTP-запросы, взаимодействуют с базами данных, вызывают другие веб-сервисы и формируют ответы клиенту. |
| JSP      | **JSP** — это технология для создания динамических веб-страниц на базе Java. По сути, это HTML с вкраплениями Java-кода, который выполняется на сервере.                                                |

### Как передать данные из сервлета в JSP?

| Под темы                 | Заметка                                                                                                                                                                                                                                                                              |
| ------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Через HttpServletRequest | **Данные сохраняются в атрибутах запроса**. У объекта HttpServletRequest, который передаётся в сервлет, вызывается метод setAttribute(). Этот метод устанавливает атрибут, который можно получить в JSP.                                                                             |
| Через HttpSession        | **Данные сохраняются в атрибутах сессии**. У объекта HttpServletRequest есть метод getSession() — он возвращает объект HttpSession.                                                                                                                                                  |
| Через ServletContext     | **Данные сохраняются в атрибутах контекста приложения**. Объект ServletContext можно получить в сервлете с помощью метода getServletContext(). У этого объекта есть метод setAttribute() — он устанавливает атрибут, который доступен всем сервлетам (включая JSP) в этом контексте. |

### Как настроить контекстное меню веб-приложения в Tomcat?

| Под темы      | Заметка                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| ------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| В server.xml  | **Не рекомендуется** размещать элементы Context непосредственно в файле server.xml. Это может привести к тому, что конфигурация контекста будет более инвазивной, так как основной файл server.xml не может быть перезагружен без перезапуска Tomcat.                                                                                                                                                                                                                                                                                 |
|               | **Однако есть исключение**: если оба атрибута autoDeploy и deployOnStartup ложны, можно вручную добавить элемент Context внутри элемента Host в файле conf/server.xml. В этом случае атрибут path определяет контекстный путь приложения. **Однако этот атрибут можно определить только если** каталог развёртывания WAR или развёрнутого развёртывания не находится в каталоге webapps или если атрибуты autoDeploy и deployOnStartup в элементе Host ложны. Если не следовать этому правилу, может произойти двойное развёртывание. |
| В context.xml | **В файле $CATALINA_HOME/conf/context.xml** — информация элемента Context будет загружаться всеми веб-приложениями.                                                                                                                                                                                                                                                                                                                                                                                                                   |
|               | **В файле $CATALINA_HOME/conf/[enginename]/ [hostname]/context.xml.default** — информация элемента Context будет загружаться всеми веб-приложениями этого хоста.                                                                                                                                                                                                                                                                                                                                                                      |
|               | **В отдельном файле** по пути /META-INF/context.xml внутри файлов приложения. Если веб-приложение упаковано в WAR, то /META-INF/context.xml будет скопирован в каталог $CATALINA_HOME/conf/[enginename]/[hostname]/ и переименован, чтобы соответствовать контекстному пути приложения.                                                                                                                                                                                                                                               |

### Как использовать фильтры в сервлетах для управления доступом?

| Под темы | Заметка                                                                                                                                                                                                                                                                                                                         |
| -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Ответ    | **Фильтры в сервлетах можно использовать для управления доступом** (например, проверки авторизации, ограничения доступа к определённым страницам). Фильтры — это специальные компоненты, которые внедряются в цепочку обработки HTTP-запросов и ответов, и могут перехватывать, изменять или даже блокировать запросы и ответы. |
|          | Фильтры не генерируют контент сами по себе (это задача сервлетов), но могут влиять на то, как этот контент обрабатывается.                                                                                                                                                                                                      |

### Как использовать паттерн MVC в разработке веб-приложения на сервлетах и JSP?

| Под темы           | Заметка                                                                                                                                                                                                                                        |
| ------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Ответ              | **Паттерн MVC (Model-View-Controller) можно использовать в разработке веб-приложения на сервлетах и JSP**. В этом подходе сервлеты выступают контроллерами, JSP — представлениями, а Java-классы — моделями                                    |
| Процесс            | Клиент отправляет HTTP-запрос на сервер.                                                                                                                                                                                                       |
|                    | Сервлет перехватывает запрос и определяет действие.                                                                                                                                                                                            |
|                    | Сервлет взаимодействует с бизнес-логикой (моделью).                                                                                                                                                                                            |
|                    | Данные сохраняются в объектах запроса, сессии или контекста.                                                                                                                                                                                   |
|                    | Сервлет перенаправляет запрос на соответствующую JSP-страницу.                                                                                                                                                                                 |
|                    | JSP-страница отображает данные и формирует HTML-ответ.                                                                                                                                                                                         |
| Сервлеты (Servlet) | **Сервлет** обрабатывает запрос, взаимодействует с бизнес-логикой и готовит данные.                                                                                                                                                            |
| JSP                | **JSP** — представление, которое отображает данные, полученные от сервлета.                                                                                                                                                                    |
| Конфигурация       | **Компоненты веб-приложения настраиваются** в дескрипторе развёртывания web.xml или с помощью аннотаций (в более новых версиях Java EE). Этот файл определяет структуру приложения, маппинги URL, инициализационные параметры и многое другое. |
| Тестирование       | Для тестирования веб-приложения, реализованного по паттерну MVC с использованием сервлетов и JSP.                                                                                                                                              |

### Какие существуют способы оптимизации производительности веб-приложения на сервлетах и JSP?

| Под темы    | Заметка                                                                                                                                                                                                                                                                      |
| ----------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Оптимизация производительности веб-приложения на сервлетах и JSP** включает методы, направленные на ускорение загрузки данных, снижение нагрузки на сервер и улучшение пользовательского опыта.                                                                            |
| Сервлеты    | **Минимизировать синхронизацию Java** в сервлетах, не использовать однопоточную модель.                                                                                                                                                                                      |
|             | **Использовать метод init() сервлета** для выполнения дорогостоящей одноразовой инициализации. Например, кэшировать статические данные или выполнять дорогостоящие операции только во время инициализации.                                                                   |
|             | **Избегать общих изменённых переменных класса** в многопоточной модели сервлета, поскольку они создают необходимость в синхронизации.                                                                                                                                        |
|             | **Использовать пул потоков** — по умолчанию сервлет создаёт новый поток для каждого запроса, что снижает производительность. Можно настроить пул потоков на основе ожидаемого количества одновременных пользователей.                                                        |
|             | **Экономно создавать сеансы** — если сеанс не требуется, не создавать его. Использовать javax.servlet.http.HttpSession.invalidate() для освобождения сеансов, когда они больше не нужны.                                                                                     |
| JSP         | **Минимизировать HTTP-запросы** — например, объединять и минифицировать файлы CSS и JavaScript.                                                                                                                                                                              |
|             | **Использовать директивы JSP include** для модульности кода — разбивать страницу на небольшие компоненты, которые можно кэшировать и использовать на нескольких страницах.                                                                                                   |
|             | **Кэшировать результаты** — если тратится много времени на повторный запуск того же сервлета или JSP, можно кэшировать его результаты и возвращать результаты из кэша при следующем запуске. Это полезно для общих запросов, выполняемых всеми посетителями сайта.           |
|             | **Использовать предварительно скомпилированные JSP** — если JSP не меняются, можно предварительно скомпилировать все JSP, присутствующие в веб-приложении. При обращении к JSP он не компилируется, а вместо этого используется его предварительно скомпилированный сервлет. |
|             | **Минимизировать использование скриптлетов** — вместо встраивания Java-кода непосредственно в страницы JSP с помощью скриптлетов можно использовать теги JSTL (JavaServer Pages Standard Tag Library) и пользовательские теги для лучшего разделения задач.                  |
