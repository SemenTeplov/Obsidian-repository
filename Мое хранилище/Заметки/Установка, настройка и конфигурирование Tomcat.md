### Для чего используется сервер приложений Tomcat?

| Под темы    | Заметка                                                                                                                                                                                                                       |
| ----------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Сервер приложений Apache Tomcat используется для запуска и управления веб-приложениями на языке Java**. Это контейнер сервлетов, который обеспечивает среду выполнения для Java-приложений, работающих через HTTP-протокол. |
| Приминение  | В корпоративных системах, которые работают на Java;                                                                                                                                                                           |
|             | На веб-сайтах и онлайн-сервисах с динамическими данными;                                                                                                                                                                      |
|             | В банковских системах, онлайн-торговле, обучающих платформах.                                                                                                                                                                 |

### Как установить Tomcat на операционную систему Windows?

| Под темы                         | Заметка                                                                                                                                                                                        |
| -------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Через установщик Windows Service | **Скачать установщик** с официального сайта Apache Tomcat.                                                                                                                                     |
|                                  | **Запустить установщик**.                                                                                                                                                                      |
|                                  | **Следовать мастере установки**.                                                                                                                                                               |
| Из архива                        | **Скачать архив** (в зависимости от версии Windows — 32-разрядный или 64-разрядный).                                                                                                           |
|                                  | **Распаковать архив** — щёлкнуть по файлу правой кнопкой мыши и выбрать «Распаковать всё...».                                                                                                  |
|                                  | **Выбрать, где распаковать содержимое архива** — для удобства рекомендуется распаковать в корень жёсткого диска. Можно дать каталогу короткое имя, чтобы облегчить конфигурацию сервера позже. |
|                                  | **Перейти в поддиректорий conf** внутри распакованного каталога и найти файл server.xml. **Важно**: перед изменениями в файлах .xml рекомендуется сделать резервную копию.                     |
|                                  | **После установки** можно настроить Tomcat, например, изменить порт в файле server.xml. Затем запустить сервер с помощью командной строки из каталога bin.                                     |

### Как проверить, что Tomcat успешно установлен и запущен на вашем компьютере?

| Под темы  | Заметка                                                                                                                                                                                                             |
| --------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| В Windows | **Проверить, запущен ли сервис Tomcat**. Если Tomcat установлен как сервис, он автоматически запускается при запуске системы.                                                                                       |
|           | Открыть меню «Пуск» и найти «Службы».                                                                                                                                                                               |
|           | В окне «Службы» найти сервис Apache Tomcat. Столбец «Статус» указывает, запущен ли сервис.                                                                                                                          |
|           | Если Tomcat запущен, в браузере можно увидеть стартовую страницу сервера по адресу, например, http://localhost:8080.                                                                                                |
| В Linux   | **Проверить, запущен ли сервис Tomcat**. Для этого можно использовать команду systemctl status tomcat. Если сервис запущен, в выводе будет указано, что он активен (запущен).                                       |
|           | **Проверить, работает ли сервер**. Для этого нужно ввести в браузере адрес http://<YourIPAddress>:8080. Если установка и настройка прошли успешно, в браузере должна появиться стартовая страница Tomcat.           |
|           | **Проверить версию установленного программного обеспечения**. Для этого можно использовать команду java -cp /opt/tomcat/lib/catalina.jar org.apache.catalina.util.ServerInfo. В выводе будет указана версия Tomcat. |

### Как сконфигурировать Tomcat для использования другого порта, отличного от стандартного 8080?

| Под темы      | Заметка                                                                                                                                                                                                                                                                                                                                                      |
| ------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| На Windows    | **Остановить сервер Tomcat** — выполнить скрипт shutdown, расположенный в каталоге bin установки Tomcat.                                                                                                                                                                                                                                                     |
|               | **Изменить файл server.xml** в текстовом редакторе. Найти элемент, который указывает атрибут port — номер порта, на котором Tomcat прослушивает HTTP-запросы. По умолчанию настройка обычно выглядит так: Connector port="8080" protocol="HTTP/1.1" connectionTimeout="20000" redirectPort="8443".                                                           |
|               | **Изменить значение атрибута port** на желаемый номер порта. Например, чтобы изменить порт на 8081, элемент нужно настроить так: Connector port="8081" protocol="HTTP/1.1" connectionTimeout="20000" redirectPort="8443".                                                                                                                                    |
|               | **Сохранить изменения и перезапустить Tomcat**. Можно запустить Tomcat, запустив скрипт запуска, расположенный в каталоге bin установки Tomcat: на Windows — startup.bat.                                                                                                                                                                                    |
|               | **Проверить новую конфигурацию порта** — открыть веб-браузер и посетить http://localhost:new_port_number, заменив new_port_number на номер порта, указанный в server.xml. Если видна страница приветствия Tomcat, изменение порта было успешным.                                                                                                             |
| На Linux/Unix | **В некоторых системах Linux и Unix** порт, отличный от стандартного, может быть привилегированным — он зарезервирован для программ, работающих под пользователем root. Если Tomcat запускается под другим пользователем, может потребоваться настроить переадресацию портов или использовать программу authbind для разрешения доступа к защищённому порту. |

### Как осуществляется деплой WAR-файла на сервер Tomcat?

| Под темы          | Заметка                                                                                                                                                                                                                                                                                                                      |
| ----------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Копирование       | **WAR-файл копируют в каталог веб-приложений Tomcat** — папку webapps внутри установленного каталога Tomcat.                                                                                                                                                                                                                 |
|                   | Скопировать WAR-файл в папку  [TOMCAT_HOME]/webapps.                                                                                                                                                                                                                                                                         |
|                   | Запустить или перезапустить Tomcat. Если сервер настроен на автоматическое развёртывание, он должен развернуть файл автоматически. После успешного развёртывания в папке webapps появится одноимённая папка с содержимым приложения.                                                                                         |
| Через менеджер    | **Apache Tomcat предоставляет веб-интерфейс — Tomcat Manager**, который позволяет управлять приложениями, включая их развёртывание и удаление.                                                                                                                                                                               |
|                   | Открыть http://localhost:8080/manager/html в браузере.                                                                                                                                                                                                                                                                       |
|                   | Авторизоваться, используя имя пользователя и пароль, указанные в файле tomcat-users.xml.                                                                                                                                                                                                                                     |
|                   | В разделе Deploy выбрать WAR-файл или указать путь к нему.                                                                                                                                                                                                                                                                   |
|                   | Нажать кнопку Deploy, и Tomcat развернёт приложение.                                                                                                                                                                                                                                                                         |
| С помощью Maven   | **Можно добавить функции развёртывания в файл POM** и указать URL и учётные данные, необходимые для подключения и развёртывания на Tomcat. После правильной конфигурации файла POM команда Maven deploy приведёт к полной сборке, и упакованное веб-приложение будет развёрнуто в виде WAR-файла на Tomcat.                  |
| С помощью Jenkins | **Для развёртывания WAR-файла с помощью системы непрерывной интеграции Jenkins** можно использовать **плагин Deploy to Container**. После настройки этого плагина любое задание сборки Jenkins может развернуть WAR-файл на Tomcat — для этого в задание добавляется действие «Развернуть WAR/EAR в контейнер» после сборки. |

### Что такое файл server.xml в Tomcat и для чего он используется?

| Под темы    | Заметка                                                                                                                                                        |
| ----------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение | **Файл server.xml в Apache Tomcat** — **основной файл конфигурации**. Он определяет, как работает Tomcat: его порты, сервисы, соединители и виртуальные хосты. |
|             | **Расположение**: по умолчанию server.xml находится в каталоге /conf/ установки Tomcat.                                                                        |
|             | **Цель**: с помощью этого файла можно настроить различные типы компонентов в контейнере Tomcat с помощью тегов.                                                |

### Как настроить Tomcat для работы с SSL/TLS?

| Под темы   | Заметка                                                                                                                                                                                                                                                                             |
| ---------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Ответ      | **Tomcat можно настроить для работы с SSL/TLS** через конфигурацию коннектора в файле server.xml.                                                                                                                                                                                   |
|            | **JSSE** — предоставляется частью Java Runtime (начиная с версии 1.4).                                                                                                                                                                                                              |
|            | **APR** — включена по умолчанию в версии 5.5.28 и основана на OpenSSL.                                                                                                                                                                                                              |
| Инструкция | **Создать keystore**. Например, с помощью команды: \$JAVA_HOME/bin/keytool -genkey -alias tomcat -keyalg RSA. Указать пароль для доступа к ключу, желательно, «changeit».                                                                                                           |
|            | **Настроить коннектор** в файле server.xml. Обычно используется коннектор с портом 443 или 8443. Нужно раскомментировать узел с портом и схемой https, указать имя файла хранилища ключей и пароль.                                                                                 |
|            | **Указать реализацию SSL**. Если при настройке коннектора указан generic protocol="HTTP/1.1", то реализация, используемая Tomcat, выбирается автоматически. При необходимости можно избежать автоматического выбора реализации — указать classname в атрибуте протокола коннектора. |
|            | **Перезапустить Tomcat** после изменений конфигурации. После этого должен быть доступен доступ к любому веб-приложению, поддерживаемому Tomcat, через SSL.                                                                                                                          |

### Как реализовать кластеризацию в Tomcat для обеспечения высокой доступности приложений?

| Под темы            | Заметка                                                                                                                                                                             |
| ------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Определение         | **Кластеризацию в Tomcat для обеспечения высокой доступности приложений** можно реализовать, настроив работу нескольких экземпляров Tomcat как кластера.                            |
| Это позволяет       | **Распределять входящие запросы** по нескольким узлам, чтобы ни один узел не перегружался.                                                                                          |
|                     | **Обеспечивать устойчивость к сбоям**: если один узел выходит из строя, оставшиеся узлы могут продолжать обрабатывать запросы без прерывания.                                       |
|                     | **Масштабировать инфраструктуру**: по мере роста приложения можно добавлять больше экземпляров Tomcat, чтобы эффективно обрабатывать увеличенный трафик.                            |
| Конфигурация        | Чтобы включить кластеризацию в Tomcat, нужно добавить элемент Cluster className="org.apache.catalina.ha.tcp.SimpleTcpCluster"/в элемент Engine или Host.                            |
| Варианты репликации | **DeltaManager** — каждая сессия реплицируется на все остальные узлы. Такой подход хорошо работает для небольших кластеров (до 4 узлов), но не рекомендуется для больших кластеров. |
|                     | **BackupManager** — реплицирует данные сессии только на один резервный узел, и только на те узлы, где приложение развёрнуто.                                                        |

### Как настроить Tomcat для использования внешнего источника данных (DataSource) для JNDI?

| Под темы   | Заметка                                                                                                                                                  |
| ---------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| В Tomcat   | **Объявить ресурс JNDI** в файле server.xml Tomcat. Например, для PostgreSQL или MySQL.                                                                  |
| Инструкция | **Имя ресурса** — например, jdbc/DatabaseName.                                                                                                           |
|            | auth — механизм аутентификации для кода приложения (может быть Application или Container);                                                               |
|            | type — полное имя класса Java, ожидаемое веб-приложением при поиске ресурса;                                                                             |
|            | maxActive — максимальное количество соединений с базой данных в пуле;                                                                                    |
|            | maxIdle — максимальное количество idle-соединений, которые должны храниться в пуле;                                                                      |
|            | maxWait — максимальное время ожидания, пока соединение с базой данных станет доступным.                                                                  |
|            | **Ссылаться на ресурс JNDI** из веб-контекста Tomcat.xml:  ResourceLink name="jdbc/DatabaseName" global="jdbc/DatabaseName" type="javax.sql.DataSource". |
| В Spring   | **Настроить JNDI DataSource** в приложении Spring, используя XML- или Java-подходы.                                                                      |
|            | **Инъектировать бин DataSource** в компоненты Spring для работы с источником данных, управляемым Tomcat.                                                 |
